# 3. 프로젝트 목표

## 3.1 핵심 목표

- 방문객 수가 많은 혼잡 관광지 대신 대체 가능한 소도시 여행지를 추천한다.
- 추천 1건은 소도시 1곳을 최종 목적지로 삼고, 생성 일정은 그 도시 서비스 범위 안에서 완성한다.
- 한국과 일본 소도시를 나라별 독립 트랙으로 제공하되, PoC는 한국 강원·경북 및 일본 관동 우선 수집 범위를 기준으로 검증하고 최종 목표는 한일 전국 도시로 확장한다.
- 여행 시기, 축제, 날씨, 이동 경로, 주변 관광 정보를 결합한 여정을 제안한다.
- RAG 기반 검색과 멀티스텝 에이전트로 추천 근거와 안정성을 확보한다.
- 추천 이유, 출처, 검증 상태를 함께 보여주는 설명 가능한 추천 경험을 제공한다.
- KICK(유사 취향 공개 일정·1탭 복제)을 P2 설계 범위에 포함해 선택 불안을 줄이고 저장·외부 링크 전환 가능성을 검증한다.

## 3.2 성공 기준

| 구분 | 기준 |
| --- | --- |
| 추천 경험 | 사용자가 국가와 여행 시기, 여행 일정 길이, 자연어 취향 또는 지도 마커를 입력하면 조건에 맞는 소도시 1곳과 day-by-day 일정이 생성된다. |
| 근거 제공 | 추천 결과에는 추천 이유, 지도 링크, 날씨 정보, 외부 탐색 링크, 출처·검증 상태 중 핵심 정보가 함께 제공된다. |
| KICK 전환 | 유사 취향 공개 일정이 노출되고 사용자가 1탭 복제 후 저장 또는 부분 수정으로 이어지는 흐름을 설계 산출물로 확보한다. |
| 범위 통제 | PoC에서는 P1 기능을 구현하고, P2(KICK 포함)~P3는 설계 중심으로 정리하며, P5는 제외 범위로 관리한다. |
| 확장성 | 신규 지역, 축제, 촬영지, 체험 정보, 외부 데이터 소스를 추가할 수 있는 공통 데이터 모델과 운영 구조를 유지한다. |
