# 7. 기능 구성

## 7.1 사용자 기능

- 선택 로그인 및 온보딩
- 온보딩에서 테마와 기본 국내 지역 선호 입력
- 자연어 기반 여행 조건 입력
- 국내 지역, 여행 시기, 여행 일정 길이를 먼저 고정한 뒤 취향과 Wikipedia 기반 월별 날씨 경향을 반영하는 추천
- 추천 여정 카드와 메인·서브 관광지 확인
- 식사 슬롯에서 식당 선택
- 지도, 마커 간 이동 동선, 월별 날씨 경향, 외부 링크 확인
- 추천 결과 비교 및 후속 질의
- 자연어 또는 관광지 목록 직접 선택 기반 일정 수정
- 공개 일정 참고, 1탭 복제 검토, 챗봇 부분 조정
- 저장, 피드백, 재방문 개인화

## 7.2 추천 기능

| 기능 | 설명 |
| --- | --- |
| 혼잡 회피 추천 | 혼잡도가 높은 유명 관광지 대신 대체 소도시와 방문 시간대를 제안 |
| 축제 기반 추천 | 한국 강원·경북의 덜 알려진 축제와 주변 관광 자원을 결합 |
| 날씨 기반 추천 | Wikipedia에서 추출한 월별 날씨 경향을 후보 스코어링과 계절 정합성 검증에 사용하고, 비·폭설·장마·태풍 등 월별 기상 영향이 있는 경우 대체 일정 추가 여부를 알림 팝업으로 확인 |
| 일정 구성 | 각 날마다 메인 관광지 1개와 서브 관광지를 구성하고, 장소 타입 균형과 식사 슬롯 분리를 유지 |
| 일정 수정 | 메인 관광지는 테마 유지 조건에서 교체하고, 서브 관광지는 테마 고정 없이 사용자가 자유롭게 지정 |
| 지도 이동 동선 | OpenRouteService Directions API로 일정 마커 간 이동 경로를 계산해 지도 위에 폴리라인으로 표시하고, Matrix API는 후보 장소 간 이동 시간·거리 비교 보조로 활용 |
| 설명 가능한 추천 | 추천 이유, 출처 링크, 갱신 시점, 검증 상태를 함께 제공해 데이터 기반 큐레이션임을 드러냄 |
| 미디어 촬영지 추천 | K-Drama 촬영지 등 국내 미디어 장소를 소도시 여정과 결합 |
| 철도 경로 안내 | 철도 경로, 주요 역, 환승, 주변 관광지를 함께 안내 |
| 지역 문화 체험 | 공예, 차, 지역 요리, 전통문화, 농어촌 활동 등 지역 문화를 체험할 수 있는 목적지 추천 |
| 스키 액티비티 | 스키장 접근성, 적설·날씨 조건, 강습·장비 정보를 기반으로 겨울 액티비티 추천 |
| 유사 취향 공개 일정 | 공개 일정의 유사성 근거를 과장하지 않고 보여주며, 1탭 복제와 저장 전환은 P2 설계에서 검토 |
| 타이밍·시즌성 넛지 | 국내 시즌성, 축제, 월별 날씨 정합성, 혼잡 분산 조건을 결합해 지금 갈 이유를 제시 |

## 7.3 운영·관리 기능

- 역할 기반 인증 및 인가
- 관광 운영자와 데이터 제공 기관의 데이터 제안
- 서비스 관리자의 데이터 검토, 승인, 반려, 수정 요청
- 데이터 제안·검토 이력 관리
- 공개 일정 운영, 신고, 익명화, 공개 범위 기준 검토
- 추천 정책, 공지사항, 데이터 갱신 상태 관리
- B2G 월간 큐레이션인 “이번 달 여행지” 운영 검토
- 소도시별 추천 노출 수, 클릭 수, 일정 생성·저장 등 파트너 리포트 확장 검토

## 7.4 비즈니스 모델

B2C·B2B·B2G는 서로 다른 수익화 단계가 아니라 수요 검증, 성과형 제휴, 공익 협력을 분리해 관리하는 사업 트랙이다. 가격 실험, 수익성 시뮬레이션, 파트너별 제안 전문은 상세 문서에 둔다.

B2C 과금·가격 모델과 B2G 영리화 시나리오는 대표 기획서 범위에서 제외한다. 비즈니스 모델 세부 내용은 보조 문서 `supplemental/business_model.md`에서 관리한다.

### B2C: 수요·가치 검증 트랙

1차 타깃은 한국 강원·경북 소도시 여행자다. 서비스 가치는 정보 열람이 아니라 제한된 시간 안에서 실행 가능한 일정, 이동 실패 방지, 예약 가능한 다음 행동에 둔다.

핵심 KPI는 다음과 같다.

- 일정 저장률
- 공유·내보내기율
- 예약 파트너 클릭률
- 재방문율

### B2B: 성과형 제휴 트랙

B2B는 취향·일정 맥락을 예약 가능한 상품 탐색으로 연결하고, 성과형 제휴 가능성을 검토하는 추천 중개 트랙이다.

| 사용자 | 우선 검토 제휴 |
| --- | --- |
| 한국 강원·경북 여행자 | 국내 숙박·액티비티·지도·탐색 플랫폼 |
| 한국 전국 확장 여행자 | 지역별 숙박·액티비티·교통 탐색 플랫폼 |

수익 인식 기준은 총 예약액보다 승인 매출 기준으로 검토한다. 성과 구간, CPA, 추천 모듈·API, 화이트라벨, 공동브랜드 같은 장기 확장 후보는 보조 문서에서 관리한다.

### B2G: 비과금 공익 협력 트랙

B2G는 지자체·공공기관과 검증된 관광 정보를 “이번 달 여행지” 월간 큐레이션으로 노출하는 비과금 협력이다.

성과 지표는 재정 수치보다 다음 비재정 지표를 우선한다.

- 노출
- 클릭
- 일정 생성
- 일정 저장
- 공식 정보 이동
- 방문 의향

공식 정보 링크와 제휴 링크는 분리한다. 관문도시 방문은 소도시 성과로 보지 않고, 지역 숙박·지역 상품 클릭·공식 정보 이동을 별도로 집계한다.
