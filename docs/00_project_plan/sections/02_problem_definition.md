# 2. 문제 정의

## 2.1 사용자 문제

| 문제 | 설명 |
| --- | --- |
| 유명 관광지 편중 | 여행자는 익숙한 대도시와 유명 관광지 위주로 정보를 찾게 된다. |
| 소도시 정보 부족 | 방문객 수가 낮은 관광지와 그 주변 소도시의 계절성, 축제, 이동 경로, 주변 관광 정보를 한 번에 파악하기 어렵다. |
| 여행 조건 비교 어려움 | 날씨, 교통, 혼잡도, 동행 유형, 체험 요소를 동시에 고려하기 어렵다. |
| 추천 신뢰도 부족 | 일반 생성형 답변은 출처와 근거가 약해 실제 여행 계획에 바로 쓰기 어렵다. |
| 선택 불안 | 낯선 소도시는 정보가 적어 실제 저장·예약·이동으로 전환되기 어렵다. |

## 2.2 시장·운영 문제

| 문제 | 설명 |
| --- | --- |
| 지역 관광 수요 불균형 | 일부 지역에 관광객이 집중되고 다른 지역은 노출 기회가 부족하다. |
| 데이터 분산 | 관광지, 축제, 교통, 날씨, 지도 정보가 여러 플랫폼에 흩어져 있다. |
| 운영 검증 필요 | 지자체·관광 운영자·데이터 제공 기관이 제안한 정보를 검토하고 반영할 흐름이 필요하다. |
| 성과 측정 어려움 | 관문 대도시 방문과 실제 지역 소도시 방문 준비 행동을 분리해 보지 않으면 수요 분산 효과를 잘못 판단할 수 있다. |
