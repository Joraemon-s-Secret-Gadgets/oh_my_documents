# 5. 주요 사용자 및 이해관계자

| 구분 | 역할 | 주요 니즈 |
| --- | --- | --- |
| 1차 B2C 사용자 | 한국 강원·경북 소도시 여행자 | 짧은 휴가 안에서 실패 가능성이 낮은 국내 소도시 여정 탐색 |
| 2차 B2C 사용자 | 한국 전국 소도시 확장 사용자 | 서울·부산 등 대도시 방문을 넘어 전국 지역 여행 후보 탐색 |
| 자유여행자 | 일반 여행 사용자 | 혼잡하지 않고 취향에 맞는 소도시 여행지 탐색 |
| 지자체·관광 운영자 | 관광 운영자 | 지역 관광 자원 노출, 수요 분산, 데이터 제안 |
| 예약 제휴사 | 국내 숙박·액티비티·지도·탐색 플랫폼 | 추천 맥락에서 예약 가능한 상품으로 이어지는 고의도 트래픽 확보 |
| 서비스 관리자 | 내부 운영자 | 추천 정책, 데이터 승인, 공개 일정 운영, 신고·익명화, 운영 상태 관리 |
| 공공 데이터 제공 기관 | 데이터 제공자 | 관광지·축제·교통 데이터 제공 및 갱신, Wikipedia 월별 날씨 경향 추출 근거 확인 |
| 외부 API/시스템 | 시스템 연동 주체 | 지도, 장소, 이동 동선, 외부 탐색 링크 제공 |

초행 여행자는 무료 정보로도 충분할 수 있어 후순위로 본다. 공통 우선 세그먼트는 짧은 휴가의 직장인·커플, 테마형 여행자, 재방문 경험이 있는 FIT 사용자이며, 연령·성별보다 방문 횟수, 여행 기간, 교통 허용도, 테마, 예약 시점으로 구분한다.

장기적으로 로브는 지자체·공공 데이터·현지 운영자와 함께 만드는 소도시 추천 플랫폼을 지향한다. 지역 파트너는 데이터 제안과 검증뿐 아니라 소도시별 추천 노출 수, 클릭 수, 일정 생성·저장 같은 운영 지표를 확인하는 대시보드로 확장할 수 있다.
