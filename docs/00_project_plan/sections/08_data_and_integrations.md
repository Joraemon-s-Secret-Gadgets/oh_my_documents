# 8. 데이터 및 외부 연동

## 8.1 필요 데이터

필요 데이터는 PoC 우선 데이터, 확장 후보 데이터, 재검토 데이터로 나눠 관리한다. PoC는 한국 강원·경북 소도시 추천과 일정 구성에 필요한 기본 정보, 축제·행사, 혼잡도·방문 통계, 월별 날씨 경향, 지도·장소, 출처·검증 정보, 일정 구성 후보를 우선한다.

| 구분 | 데이터 범위 | 활용 기준 |
| --- | --- | --- |
| PoC 우선 | 소도시 기본 정보, 축제·행사, 지도·장소 정보 | 목적지 설명, 시기별 추천, 주변 명소와 외부 탐색 링크 구성 |
| PoC 보정 | 혼잡도·방문 통계, Wikipedia 월별 날씨 경향 | 대체 여행지 추천, 수요 분산 근거, 계절 정합성 검증, 대체 일정 알림 |
| PoC 신뢰 | 출처·검증 정보, 일정 구성 후보 정보 | 출처·갱신 시점·검증 상태를 표시하고 DynamoDB 서브 관광지 후보, 장소 타입, 식사 슬롯 후보를 재사용 |
| P2 확장 | 한국 전국 소도시 데이터 | AdministrativeArea, City, Attraction, Festival, VisitorStatistics 공통 모델로 전국 확장 |
| P3 확장 | 미디어 촬영지, 철도·교통, 지역 문화 체험, 공개 일정 데이터 | 테마 추천, 이동 안내, 체험 후보, 익명 일정 복제·저장 전환 검증 |
| P5 재검토 | 일본 City·Attraction·Festival·VisitorStatistics | 수집 경로 복구와 로컬 검증 산출물 확보 후 재승격 검토 |

PoC는 한국 강원·경북을 우선 수집 범위로 삼고, 현재 범위는 전국 선제 구축이 아니라 검증 후보 도시를 깊게 구성하기 위한 출발점이다. 한국 전국 확장은 P2로 두며, 일본 데이터는 수집 실패 리스크가 해소될 때까지 P5로 보류한다.

## 8.2 외부 연동

외부 연동은 서비스 기능별로 지도·장소, 이동 경로, 관광·통계, 예약 탐색, 공공 관광 데이터, P5 일본 재검토 연동으로 나눠 관리한다. 클라이언트에서 직접 호출하기 어려운 API는 딥링크 또는 백엔드 프록시로 대체한다.

| 구분 | 연동 대상 | 운영 기준 |
| --- | --- | --- |
| 지도·장소 | Google Maps Platform / Kakao Maps | 지도 표시, 장소 검색, 외부 탐색 딥링크 연결 |
| 이동 경로 | OpenRouteService | 일정 마커 간 경로 폴리라인, 이동 시간·거리 계산, 후보 장소 간 Matrix 비교 |
| 관광·통계 | TourAPI / DataLab, 공공 관광 데이터 | 관광지·축제·방문 통계, 교통, 지역 공식 정보 보강 |
| 예약 탐색 | 국내 숙박·액티비티 플랫폼 | 예약·숙박·액티비티 탐색 링크 연결 |
| P5 재검토 | JNTO / 일본 관광청(JTA) / e-Stat / Yahoo Japan | 일본 방문 통계, 지역 검증, 현지 탐색 딥링크는 수집 경로 복구 후 검토 |

연동 운영 원칙은 다음과 같다.

| 운영 항목 | 기준 | 폴백·제한 |
| --- | --- | --- |
| 월별 날씨 경향 | Wikipedia 지역별 기후·날씨 표와 설명을 추출한 정적 데이터셋으로 관리 | 월별 경향 외 날씨 데이터는 호출하거나 화면에 표시하지 않음 |
| 대체 일정 | 비·폭설·장마·태풍 등 월별 여행 영향 감지 시 알림 팝업으로 추가 여부 확인 | 사용자 동의 시에만 실내·우천 적합 대체 일정 추가 |
| 경로 계산 | Directions API는 선택 일정 순서의 경로 표시, Matrix API는 서브 관광지 이동 부담 비교에 활용 | 호출 실패 시 직선 연결 또는 장소 목록 중심 UI로 폴백 |
| API Key | 백엔드 프록시 또는 서버 환경 변수에서 관리 | 클라이언트 직접 노출 금지 |
