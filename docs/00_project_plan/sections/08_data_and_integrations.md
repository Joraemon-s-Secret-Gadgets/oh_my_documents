# 8. 데이터 및 외부 연동

## 8.1 필요 데이터

우선 데이터는 소도시 기본 정보, 축제·행사, 혼잡도·방문 통계, 월별 기후 경향, 지도·장소, 출처·검증 정보로 구성한다. 실시간 날씨는 상세 화면 표시용으로 두고, 미디어 촬영지, 철도·교통, 지역 문화 체험, 공개 일정 데이터는 P2 이후 확장 데이터로 검토한다.

| 구분 | 데이터 | 활용 |
| --- | --- | --- |
| 우선 | 소도시 기본 정보 | 목적지 설명, 지역, 태그, 추천 이유 구성 |
| 우선 | 축제·행사 정보 | 시기별 추천과 주변 여정 구성 |
| 우선 | 혼잡도·방문 통계 | 대체 여행지 추천, 수요 분산 근거, 관문도시 성과 오인 방지 |
| 우선 | 월별 기후 경향 | 월별 추천 적합도와 계절 정합성 검증 |
| 우선 | 지도·장소 정보 | 위치 표시, 주변 명소, 외부 탐색 링크 |
| 우선 | 출처·검증 정보 | 데이터 출처, 갱신 시점, 검증 상태, 신뢰도 표시 |
| 표시 | 실시간 날씨 정보 | 목적지 상세 화면의 현재 날씨 표시 |
| 확장 | 미디어 촬영지 정보 | 작품명, 장소, 공식성, 신뢰도, 근거 URL |
| 확장 | 철도·교통 정보 | 주요 역, 환승, 소요 시간, 패스 활용 가능 여부 |
| 확장 | 지역 문화 체험 정보 | 체험 유형, 예약 필요 여부, 난이도, 우천 적합성 |
| 확장 | 공개 일정 데이터 | 익명화된 일정, 복제·저장 전환 측정 |

PoC는 한국 강원·경북과 일본 관동을 우선 수집 범위로 삼는다. 현재 범위는 전국 선제 구축이 아니라 검증 후보 도시를 깊게 구성하기 위한 출발점이다. 최종 목표는 한국·일본 전체 도시 확장이다. 확장 시에도 AdministrativeArea, City, Attraction, Festival, VisitorStatistics를 중심으로 한 공통 데이터 모델을 유지한다.

## 8.2 외부 연동

외부 연동은 지도·장소, 날씨, 관광·방문 통계, 예약 딥링크, 공공 관광 데이터로 나뉜다. 클라이언트에서 직접 호출하기 어려운 API는 딥링크 또는 백엔드 프록시로 대체한다.

| 출처 | 역할 |
| --- | --- |
| TourAPI / DataLab | 한국 관광지·축제·방문 통계 |
| JNTO / 일본 관광청(JTA) / e-Stat | 일본 방문 통계와 지역 검증 |
| WeatherAPI | 목적지 상세 화면의 실시간 현재 날씨 |
| 기상청 API허브 / 일본기상청(JMA) | 월별 기후 경향, ComfortScore, recommended_months |
| Google Maps Platform / Kakao Maps / Yahoo Japan | 지도, 장소, 외부 탐색 딥링크 |
| 클룩·라쿠텐 트래블·KKday | 예약·숙박·액티비티 탐색 링크 연결 |
| 공공 관광 데이터 | 관광지, 축제, 교통, 지역 공식 정보 |

WeatherAPI는 상세 화면에서 현재 날씨를 보여주는 역할이다. 월별 기후 경향, 추천 적기, 기후 정합성 검증은 기상청 API허브와 일본기상청(JMA)을 기준으로 한다. 두 데이터는 역할이 다르므로 함께 사용한다.
