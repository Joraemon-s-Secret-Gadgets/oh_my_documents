## KICK 정의

KICK은 나와 취향이 비슷한 사용자의 공개 일정을 보고, 1탭으로 복제한 뒤, 챗봇으로 일부 조건을 조정해 내 일정으로 저장하는 전환 장치다.

대표 기획서에서는 KICK을 추천 축 자체가 아니라 선택 불안을 줄이는 P2 설계 범위로 다룬다.

## 핵심 사용자 경험

1. 로그인·온보딩으로 기본 취향을 확보한다.
2. 공개 일정 카드에서 유사 취향 근거를 보여준다.
3. 사용자는 공개 일정을 1탭으로 복제한다.
4. 챗봇이 일정 길이, 이동 강도, 동행, 예산 등 일부 조건을 조정한다.
5. 복제본은 원본 공개 일정과 사본 계보를 가진 내 일정으로 저장된다.

## 원칙

- 공개 일정 부족 시 가짜 사용자 일정을 만들지 않는다.
- 초기에는 명시적인 `Lovv 큐레이션`으로 노출한다.
- 유사성 표현은 실제 일치 근거만 표시하고 과장하지 않는다.
- 공개 일정은 불변 스냅샷으로 보존하고, 복제본은 사본 계보를 가진다.
- 개인정보는 구간화, 선택 공개, 익명화, 최소 집단 크기 원칙을 적용한다.

## 성공 기준

- 공개 일정 노출
- 유사 취향 근거 확인
- 1탭 복제
- 챗봇 부분 수정
- 일정 저장
- 외부 링크 클릭
- 재방문
