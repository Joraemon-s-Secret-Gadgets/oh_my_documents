## 정리 원칙

- 대표 기획서와 이 보조 문서는 동일한 기능 범위 상세를 유지한다.
- 단계별 사용자 흐름, 추천 축별 설명, 운영 기능 목록은 대표 기획서 본문에도 동일하게 반영한다.
- 상세 명세서와 충돌하면 상세 명세서의 결정을 우선한다.

## 사용자 흐름

| 단계 | 사용자 입력 | 시스템 처리 | 결과 |
| --- | --- | --- | --- |
| 1 | 회원가입·로그인, 온보딩 | 테마·기본 국가 선호 저장 | 개인화 초기 조건 설정 |
| 2 | 카드 슬롯, 지도 마커, AI 챗봇 중 진입점 선택 | 진입점을 추천 컨텍스트로 변환 | 일정 생성 흐름 시작 |
| 3 | 국가, 여행 시기, 일정 길이 입력 | 필수 조건 고정 | 추천 대상 범위 설정 |
| 4 | 자연어 취향 입력 | 분위기·관심사·동행·이동 강도 등을 테마로 해석 | 추천 기준 생성 |
| 5 | 후보 비교 | RAG 후보 검색, 방문객 수, 계절, 축제, 월별 기상 경향, 개인화 피드백 기준 정렬 | 소도시 1곳 선정 및 일정 구성 |
| 6 | 추천 결과 확인 | day-by-day 일정, 추천 이유, 지도, 날씨, 외부 링크, 출처·검증 상태 제공 | 여행 계획 보조 |
| 7 | 공개 일정 복제 또는 후속 질의 | 유사 취향 공개 일정 1탭 복제, 조건 변경 기반 재검색 | 저장·부분 수정·재추천 |

## 사용자 기능

- 회원가입·로그인
- 온보딩에서 테마와 기본 국가 선호 입력
- 자연어 기반 여행 조건 입력
- 국가, 여행 시기, 여행 일정 길이를 먼저 고정한 뒤 취향, 날씨 조건을 반영하는 추천
- 추천 여정 카드 확인
- 지도, 날씨, 외부 링크 확인
- 추천 결과 비교 및 후속 질의
- 공개 일정 탐색, 1탭 복제, 챗봇 부분 조정
- 저장, 피드백, 재방문 개인화

## 추천 기능

| 기능 | 설명 |
| --- | --- |
| 혼잡 회피 추천 | 혼잡도가 높은 유명 관광지 대신 대체 소도시와 방문 시간대를 제안 |
| 축제 기반 추천 | 한국·일본의 덜 알려진 축제와 주변 관광 자원을 결합 |
| 날씨 기반 추천 | 일주일 이내 여행 조건에서는 예보를 활용하고, 월별 기후 경향은 정합성 검증에 활용 |
| 설명 가능한 추천 | 추천 이유, 출처 링크, 갱신 시점, 검증 상태를 함께 제공해 데이터 기반 큐레이션임을 드러냄 |
| 미디어 성지순례 | K-Drama 촬영지와 애니메이션 성지를 하나의 미디어 테마로 추천 |
| 철도 경로 안내 | 철도 경로, 주요 역, 환승, 주변 관광지를 함께 안내 |
| 지역 문화 체험 | 공예, 차, 지역 요리, 전통문화, 농어촌 활동 등 지역 문화를 체험할 수 있는 목적지 추천 |
| 스키 액티비티 | 스키장 접근성, 적설·날씨 조건, 강습·장비 정보를 기반으로 겨울 액티비티 추천 |
| 유사 취향 공개 일정 | 공개 일정의 유사성 근거를 과장하지 않고 보여주고 1탭 복제·저장으로 연결 |
| 타이밍·시즌성 넛지 | 한일 시즌성, 축제, 기후 정합성, 혼잡 분산 조건을 결합해 지금 갈 이유를 제시 |

## 운영·관리 기능

- 역할 기반 인증 및 인가
- 관광 운영자와 데이터 제공 기관의 데이터 제안
- 서비스 관리자의 데이터 검토, 승인, 반려, 수정 요청
- 데이터 제안·검토 이력 관리
- 공개 일정 운영, 신고, 익명화, 불변 스냅샷·사본 계보 관리
- 추천 정책, 공지사항, 데이터 갱신 상태 관리
- B2G 월간 큐레이션인 “이번 달 여행지” 운영
- 소도시별 추천 노출 수, 클릭 수, 일정 생성·저장 등 파트너 리포트 확장 검토
