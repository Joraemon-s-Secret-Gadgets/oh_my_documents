## 핵심 aha

Lovv의 핵심 aha는 “이런 곳이 있는 줄 몰랐는데, 지금 내 조건에 맞고, 진짜 갈 수 있겠다”이다.

이 문장은 세 요소로 나뉜다.

| 요소 | 의미 |
| --- | --- |
| 발견 | 사용자가 몰랐던 소도시를 만난다. |
| 납득 | 왜 이 도시가 지금 내 조건에 맞는지 이해한다. |
| 실행가능성 | 일정, 지도, 날씨, 외부 링크로 실제 행동이 가능해진다. |

## 추천 여정 funnel

| 단계 | 목표 |
| --- | --- |
| 0. 첫 진입 | 온보딩 직후 행동 이력이 없어도 타이밍·축제·혼잡 정보로 첫 발견을 만든다. |
| 1. 추천 기준 | 왜 이 도시인지 설명해 납득을 만든다. |
| 2. 후속 질의 | 세션 내 신호로 미니 개인화를 제공한다. |
| 3. 일정 생성·저장 | day-by-day 일정과 저장으로 실행가능성을 만든다. |
| 4. 재방문 | 선언·행동·세션 신호를 누적해 개인화 정확도를 높인다. |

## Cold Start 전략

행동 이력이 없어도 타이밍, 날씨, 축제, 혼잡은 누구에게나 현재 존재한다. Lovv는 초기 추천에서 이 공통 신호를 활용하고, 사용자의 세션 내 선택과 후속 질의를 빠르게 반영한다. 갔던 곳 기반 개인화는 재방문 단계의 무기로 분리한다.

## 취향 프로필

취향 프로필은 세 계층으로 다룬다.

- 선언 신호: 온보딩에서 입력한 테마·국가·여행 스타일
- 행동 신호: 저장, 클릭, 복제, 외부 링크 이동
- 세션 신호: 현재 대화에서 나온 여행 시기, 일정 길이, 동행, 이동 강도
