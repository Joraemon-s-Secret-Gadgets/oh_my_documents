# 로브 (Lovv) 서비스 흐름 명세서

> 문서 버전: v0.1
> 문서 상태: 기획 단계 (Planning)
> 기준 문서: `docs/01_requirements/01_requirements.md` v1.5

# 1. 문서 개요

## 1.1 목적

본 문서는 로브 서비스의 사용자 진입, 온보딩, 지도 탐색, 챗봇 추천, 결과 확인, 저장·피드백 흐름을 화면과 상태 전이 관점에서 정의한다.
요구사항 명세서의 기능 요구사항을 화면 흐름과 사용자 행동 단위로 연결해 UX 설계와 프론트엔드 구현의 기준으로 사용한다.

## 1.2 적용 범위

- 신규 사용자 필수 온보딩
- 메인 지도 기반 소도시 탐색
- 지도 마커 클릭 추천 진입
- 챗봇 직접 질의 추천 진입
- 축제·행사 포함 여부 확인
- 소도시 1곳 집중 일정 추천 결과
- 일정 저장, 좋아요/싫어요 피드백, 마이페이지 조회

# 2. 사용자 유형과 진입점

| 사용자 | 진입점 | 핵심 목적 |
| --- | --- | --- |
| 신규 자유여행자 | 온보딩 → 메인 지도 또는 챗봇 | 선호 테마를 설정하고 첫 추천을 받는다 |
| 재방문 자유여행자 | 메인 지도 또는 챗봇 | 저장 선호와 피드백을 반영한 추천을 받는다 |
| 관광 운영자 | 운영 화면 | 담당 지역 데이터와 제안 상태를 확인한다 |
| 서비스 관리자 | 관리자 화면 | 데이터 제안 승인, 추천 정책, 공지사항을 관리한다 |

# 3. 전체 서비스 흐름

| 단계 | 화면/상태 | 사용자 행동 | 시스템 처리 | 다음 상태 |
| --- | --- | --- | --- | --- |
| 1 | 서비스 진입 | 앱 또는 웹 첫 화면 접근 | 온보딩 완료 여부 확인 | 온보딩 또는 메인 |
| 2 | 온보딩 | 대도시 스타일 선택 | 선택지를 6개 기본 테마로 매핑 | 메인 지도 |
| 3 | 메인 지도 | 국가·월·테마 필터 조정 | 소도시 마커 목록 필터링 | 지도 탐색 |
| 4A | 지도 마커 팝업 | 소도시 마커 클릭 | 목적지 고정 | 일정 유형 선택 |
| 4B | 챗봇 | 자연어 조건 입력 | 조건 구조화 및 진입점 분류 | 축제 포함 여부 확인 |
| 5 | 추천 직전 확인 | 축제·행사 포함 여부 선택 | 세션 추천 조건에 반영 | 추천 생성 |
| 6 | 추천 결과 | 일정 카드 확인 | 소도시 1곳 일별 세부 일정 표시 | 저장·피드백 |
| 7 | 상세 패널 | 지도, 현재 날씨, 외부 링크 확인 | 외부 API/딥링크 정보 표시 | 결과 화면 유지 |
| 8 | 저장·피드백 | 저장, 좋아요/싫어요 선택 | 로컬 스토리지 또는 계정 저장 | 마이페이지/다음 추천 |

# 4. 온보딩 흐름

## 4.1 조건

- 온보딩은 신규 사용자에게 필수로 제공한다.
- PoC에서는 브라우저 로컬 스토리지에 완료 여부와 선호 프로필을 저장한다.
- Production에서는 로그인 사용자 계정에 선호 프로필을 저장한다.

## 4.2 질문 구조

| 단계 | 질문 | 선택 결과 |
| --- | --- | --- |
| 1 | 어떤 스타일의 여행지를 선호하나요? | 한국 또는 일본 대도시 스타일 선택 |
| 2 | 선택 확인 | 대도시 스타일을 기본 테마로 매핑 |
| 3 | 완료 | 선호 테마 프로필 저장 |

## 4.3 대도시 스타일 매핑

| 기본 테마 | 한국 예시 | 일본 예시 |
| --- | --- | --- |
| 온천·휴양 | 아산/온양 | 벳푸 |
| 바다·해안 | 부산 | 오키나와 |
| 역사·전통 | 경주 | 교토 |
| 미식·노포 | 전주 | 오사카 |
| 자연·트레킹 | 제주 | 닛코 |
| 예술·감성 | 강릉 | 가나자와 |

# 5. 지도 진입 흐름

## 5.1 메인 지도 상태

메인 지도는 추천 대상 소도시를 마커로 표시한다.
사용자는 국가, 월, 테마 필터를 조합해 마커를 좁힐 수 있다.

| UI 요소 | 동작 |
| --- | --- |
| 국가 필터 | 한국, 일본, 전체 중 선택 |
| 월 필터 | 1월~12월 중 선택 |
| 테마 필터 | 기본 테마와 확장 테마 선택 |
| 소도시 마커 | 클릭 시 목적지 후보로 고정 |

## 5.2 마커 클릭 후 흐름

| 단계 | 사용자 행동 | 시스템 처리 |
| --- | --- | --- |
| 1 | 마커 클릭 | 소도시명, 대표 태그, 추천 월 표시 |
| 2 | 일정 유형 선택 | 당일치기~4박5일 중 하나를 추천 조건에 추가 |
| 3 | 축제·행사 포함 여부 선택 | 세션 조건에만 저장 |
| 4 | 추천 시작 | 해당 소도시 1곳의 일별 세부 일정 생성 |

# 6. 챗봇 진입 흐름

## 6.1 자연어 입력 예시

| 입력 예시 | 구조화 조건 |
| --- | --- |
| "6월에 비 덜 오는 일본 소도시 추천해줘" | 국가=일본, 월=6월, 진입점=월별 기상 경향 |
| "혼자 조용히 쉬는 바다 여행 가고 싶어" | 동행=혼자, 테마=바다·해안/휴양 |
| "가을에 축제 있는 한국 소도시 알려줘" | 국가=한국, 시기=가을, 축제 포함 후보 |

## 6.2 챗봇 추천 생성 전 필수 확인

챗봇은 추천 생성 직전 아래 항목을 확인한다.

| 확인 항목 | 기본 처리 |
| --- | --- |
| 국가 | 명시되지 않으면 한국/일본 중 선택 질문 |
| 여행 시기 | 월 또는 계절 입력 요청 |
| 일정 유형 | 명시되지 않으면 당일치기~4박5일 선택 질문 |
| 축제·행사 포함 여부 | 예/아니오 선택 질문 |

# 7. 추천 결과 흐름

## 7.1 결과 카드 구성

| 영역 | 표시 내용 |
| --- | --- |
| 추천 대상 | 소도시명, 국가, 지역, 대표 이미지 |
| 추천 이유 | 사용자 조건, 계절 적합도, 혼잡도, 월별 기상 경향 중 2개 이상 |
| 일정 | 일정 유형에 맞춘 일별 방문 순서 |
| 축제·행사 | 사용자가 포함을 선택한 경우 해당 월 행사 정보 |
| 외부 링크 | 지도, 공식 관광 정보, 숙소 검색 링크, 현지 플랫폼 링크 |
| 피드백 | 좋아요, 싫어요, 저장 |

## 7.2 대체 일정 토글

월별 기상 경향상 우천, 폭설, 장마, 태풍 가능성이 높은 목적지는 "기상 악화 시 대체 일정 보기" 토글을 제공한다.
토글을 활성화하면 동일 목적지 안에서 실내 중심 일정으로 전환한다.

# 8. 저장·피드백·마이페이지 흐름

| 기능 | PoC | Production |
| --- | --- | --- |
| 일정 저장 | 로컬 스토리지 | 로그인 계정 서버 저장 |
| 온보딩 선호 | 로컬 스토리지 | 사용자 프로필 저장 |
| 좋아요/싫어요 | 로컬 스토리지 | 추천 피드백 테이블 저장 |
| 마이페이지 | 기본 조회 | 일정 상세, 선호 재설정, 피드백 이력 |

# 9. 예외 흐름

| 상황 | 처리 |
| --- | --- |
| 필수 조건 누락 | 챗봇이 국가, 월, 일정 유형을 추가 질문 |
| 추천 후보 없음 | 조건 완화 제안 및 다른 월/테마 안내 |
| 외부 API 실패 | 기본 데이터 표시 후 현재 날씨 또는 장소 상세 미제공 안내 |
| 로컬 저장 실패 | 저장 실패 메시지와 재시도 제공 |
| 권한 없는 운영 화면 접근 | 로그인 또는 접근 불가 안내 |
