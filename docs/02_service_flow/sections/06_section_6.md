# 6. 목적지 탐색 흐름

## 6.1 메인 탐색 상태

메인 화면은 국가, 여행 시기, 테마, 자연어 취향을 조합해 추천 후보를 좁히는 탐색 워크스페이스다.
지도 마커와 후보 비교 카드는 같은 조건 상태를 공유한다.

| UI 요소 | 동작 | 상태 |
| --- | --- | --- |
| 국가 필터 | 한국, 일본, 전체 중 선택 | `country` |
| 시기 필터 | 월 또는 계절 선택 | `travelMonth` 또는 `season` |
| 테마 필터 | 기본 테마와 확장 테마 선택 | `themes` |
| 자연어 입력 | "드라마 촬영지 중심의 조용한 곳" 같은 조건 입력 | `naturalLanguageQuery` |
| 소도시 후보 카드 | 추천 이유, 혼잡도, 계절성, 출처 뱃지 비교 | `candidateDestinations` |
| 소도시 마커 | 클릭 시 목적지 후보로 고정 | `selectedDestination` |

## 6.2 후보 비교 후 흐름

| 단계 | 사용자 행동 | 시스템 처리 | 다음 상태 |
| --- | --- | --- | --- |
| 1 | 국가와 시기 입력 | 목적지 DB와 월별 기상 경향 기준으로 1차 필터링 | 테마 설정 |
| 2 | 자연어 또는 테마 칩 입력 | RAG 검색과 스코어링으로 후보군 생성 | 후보 비교 |
| 3 | 후보 카드 또는 지도 마커 클릭 | 소도시명, 대표 태그, 추천 월, 검증 상태 표시 | 소도시 확정 |
| 4 | AI 일정 짜기 클릭 | 챗봇 세션과 PlanDraft 초기 상태 생성 | 대화형 일정 생성 |
