# 8. 추천 결과 흐름

## 8.1 결과 카드 구성

| 영역 | 표시 내용 |
| --- | --- |
| 추천 대상 | 소도시명, 국가, 지역, 대표 이미지 |
| 추천 이유 | 사용자 조건, 계절 적합도, 혼잡도, 월별 기상 경향 중 2개 이상 |
| 검증 상태 | 출처, 최신성, 축제 날짜 검증 상태, 데이터 결측 안내 |
| 일정 | 일정 유형에 맞춘 일별·시간대별 방문 순서 |
| 축제·행사 | 사용자가 포함을 선택한 경우 해당 월 행사 정보. 날짜 미확정 축제는 확정 일정으로 배치하지 않는다 |
| 외부 링크 | 지도, 공식 관광 정보, 숙소 검색 링크, 현지 플랫폼 링크 |
| 피드백 | 좋아요, 싫어요, 저장, 추천 다시 받기 |

## 8.2 대체 일정 토글

월별 기상 경향상 우천, 폭설, 장마, 태풍 가능성이 높은 목적지는 "기상 악화 시 대체 일정 보기" 토글을 제공한다.
토글을 활성화하면 동일 목적지 안에서 실내 중심 일정으로 전환한다.
WeatherAPI 현재 날씨는 상세 화면 표시용으로만 사용하고 추천 스코어링 기준으로 사용하지 않는다.

## 8.3 실시간 피드백 반영

| 피드백 유형 | 처리 |
| --- | --- |
| 이동 부담 감소 | 동선 거리와 이동 횟수를 줄이고 일정 블록을 재배치한다 |
| 카페/맛집 추가 | 장소 후보를 추가하되 출처와 외부 탐색 링크를 함께 표시한다 |
| 축제 제외 | 축제 블록을 제거하고 같은 시간대 대체 방문지를 제안한다 |
| 우천 대비 | 실내 중심 대체 일정 토글을 기본 활성화한다 |
| 추천 다시 받기 | 현재 조건을 유지하되 직전 추천 목적지는 제외하거나 감점한다 |
