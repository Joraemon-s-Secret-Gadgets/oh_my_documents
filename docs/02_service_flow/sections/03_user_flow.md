# 3. User Flow

| 단계 | 화면 / 상태 | 사용자 행동 | 시스템 반응 | 사용자에게 보이는 결과 | 완료 조건 |
| --- | --- | --- | --- | --- | --- |
| 1 | 진입 및 인증 | 사용자가 Lovv에 접속 후 소셜/자체 로그인을 진행한다 | 백엔드가 OAuth 또는 자체 로그인 인증 후 Access Token과 Refresh Token을 발급하고 사용자 상태, 기존 취향, 저장 일정을 로드한다 | 로그인 세션이 유지된 맞춤형 메인 대시보드가 노출된다 | 로그인 완료 및 메인 화면 진입 |
| 2 | 목적지 탐색 1단계 | "언제, 어느 나라로 떠나세요?" 질문에 국가와 시기를 입력한다 | 입력된 시기의 공공 관광 데이터, 혼잡도, 계절성 데이터를 기준으로 후보 필터링 조건을 준비한다 | 국가/시기 선택 완료 상태가 UI에 반영된다 | 국가 및 시기 확정 |
| 3 | 테마 설정 2단계 | 자연어 또는 테마 태그 칩으로 취향을 입력한다 | RAG 엔진이 벡터 DB에서 조건에 맞는 소도시 후보를 검색하고 스코어링한다 | 대도시와 유사하지만 덜 붐비는 소도시 후보군 비교 카드가 노출된다 | 추천 소도시 리스트 확인 |
| 4 | 소도시 확정 및 챗봇 | 추천된 소도시 중 1곳을 선택하고 AI 일정 짜기에 진입한다 | 대화형 워크스페이스가 열리고 선택 소도시의 날씨 경향, 축제 정보, 검증 상태를 초기 컨텍스트에 주입한다 | "선택한 도시의 현지 축제 정보를 일정에 포함할까요?" 안내 메시지가 노출된다 | 대화 시작 상태 진입 |
| 5 | 동적 일정 생성 3단계 | 사용자가 기간, 이동 성향, 축제 포함 여부 등 세부 요구사항을 입력한다 | 멀티스텝 Agent가 외부 API와 내부 데이터를 조회하고 동선을 최적화한 뒤 Structured Output 형태로 일정 초안을 생성한다 | 추천 이유와 데이터 출처 뱃지가 포함된 시간대별 타임라인 UI가 생성된다 | 초안 일정 및 근거 확인 |
| 6 | 실시간 피드백 및 조정 | "이 날은 덜 걷게 해줘", "카페 추가해줘" 등 추가 자연어를 입력한다 | Agent 파이프라인이 기존 PlanDraft와 신규 요구사항을 재평가하고 특정 일정 블록과 대체 일정을 갱신한다 | 실시간 갱신된 타임라인과 실내 중심 대체 일정 토글을 확인한다 | 만족하는 일정 확정 |
| 7 | 일정 저장 및 연동 | 일정을 마이페이지에 저장하거나 맛집/숙소 상세 정보를 클릭한다 | 서버가 완성된 일정과 저장 범위 내 선호·피드백 데이터를 저장하고 외부 플랫폼 딥링크를 생성한다 | 내 저장 목록에 일정이 반영되고 현지 신뢰 플랫폼으로 이동할 수 있다 | 외부 예약 이동 또는 마이페이지 저장 완료 |
