# 2. 사용자 유형과 진입점

| 사용자 | 진입점 | 핵심 목적 |
| --- | --- | --- |
| 신규 자유여행자 | 로그인 또는 비로그인 진입 → 온보딩 → 목적지 탐색 | 선호 테마를 설정하고 첫 추천을 받는다 |
| 재방문 자유여행자 | 세션 검증 → 메인 대시보드 또는 챗봇 | 저장 선호와 피드백을 반영한 추천을 받는다 |
| 관광 운영자 | 운영 화면 | 담당 지역 데이터와 제안 상태를 확인하고 신규 로컬 콘텐츠를 제안한다 |
| 서비스 관리자 | 관리자 화면 | 데이터 제안 승인, 추천 정책, 검증 뱃지, 공지사항을 관리한다 |
