# 로브 (Lovv) Agent 명세서

> 문서 버전: v0.1
> 문서 상태: 기획 단계 (Planning)
> 기준 문서: `docs/01_requirements/01_requirements.md` v1.5

# 1. 문서 개요

## 1.1 목적

본 문서는 로브 추천 Agent의 역할, 단계별 입력·출력, 도구 사용, 폴백, 검증 기준을 정의한다.
Agent는 단일 LLM 호출이 아니라 조건 분류, 검색, 선정, 일정 구성, 설명 생성으로 나뉜 멀티스텝 파이프라인으로 동작한다.

# 2. Agent 목표

| 목표 | 설명 |
| --- | --- |
| 조건 구조화 | 자연어와 지도 진입 조건을 추천 조건 객체로 변환 |
| 근거 기반 추천 | RAG 검색 결과와 정적 데이터를 근거로 소도시 1곳 선정 |
| 개인화 반영 | 온보딩 선호와 좋아요/싫어요 피드백 반영 |
| 일정 구성 | 일정 유형에 맞는 일별 세부 일정 생성 |
| 설명 가능성 | 추천 이유와 일정 흐름 이유를 자연어로 제공 |
| 안전한 폴백 | 데이터 부족, API 실패, 조건 충돌 시 대체 안내 |

# 3. Agent 입력

| 입력 | 출처 | 설명 |
| --- | --- | --- |
| naturalLanguageQuery | 챗봇 | 사용자 자연어 요청 |
| entryType | UI | chatbot 또는 map_marker |
| country | UI/챗봇 | KR 또는 JP |
| travelMonth | UI/챗봇 | 1~12월 |
| destinationId | 지도 | 마커 클릭 시 고정 목적지 |
| tripType | UI/챗봇 | daytrip~5d4n |
| includeFestivals | UI/챗봇 | 축제·행사 포함 여부 |
| onboardingProfile | 저장 데이터 | 기본 선호 테마 |
| feedbackHistory | 저장 데이터 | 좋아요/싫어요 이력 |

# 4. Agent 출력

| 출력 | 설명 |
| --- | --- |
| selectedDestination | 선정된 소도시 1곳 |
| itinerary | 일정 유형에 맞춘 일별 세부 일정 |
| recommendationReasons | 사용자 조건, 계절, 혼잡도, 기상 경향 기반 추천 이유 |
| itineraryFlowReason | 일정 순서와 동선 흐름 이유 |
| alternativeItinerary | 기상 악화 시 실내 중심 대체 일정 |
| externalLinks | 지도, 숙소 검색, 현지 탐색 링크 |
| confidence | 추천 신뢰도와 데이터 결측 안내 |

# 5. 파이프라인

| 단계 | Agent 역할 | 입력 | 출력 |
| --- | --- | --- | --- |
| 1 | Condition Parser | 자연어, UI 조건 | 구조화된 여행 조건 |
| 2 | Entry Classifier | 구조화 조건 | map_marker, weather_first, theme_first 등 |
| 3 | Retriever | 국가, 월, 테마 | 후보 소도시 목록 |
| 4 | Ranker | 후보, 혼잡도, 기상 경향, 피드백 | 상위 1곳 |
| 5 | Itinerary Planner | 선정 소도시, 일정 유형, 축제 포함 여부 | 일별 일정 |
| 6 | Explanation Writer | 조건, 점수, 일정 | 추천 이유와 일정 흐름 이유 |
| 7 | Output Validator | 최종 결과 | 누락·환각·정책 위반 검증 |

# 6. 단계별 명세

## 6.1 Condition Parser

| 항목 | 내용 |
| --- | --- |
| 책임 | 자연어에서 국가, 월, 테마, 동행, 일정 유형을 추출 |
| 필수 출력 | `country`, `travelMonth`, `tripType` |
| 폴백 | 필수 값 누락 시 추가 질문 생성 |

## 6.2 Retriever

| 항목 | 내용 |
| --- | --- |
| 검색 대상 | 목적지 기본 정보, 테마 태그, 계절 적합도, 월별 기상 경향, 축제 정보 |
| 제약 | 한국 요청에는 한국 데이터만, 일본 요청에는 일본 데이터만 사용 |
| 출력 | 후보 소도시 N개와 근거 데이터 |

## 6.3 Ranker

| 점수 요소 | 방향 |
| --- | --- |
| 사용자 조건 매칭 | 높을수록 가점 |
| 계절 적합도 | 높을수록 가점 |
| 혼잡도 | 낮을수록 가점 |
| 월별 기상 악화 경향 | 높을수록 감점 |
| 좋아요 테마 태그 | 가점 |
| 싫어요 소도시 | 감점 |

## 6.4 Itinerary Planner

| 일정 유형 | 구성 기준 |
| --- | --- |
| 당일치기 | 핵심 명소 2~3개, 이동 부담 최소화 |
| 1박2일 | 대표 명소, 식사, 휴식, 숙소 검색 링크 |
| 2박3일 이상 | 일자별 테마 분리, 과도한 이동 방지 |

## 6.5 Explanation Writer

추천 이유는 아래 항목 중 2개 이상을 포함한다.

- 사용자 조건과의 매칭
- 계절 적합도
- 혼잡 회피 근거
- 월별 기상 경향
- 일정 흐름과 이동 효율

# 7. 도구 및 데이터 사용

| 도구/데이터 | 사용 목적 |
| --- | --- |
| Destination Search | 후보 검색 |
| Weather Monthly Trends | 추천 스코어링과 대체 일정 분기 |
| Festival Search | 축제 포함 일정 구성 |
| Map Link Builder | 지도 링크 생성 |
| Stay Link Builder | 숙소 검색 링크 생성 |
| WeatherAPI Proxy | 현재 날씨 표시용 데이터 조회 |

# 8. 금지 사항

- 대화 로그 전문을 장기 저장하지 않는다.
- WeatherAPI 실시간 예보를 추천 후보 스코어링 기준으로 사용하지 않는다.
- 출처 없는 장소 정보를 확정 사실처럼 제시하지 않는다.
- 한국 요청에 일본 목적지를 섞거나, 일본 요청에 한국 목적지를 섞지 않는다.
- 숙소를 직접 추천하지 않고 검색 링크만 제공한다.

# 9. 품질 검증 기준

| 검증 항목 | 기준 |
| --- | --- |
| 조건 충족 | 국가, 월, 일정 유형이 결과에 반영됨 |
| 근거성 | 추천 이유가 데이터 근거와 연결됨 |
| 단일 목적지 | 최종 추천은 소도시 1곳 중심 |
| 설명 가능성 | 추천 이유와 일정 흐름 이유가 모두 존재 |
| 안전성 | 결측 데이터와 API 실패를 안내 |
