# 로브(Lovv) B2B 제휴 및 수익화 전략 — 클룩·라쿠텐 트래블·KKday

> 문서 버전: v0.3
> 문서 상태: 검토 중 (Review)
> 작성일: 2026-06-18
> 조사 기준일: 2026-06-18
> 적용 범위: 여행 추천에서 외부 예약으로 이어지는 B2B 제휴 모델과 초기 사업성 검증

## 1. 결론 요약

로브의 초기 B2B 모델은 **여행자의 취향과 일정 맥락을 예약 가능한 상품으로 연결하고, 실제 이용이 완료된 거래에 대해 성과형 수수료를 받는 추천 중개 모델**로 정의한다.

다만 “클룩과 라쿠텐 트래블은 예약만 제공하고 추천 서비스가 약하다”는 문장은 그대로 사용하기 어렵다. 클룩과 KKday는 액티비티 탐색·예약 기능을, 라쿠텐 트래블은 키워드·지역별 링크와 숙소/공실 API를 제공한다. 더 정확한 사업 기회는 다음과 같다.

> 세 사업자는 예약 재고와 거래 인프라에는 강하지만, 외부 서비스에서 수집한 취향·동행·일정·예산 맥락을 바탕으로 **여정 전체를 조합하고 선택 이유까지 설명한 뒤 적절한 예약 상품으로 넘기는 역할**은 제휴사가 수행해야 한다.

로브는 이 공백을 채워 다음 가치를 제공할 수 있다.

- 파트너에는 구매 의도가 높은 맥락형 트래픽과 신규 예약을 제공한다.
- 사용자에게는 검색 결과 나열이 아닌 취향 기반 일정과 예약 가능한 다음 행동을 제공한다.
- 로브에는 낮은 초기 운영 위험으로 검증 가능한 성과형 매출원을 만든다.

수익성은 라쿠텐 트래블의 공개 요율과 클룩·KKday의 실제 계약 조건을 기준으로 검증한다. 초기에는 제휴 수수료로 전환을 확인하되, 장기적으로는 `성과 수수료 + B2B 추천 모듈/API 사용료 + 공동 마케팅비`의 혼합 모델을 지향한다.

## 2. 공식 조사로 확인한 사실

### 2.1 클룩

| 확인 항목 | 공식 자료에서 확인한 내용 | 사업적 해석 |
| --- | --- | --- |
| 제휴 수익 | 제휴사가 보낸 트래픽을 통해 예약되고 실제 이용 완료된 여행 상품에 대해 커미션 지급 | 클릭이나 예약 생성만으로 매출이 확정되지 않으며 취소·미이용을 반영해야 함 |
| 어트리뷰션 | 사용자가 제휴 경로로 클룩을 방문한 뒤 30일 이내 생성한 예약이 커미션 대상 | 일정 저장 후 나중에 예약하는 사용자까지 추적할 여지가 있음 |
| 공개 수수료율 | 공식 공개 페이지에서 단일 고정 요율을 확인할 수 없음 | 가입 승인 후 상품·국가별 실제 조건 확인 필요 |
| 제휴 도구 | 전용 코드, 위젯, 검색 박스, 배너, 추천 콘텐츠, 대시보드 제공 | 단순 링크 제공만으로는 차별화가 어려움 |
| 고급 B2B 연동 | 데이터 피드, API, 화이트라벨 제공 | 트래픽과 전환 실적을 증명하면 임베디드 예약 경험으로 확장 가능 |

**남아 있는 pain point**

- 제휴 도구는 제공하지만, 사용자의 취향·동행 구성·여행 기간·예산을 로브 수준으로 알고 있지는 않다.
- 액티비티별 탐색을 넘어 “이 여행자에게 이 일정 조합이 왜 맞는가”를 설명하는 여정 단위 추천이 필요하다.
- 제휴사는 어떤 콘텐츠와 상품을 어떤 사용자에게 노출할지 스스로 최적화해야 한다.

**로브의 해결 방식**

- 취향 프로필과 일정 슬롯을 기준으로 투어·티켓·교통 상품을 배치한다.
- 추천 이유, 이동 동선, 운영 시간과 일정 충돌 여부를 먼저 설명해 저품질 클릭을 줄인다.
- 상품 ID와 일정 문맥을 함께 기록해 어떤 추천 근거가 실제 완료 예약으로 이어졌는지 학습한다.

### 2.2 라쿠텐 트래블

| 확인 항목 | 공식 자료에서 확인한 내용 | 사업적 해석 |
| --- | --- | --- |
| 일본 제휴 요율 | 숙박·항공 등은 이용액의 1%, 렌터카는 2%, 관광 체험은 전용 링크 2%·공통 쿠키 경유 1% | 상품별 공개 요율이 다르므로 실제 연결 상품 기준으로 검토해야 함 |
| 성과 인정 | 제휴 링크 경유 후 첫 예약만 대상이며, 실제 출발·탑승·체크아웃 등 이용 완료가 필요 | 예약 생성 수가 아니라 유효 완료 예약과 승인 매출을 KPI로 삼아야 함 |
| 이용 완료 기한 | 예약일로부터 다다음 달 말까지 이용 완료되어야 성과 대상 | 장기 선예약 상품은 성과 누락 위험이 있음 |
| 다국어 사이트 | `travel.rakuten.com` 다국어 사이트는 2022-05-31 이후 제휴 성과 대상에서 제외 | 한국 사용자에게 영문/다국어 링크를 제공하는 모델은 현재 공개 정책상 위험 |
| 링크·API | 키워드·지역별 숙소 링크와 숙소 정보·공실 정보 API 제공 | 일본어 예약 경로와 정책을 전제로 한 딥링크/API PoC가 가능 |
| 콘텐츠 사용 | API 이미지에는 크레디트가 필요하고, 숙소 유도 외 용도 및 API 이미지의 SNS 게시 등에 제한 | 추천 카드의 이미지·가격 표시는 출시 전 정책 검토와 QA 필요 |

**남아 있는 pain point**

- 지역·키워드 검색과 API는 제공하지만 사용자의 여행 취향과 전체 일정에 맞는 숙소 선택 이유를 외부 제휴사가 설계해야 한다.
- 숙박과 투어·이동을 하나의 여정에서 연결하는 교차 카테고리 의사결정 지원이 제한적이다.
- 한국 사용자를 일본어 예약 경로로 전환할 때 언어와 결제 경험이 전환 저해 요인이 될 수 있다.

**로브의 해결 방식**

- 일정의 중심 지역, 이동 거리, 체크인/체크아웃 시각과 취향 테마를 숙소 후보 선정에 반영한다.
- “온천 중심”, “야간 이동 최소화”, “아이 동반”처럼 숙소 검색어보다 구체적인 선택 근거를 제공한다.
- 라쿠텐의 API·딥링크로 가격과 공실을 최종 확인하게 하고, 로브는 추천과 일정 조합에 집중한다.

### 2.3 한국인·일본인별 수요와 사용 규모

**측정 기준과 한계**

세 서비스가 국적별 MAU·예약자 수·시장점유율을 같은 기준으로 공개하지 않기 때문에 숫자를 한 줄로 직접 비교할 수는 없다. 이 문서에서는 증거를 다음 세 등급으로 구분한다.

| 증거 등급 | 의미 | 사용 방법 |
| --- | --- | --- |
| A | 표본과 설문 문항이 공개된 실제 이용률 조사 | 시장 침투도 판단에 직접 사용 |
| B | 회사 또는 신뢰할 수 있는 매체가 공개한 국적별 활성 사용자·예약 성장 | 규모와 성장 방향 판단에 사용 |
| C | 국가별 Apple App Store 누적 평가 수 | 실제 이용률이 아닌 앱 설치·사용 관심의 대체지표로만 사용 |

App Store 평가 수는 앱 출시 시점, iOS 점유율, 리뷰 요청 캠페인과 웹 예약 비중의 영향을 받는다. 따라서 **평가 수 2배를 사용자 수 2배로 해석해서는 안 된다.** 다만 동일 국가·유사 카테고리에서 공개 이용률이 없을 때 브랜드별 소비자 접점의 상대적 크기를 보는 보조 신호로는 사용할 수 있다.

**한국인 사용자**

한국인만을 대상으로 세 서비스를 같은 설문에서 비교한 최신 공개 이용률은 확인하지 못했다. 2026-06-18 기준 한국 App Store의 누적 평가 수는 다음과 같다.

| 서비스 | 한국 iOS 누적 평가 수 | 확인 가능한 수요 신호 | 판단 |
| --- | ---: | --- | --- |
| KKday | 29,857건 | 세 서비스 중 평가 수가 가장 많고 클룩의 약 2.9배 | 한국인 액티비티 수요에서 강한 후보. 클룩보다 우선 또는 최소 동시 검증 필요 |
| 클룩 | 10,292건 | 한국어 앱과 한국 지사를 운영하며, 2025년 소비자추천 글로벌 여행 플랫폼 부문 1위 수상 | 브랜드 인지도와 상품 폭은 강하지만 공개 이용률은 없음 |
| 라쿠텐 트래블 | 1,303건 | 한국 App Store에 글로벌 호텔 예약 앱이 제공됨 | 일본 숙박 목적의 틈새 수요는 있으나 한국 대중 시장 접점은 상대적으로 작음 |

2020년 발간된 한국소비자원 보고서는 당시 Google Play 설치 구간을 기준으로 KKday가 10만 건 이상이었다고 기록했지만 오래된 자료이므로 현재 규모 판단에는 사용하지 않는다. 현재 시점에는 App Store 평가 수와 향후 로브 자체 전환 실험을 우선한다.

**해석**

- 한국인 대상 액티비티 제휴는 `KKday 단독` 또는 `클룩 단독`으로 미리 결론내리기보다 동일 일정·동일 상품군에서 두 파트너를 병렬 비교해야 한다.
- 누적 평가 수만 보면 KKday의 소비자 접점이 더 크지만, 클룩은 교통·eSIM·렌터카까지 묶을 수 있어 객단가와 교차 판매에서 우위가 있을 수 있다.
- 라쿠텐 트래블은 한국인 전체가 아니라 `일본 숙박을 찾는 사용자`로 대상을 좁혀야 한다.

**일본인 사용자**

일본은 라쿠텐 트래블에 대해 실제 이용률 자료가 있고, 클룩과 KKday는 공개 시장점유율 대신 사용자 규모와 앱 평가 수를 참고할 수 있다.

| 서비스 | 실제 이용률·규모 근거 | 일본 iOS 누적 평가 수 | 판단 |
| --- | --- | ---: | --- |
| 라쿠텐 트래블 | MMD연구소 2024 조사에서 국내 온라인 여행 예약 이용 58.0%, 최다 이용 39.8%. 해외여행 예약에서도 이용 22.7%, 최다 이용 14.4% | 773,392건 | 일본인의 숙박·종합 여행 예약에서 명확한 1순위 |
| KKday | 일본경제신문이 2024-12 보도한 회사 자료 기준 글로벌 MAU 1,000만 명 이상 중 일본 비중 약 20%로, 단순 환산 시 약 200만 명 규모 | 36,260건 | 일본인 액티비티 시장에서 강한 소비자 기반. 클룩보다 우선 검증할 근거가 있음 |
| 클룩 | 일본인만의 공개 MAU·이용률은 확인하지 못함 | 8,253건 | 일본에서 사용되고 있으나 공개 소비자 접점은 KKday보다 작음. 도전자 채널로 평가 |

MMD연구소 조사는 2024-10-28~30에 일본 18~69세 7,000명을 예비 조사했고, 국내 여행 경험자 3,780명과 해외여행 경험자 2,011명의 온라인 예약 서비스 이용을 복수응답으로 집계했다. 따라서 라쿠텐 트래블 수치는 앱 사용률이 아니라 **온라인 여행 예약에서 해당 서비스를 이용한 비율**이다.

MMD연구소는 공개 조사 데이터의 상업적 판촉 이용 시 별도 문의를 요구한다. 이 문서의 수치는 내부 파트너 검토 근거로만 사용하고, 대외 영업자료·보도자료에 전재하기 전 이용 허가 조건을 다시 확인한다.

Apple App Store 평가 수에서는 라쿠텐 트래블이 KKday의 약 21배, KKday가 클룩의 약 4.4배다. 서비스 범위와 앱 역사가 달라 시장점유율로 환산할 수는 없지만, `일본 숙박=라쿠텐`, `일본인 액티비티=KKday 우선·클룩 비교군`이라는 파트너 구성을 지지한다.

**국가별 파트너 우선순위**

| 대상 사용자 | 숙박 우선 파트너 | 액티비티 우선 파트너 | 비교·보완 파트너 |
| --- | --- | --- | --- |
| 한국인 | 라쿠텐 트래블은 일본 숙박 한정 실험 | KKday와 클룩 동시 A/B 테스트 | 클룩의 교통·eSIM·렌터카 교차 판매 검증 |
| 일본인 | 라쿠텐 트래블 | KKday | 클룩을 동일 상품·가격 조건의 비교군으로 운영 |

**B2B 시사점**

1. 한국인과 일본인에게 동일한 파트너 우선순위를 적용하면 안 된다.
2. 일본인 대상 서비스는 라쿠텐 숙박과 KKday 액티비티를 한 일정에서 결합하는 모델이 가장 자연스럽다.
3. 한국인 대상 서비스는 KKday와 클룩의 실제 클릭·예약·이용 완료 수익을 직접 비교하기 전까지 독점 제휴를 피한다.
4. 파트너 평가는 MAU보다 `국적별 승인 EPC(클릭 1회당 승인 수익)`와 `일정 1건당 승인 GMV`를 최종 기준으로 삼는다.

### 2.4 권장 조합 기반 B2B 운영안

로브는 한 파트너의 상품을 일괄 노출하는 방식보다 **사용자 국적과 여행 단계에 따라 숙박·액티비티·이동 상품의 강점을 조합하는 오케스트레이션 사업자**로 포지셔닝한다.

| 대상과 여행 맥락 | 권장 조합 | 로브의 역할 | 초기 검증 포인트 |
| --- | --- | --- | --- |
| 한국인의 일본 여행 | `KKday·클룩 액티비티 병렬 + 라쿠텐 트래블 숙박 조건부 연결` | 동일 일정 슬롯에서 액티비티 후보를 비교하고, 일본 숙박 수요가 명확한 사용자에게만 라쿠텐 경로 제공 | 파트너별 전환, 언어 이탈, 숙박·액티비티 교차 예약 |
| 일본인의 일본 국내 여행 | `라쿠텐 트래블 숙박 + KKday 액티비티 + 클룩 비교군` | 숙박 위치와 체크인 시간을 기준으로 이동 가능한 액티비티를 결합 | 일정 단위 예약 완성률과 교차 카테고리 전환 |
| 일본인의 한국 여행 | `KKday 우선 액티비티 + 클룩 보완 상품 + 숙박 파트너 후속 확장` | 한국 현지 체험·교통·eSIM을 일정 맥락에 맞춰 묶고 일본어 추천 이유 제공 | 일본어 콘텐츠 반응, 카테고리별 파트너 우위 |

권장 조합은 다음 순서로 B2B 사업에 적용한다.

1. **제휴 포트폴리오 구성:** 숙박은 라쿠텐 트래블, 액티비티는 KKday와 클룩을 기본 후보로 두되 독점 계약은 초기 검증 이후 판단한다.
2. **일정 단위 상품 조합:** 개별 상품 광고가 아니라 `숙박 위치 → 이동 동선 → 시간대별 액티비티 → 교통·eSIM` 순서로 예약 가능한 여정을 만든다.
3. **파트너 자동 라우팅:** 국적, 언어, 상품 재고, 일정 적합도와 실제 완료 성과를 기준으로 동일 카테고리의 연결 파트너를 선택한다.
4. **공동 PoC 제안:** 파트너에는 단순 트래픽 공급이 아니라 특정 도시·코호트에서 일정 완성률과 추가 예약을 높이는 공동 실험으로 제안한다.
5. **B2B 솔루션 확장:** 검증된 조합 로직을 추천 모듈·API·공동 브랜드 플래너로 패키징해 여행사, 숙박사, 지역 관광기관으로 확장한다.

## 3. B2B 가치 제안

### 3.1 파트너가 사는 것은 추천 알고리즘이 아니라 증분 거래다

클룩과 라쿠텐 트래블에 제안할 핵심 문장은 다음과 같다.

> 로브는 아직 목적지와 상품을 확정하지 않은 사용자의 취향 데이터를 예약 가능한 일정으로 구조화해, 구매 의도가 형성된 순간 파트너의 적합한 상품 상세로 연결한다.

| 파트너 pain point | 로브의 제공 가치 | 측정 지표 |
| --- | --- | --- |
| 검색 전 단계의 잠재 수요를 직접 포착하기 어려움 | 취향 탐색 단계에서 여행 의도를 형성 | 추천 노출 대비 예약 의도 생성률 |
| 범용 상품 목록은 선택 피로가 큼 | 일정 문맥에 맞는 1~3개 후보와 이유 제시 | 외부 링크 CTR, 상품 상세 도달률 |
| 저품질 제휴 트래픽은 전환·승인율이 낮음 | 날짜·예산·동행 조건을 확인한 뒤 전송 | 클릭→예약률, 예약→이용 완료율 |
| 단일 상품 예약으로 객단가 확장이 제한됨 | 숙소·투어·교통을 여정 단위로 연결 | 세션당 예약 상품 수, GMV/세션 |
| 제휴사가 어떤 추천이 유효한지 알기 어려움 | 익명화된 코호트별 전환 인사이트 제공 | 취향/여정 유형별 승인 GMV |

### 3.2 사용자 경험 원칙

1. 추천 결과와 광고·제휴 관계를 명확히 표시한다.
2. 수수료가 높은 상품이 아니라 일정 적합도가 높은 상품을 우선한다.
3. 가격, 공실, 취소 조건은 파트너 페이지에서 최종 확인하도록 한다.
4. 한 일정 슬롯에 파트너 링크를 과도하게 배치하지 않는다.
5. 링크 실패나 재고 소진 시 대체 탐색 경로를 제공한다.

## 4. 수익 모델

### 4.1 1단계: 성과형 제휴 수수료

사용자가 로브의 추적 가능한 링크를 통해 이동하고, 파트너 정책이 정한 기간 안에 예약 및 이용을 완료하면 거래액 또는 인정 매출의 일정 비율을 받는다.

```text
월 제휴 매출
= 여행 의도 세션 수
× 예약 추천 노출률
× 외부 링크 클릭률
× 클릭 후 예약 전환율
× 평균 예약금액
× 수수료율
× 이용 완료·승인율
```

계약과 문서에서는 **성과형 제휴 수수료**, **예약 중개 수수료** 또는 **커미션**이라는 표현을 사용한다. 로브가 결제 당사자가 아니라면 총 예약액을 로브 매출로 인식하지 않는다.

### 4.2 2단계: 성과 구간별 우대 요율

- 월 승인 GMV 또는 완료 예약 건수에 따른 요율 상향
- 특정 국가·도시·비수기 상품의 캠페인 보너스
- 신규 고객 유입에 대한 정액 CPA 추가
- 숙박 후 액티비티 또는 액티비티 후 숙박의 교차 판매 보너스

### 4.3 3단계: B2B 추천 인프라

- 파트너 앱/웹에 로브 추천 모듈을 제공하는 월 구독 또는 API 사용료
- 공동 브랜드 여행 플래너의 구축·운영비
- 익명·집계된 수요 인사이트 리포트
- 기본 사용료에 성과 수수료를 더한 하이브리드 계약

개인정보 원문이나 민감한 취향 데이터를 판매하지 않는다. B2B 데이터 상품은 최소 집단 크기를 충족한 익명·집계 통계로 제한한다.

### 5.4 손익 판단 기준

제휴 매출에서 다음 변동비를 뺀 금액을 공헌이익으로 본다.

- 추천 생성과 재랭킹에 드는 모델/API 비용
- 가격·공실 조회 및 데이터 가공 비용
- 링크 생성, 어트리뷰션, 정산 대사 운영비
- 고객 문의와 링크 오류 대응비
- 파트너별 캠페인 콘텐츠 제작비

초기 Go 기준은 `승인 예약 1건당 제휴 수익 > 추천·운영 변동비`이며, 이후 인건비와 고정비를 포함한 손익분기 GMV를 산정한다.

## 6. 권장 MVP

### 6.1 1차 범위

- 파트너: 한국인 사용자에게 클룩과 KKday를 동일 조건으로 병렬 연결하고, 라쿠텐 트래블은 일본 숙박·일본인 사용자 코호트에서 별도 실험한다.
- 지역: 도쿄 또는 오사카처럼 투어와 숙박 재고가 충분한 1개 도시
- 상품: 일정 슬롯과 직접 연결되는 액티비티 20~50개
- 노출: 일정 상세에서 최대 3개 상품과 추천 이유 제공
- 추적: `partner`, `campaign`, `itinerary_id`, `slot_id`, `recommendation_version`을 비식별 추적 파라미터로 관리
- 최종 가격·재고·취소 규정: 파트너 페이지를 기준으로 표시

한국인 액티비티 MVP에서 라쿠텐 트래블을 첫 파트너로 삼지 않는 이유는 현재 공개 정책상 다국어 사이트가 성과 대상에서 제외되어 있고, 한국 사용자에게 일본어 예약 경로를 제공할 때 전환 손실을 먼저 확인해야 하기 때문이다. 일본인 사용자 대상 MVP에서는 실제 이용률이 높은 라쿠텐 트래블을 숙박의 첫 파트너로 둔다.

### 6.2 8주 검증 계획

| 기간 | 실행 | 통과 기준 예시 |
| --- | --- | --- |
| 1~2주 | 제휴 승인, 딥링크·정산 조건 확인, 이벤트 스키마 구현 | 테스트 클릭과 주문 어트리뷰션 확인 |
| 3~4주 | 도쿄/오사카 상품 큐레이션과 일정 슬롯 연결 | 링크 유효율 98% 이상 |
| 5~6주 | 추천 이유·CTA A/B 테스트 | 추천 노출 대비 외부 CTR 10% 이상 |
| 7~8주 | 예약·완료·승인 매출 대사 | 클릭→예약 2% 이상, 승인율 75% 이상 |

위 임계값은 확정 KPI가 아니라 초기 가설이다. 실제 트래픽 규모와 상품군별 기준선을 확보한 뒤 조정한다.

## 7. 파트너별 제안안

### 7.1 클룩 제안

**제안 제목:** 취향 기반 일정에서 액티비티 완료 예약까지 연결하는 고의도 트래픽 파트너십

- 로브의 일정 슬롯과 클룩 상품을 매칭하는 딥링크 PoC
- 도시·취향 코호트별 클릭, 예약, 완료 데이터를 공동 분석
- 30일 어트리뷰션을 활용한 저장 후 재방문 예약 흐름 검증
- 기준 GMV 달성 시 성과 보너스 또는 API/화이트라벨 연동 협상

**클룩에 확인할 항목**

- 국가·카테고리별 실제 커미션율과 성과 보너스
- 딥링크가 유지하는 추적 파라미터와 앱 전환 시 어트리뷰션
- 취소, 부분 환불, 노쇼, 쿠폰 사용 시 인정 매출
- 상품·가격·이미지 API 이용 조건과 캐시 허용 시간
- 서브 ID 단위 리포트와 서버 간 전환 포스트백 지원 여부

### 7.2 라쿠텐 트래블 제안

**제안 제목:** 취향·동선·일정 제약을 반영한 일본 숙소 추천과 예약 전환 PoC

- 일본어 예약 경로를 사용할 의향이 있는 사용자 코호트로 제한
- 숙소·공실 API와 제휴 딥링크를 이용한 도쿄 1개 지역 실험
- 체크인 위치와 액티비티 동선을 결합해 검색보다 적은 후보 제공
- 숙박 1% 공개 요율을 기준선으로 하되, B2B 법인 제휴와 별도 정산 가능성 문의

**라쿠텐 트래블에 확인할 항목**

- 한국 법인/서비스의 일본 제휴 프로그램 가입 가능 여부
- 해외 숙박과 일본 국내 숙박의 최신 대상 범위
- 다국어 또는 한국어 예약 페이지의 성과 인정 대안
- 첫 예약 규칙과 다른 라쿠텐 캠페인이 기여도를 덮어쓰는 조건
- API 이미지, 가격, 공실 캐시와 크레디트 표시 세부 규정
- 현금 정산, 세금계산, 통화와 최소 지급 조건

| 마지막 클릭 기여도 상실 | 로브 추천의 매출 미인정 | 서브 ID, 딥링크, 포스트백 검증 및 파트너별 어트리뷰션 문서화 |
| 추천의 상업적 편향 | 사용자 신뢰 하락 | 제휴 표시, 적합도 우선 랭킹, 비제휴 대안 허용 |
| 가격·재고 불일치 | CS와 이탈 증가 | 짧은 캐시, 조회 시각 표시, 파트너에서 최종 확인 |
| 라쿠텐 언어·지역 정책 | 한국 사용자 전환 저하 | 일본어 수용 코호트 실험 후 확대 여부 결정 |
| 이미지·상표 정책 위반 | 제휴 중단 | 파트너 제공 소재만 사용하고 배포 채널별 이용 규정 QA |

## 9. 의사결정

### 지금 할 것

1. 클룩 제휴 프로그램에 가입해 실제 요율과 딥링크·리포트 기능을 확인한다.
2. 액티비티 20~50개로 한 도시의 일정 슬롯 연결 PoC를 구현한다.
3. `클릭 → 예약 생성 → 이용 완료 → 승인 매출` 퍼널을 분리 계측한다.
4. 1% 미만의 실효 요율이면 제휴 링크 단독 모델을 중단하거나 최소 보장/B2B 사용료를 협상한다.
5. 라쿠텐 트래블은 한국인 코호트에서는 다국어 성과 제외와 일본어 예약 전환 문제를 해결할 수 있을 때 검증하고, 일본인 코호트에서는 숙박의 1순위로 검증한다.
6. KKday는 한국인·일본인 액티비티 코호트 모두에서 클룩과 병렬 테스트하고 국적별 승인 EPC로 우선순위를 결정한다.

### 하지 않을 것

- 예약 발생 금액 전체를 로브 매출로 계산하지 않는다.
- 클릭 수만으로 파트너 가치를 주장하지 않는다.
- 공식 API·이미지·상표 정책 확인 없이 숙소 콘텐츠를 복제하지 않는다.
- 수수료가 높은 순서로 추천 결과를 왜곡하지 않는다.
