# 1. 문서 개요

## 1.1 목적

본 문서는 지역 여행 추천 서비스인 로브(Lovv)의 추천 품질과 신뢰성을 확보하기 위해 필요한 대한민국 및 일본의 데이터 수집 범위, 수집 계기 및 목적, 데이터 출처, 그리고 전처리 및 정합성 유의사항을 정리하는 데 목적이 있다.

## 1.2 수집 원칙

- **신뢰성**: 관광지 및 축제 정보의 경우 검증된 공식 출처(양국 공공데이터 및 OpenAPI)를 통해 데이터를 수집한다.
- **최신성**: 지역 관광 정보 및 링크의 유효성을 지속해서 검증하여 데이터 품질을 유지한다.
- **일관성**: 서로 다른 출처의 데이터를 통합할 때 명칭 및 행정구역 매핑 기준을 통일한다.

# 2. 데이터 수집 계획 명세

## 2.1 프로젝트 주제

로브(Lovv)는 대도시 편중 현상(오버투어리즘)을 완화하고 여행자에게 덜 알려진 대한민국(시/군/구 단위) 및 일본(시/정/촌/구 단위)의 매력적인 지역와 그에 최적화된 여행 일정 및 문화 콘텐츠를 추천하는 서비스이다. 본 계획서는 이에 필요한 양국의 지역 기반 데이터 구축 방안을 다룬다.

## 2.2 수집 데이터 정보 및 분량

로브 데이터베이스(DB) 구축을 위해 양국의 행정구역 기준 지역 레벨(한국: 시/군/구, 일본: 시/정/촌/구)까지 타깃을 좁혀 아래의 3가지 핵심 데이터 테이블을 구축한다.

### 2.2.1 지역 정보
- **행정구역 범위**: 대한민국 기준 기초지방자치단체(시/군/구 단위) 및 일본 기준 기초지방자치단체(시/정/촌/구 단위)
- **수집 대상 지역 수 (중간 발표 이전 단계 기준)**: 대한민국 내 40개 지역, 일본 내 316개 지역 지정 수집
- **수집 항목**:
  1. **지역명**: 해당 지역의 정식 명칭 및 한글/원어(일어) 표기 병기
  2. **지리 정보**: 위경도 좌표 정보 및 맞닿아 있는 인접 지역(이웃 시/군/구 및 시/정/촌/구) 리스트
  3. **지역 종합 설명**: 역사, 문화, 특징을 포괄하는 위키피디아 기반의 종합 텍스트
  4. **지역 관광 링크**: 해당 지자체가 운영하는 공식 문화관광 포털 웹사이트 링크
  5. **지역 기후**: 월별 평균 기온, 평균 강수량, 주요 계절적 기후 특징 데이터

### 2.2.2 관광지 정보
- **수집 범위**: 각 지역에 위치한 주요 관광 명소, 자연 경관, 미식/노포 및 문화 공간
- **수집 항목**: 관광지명, 상세 개요 등(수정 예정)

### 2.2.3 축제/행사 정보
- **수집 범위**: 해당 지역에서 개최되는 지역 전통 축제, 계절 행사 및 문화제
- **수집 항목**: 축제/행사명, 개최 시기(월 단위), 행사 성격 및 개요 설명, 개최 장소 등(수정 예정)

## 2.3 데이터 수집 계기

### 2.3.1 데이터 수집 배경
개인화 맟춤 여행 추천이라는 서비스의 목적을 달생하기 위해, 지역 여행지 및 축제 정보를 미리 수집하여 적재할 예정이다. 

### 2.3.2 지리 정보 수집 목적
각 지역의 지리적 위치 및 인접 지역 관계를 추출하기 위해 지리 정보를 수집한다.
- **여행 동선 최적화**: 한 지역을 방문한 여행자가 인접한 다른 지역나 현(県) 경계를 넘어 편리하게 이동할 수 있도록 인접성을 고려한 연계 여정을 설계하는 데 활용한다.

### 2.3.3 기후 정보 수집 목적
월별 기상 경향(장마 시기, 폭염 등)을 추출하기 위해 기후 정보를 수집한다.
- **최적 시기 추천**: 사용자가 입력한 여행 시기에 기상 조건이 가장 쾌적하고 여행하기 좋은 지역(예: 여름철 시원한 지역, 겨울철 적설량이 풍부한 홋카이도 등)를 과학적인 데이터 근거에 기반하여 추천한다.
- **기상 악화 대안 제안 (폴백 시스템)**: 특정 지역 추천 지역에 갑작스러운 비나 폭설 등 기상 이변 경향이 감지될 경우, 해당 기후적 특성에 맞추어 실내 활동 중심의 대체 일정으로 유연하게 전환하는 폴백 아키텍처의 의사결정 근거로 활용한다.

# 3. 데이터 전처리 방식 및 인프라 구성

본 문서의 범위는 수집 계획 단계로 제한되므로 구체적인 데이터 변환(Transform) 흐름 명세는 생략하며, 전처리 및 원본 저장을 위한 인프라 적재 방식만을 다음과 같이 규정한다.

- **S3 버킷 적재**: 웹 크롤링 및 오픈 API를 통해 수집된 초기 무정형/정형 로우 데이터(Raw Data)를 AWS S3 버킷에 저장한다.
- **Lambda 기반 전처리 수행**: S3 버킷에 데이터가 적재되면 AWS Lambda 함수가 트리거되어 텍스트 내 HTML 마크업 제거, 불필요한 메타데이터 필터링, 명칭 표준화 등 가벼운 정제 처리와 추가적인 데이터 가공 프로세스를 자동 수행하도록 인프라를 구성한다.

# 4. 데이터 품질 정합성 관리 방법

수집된 데이터의 품질 및 매핑 신뢰성을 보장하기 위해 아래 4가지 정합성 검증 방안을 적용한다.

- **본문 외 텍스트 혼선 방지 검증**: 위키피디아 텍스트 추출 시 본문 설명 텍스트와 표(Table), 인포박스, 각주 등이 단순하게 그대로 얽혀 들어가지 않고 올바르게 영역별로 분리 및 필터링되었는지 추출 샘플을 대조하여 검증한다.
- **행정구역 명칭 오매핑 검증**: 한국의 시/군/구 및 일본의 시/정/촌/구 명칭을 지리 정보 표준 코드와 대조하여, 행정구역 접미사 생략이나 오기(예: "시"와 "군"의 혼용)로 인해 엉뚱한 목적지 ID와 매핑되지 않도록 매핑 예외 목록을 구축하여 검증한다.
- **다국어 매핑 일관성 검증**: 일본 지역명 및 축제명의 번역 표기와 일본어 원어(한자/가나) 표기가 일대일로 어긋남 없이 매핑 구조를 유지하고 있는지 정합성을 철저히 대조한다.
- **관광 링크 유효성 검증**: 수집한 지자체 관광 웹사이트 URL에 대해 사전에 HTTP 요청 상태 코드 검사를 수행하여 깨진 링크(Broken Link, 404 에러 등)가 데이터베이스에 저장되는 것을 차단한다.

# 5. 법적 요소 검토

공공데이터 및 오픈 백과사전을 활용하는 만큼 기본 라이선스 기준에 근거하여 법적 의무 사항을 준수한다.

- **위키피디아 라이선스 준수**: 위키피디아 한국어/일본어 버전의 텍스트 콘텐츠는 크리에이티브 커먼즈 저작자표시-동일조건변경허락 3.0/4.0(CC BY-SA) 라이선스를 따르므로, 로브 서비스 화면 내에 사용된 설명의 원저작자 링크와 출처를 명확히 표기하여 라이선스 조건에 부합하도록 구성한다.
- **대한민국 공공데이터 활용 조건 준수**: 지자체 홈페이지 및 한국관광공사 Open API 데이터의 공공누리(KOGL) 유형별 조건에 맞추어 라이선스를 준수하며, 상업적 활용에 문제없는 데이터만을 DB에 적재한다.


# 6. 수집 정책 및 제외 범위

- **데이터 수집 주기**: 데이터의 변경 주기가 잦지 않고 초기 MVP 목적지 데이터베이스 구축에 집중하므로, 수집 주기는 **1년 단위**로 제한하여 데이터 갱신 및 재수집을 수행한다.
- **제외 범위**: 실시간 API 호출 데이터와 결합하기 위한 런타임 캐싱 정책은 데이터 수집 단계의 고려 범위를 초과하므로 **본 데이터 수집 계획 명세에서 제외**한다.
