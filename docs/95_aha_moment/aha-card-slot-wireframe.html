로브 · 메인 카드 슬롯 상태별 와이어프레임

6개 테마 × 한·일 대표 관광지 1쌍 슬롯. 메인 멘트는 "수아 님을 위한 추천(FOR YOU)"으로 고정. 평소엔 「기본」 상태로 띄우다가, 사용자 행동이 쌓이면 「개인화」 상태(맨 앞에 보라 카드 추가, 총 7장)로 전환.

🧩
카드 모델. 카드 짝(예: 경주·교토)은 그 테마의 한·일 대표 관광지(붐비는 유명지)로, 분위기를 인식시키는 기준점이다. 이 둘은 서로 대안이 아니다. 카드 하단의 payoff = 같은 분위기의 덜 붐비는 소도시 추천(로브의 실제 출력값). 소도시명은 예시이며 실제 추천 데이터로 교체.
기본 국가. 온보딩에서 테마와 함께 고른 기본 국가(한/일)로 payoff 소도시를 채우고, 대표 관광지 쌍도 기본 국가를 먼저 표기(아래는 한국 선호 예시). 여행 시기 입력 시 변경 가능한 soft default.
선언 테마 온보딩에서 고른 테마(우선 노출) 발견 카드 안 고른 테마(explore·학습용) 소도시 payoff 유명지 분위기 → 덜 붐비는 곳 타이밍/희소성 행동 기반 개인화

설계 포인트. 1~3은 선호 테마(관련성 베이스)를 타이밍(노란 태그) 좋은 순으로 정렬. 선호 테마는 3개뿐이라 4번부터는 모두 비선택 테마=초록 발견 카드. 그중 가장 눈에 띄는 4번 자리에 순수 타이밍 "이번 달 절정" 카드(테마 깸)를 둬 희소성 aha를 가장 세게 + 발견 동시. 순수 타이밍 우선으로만 가면 관심 없는 테마가 떠 사용자를 식게 만들므로 1장만 허용. 메인 멘트는 "수아 님을 위한 추천(FOR YOU)"으로 고정, 서브 멘트만 타이밍 시의성 강조.

개인화 카드 = 2가지 변형 + X 토글. 기본은 갔던 곳 기반(exploit) — "지난번 ♥ 곡성과 비슷한 〇〇". 우상단 ×를 누르면 안 가본 곳(explore) 추천으로 전환 — "아직 안 가보신 분위기, 새로운 △△"(취향과는 맞되 가본 적·저장한 적 없는 테마·지역). 갔던 곳만 반복하면 탐색이 막히므로 사용자가 비슷한 곳 ↔ 안 가본 곳을 직접 고르게 한다. X 선택도 신호로 기록(비슷한 곳 추천 비중 ↓).
📌
설계 포인트. 「기본」의 6장(순서·내용 동일)을 유지하고 맨 앞에 보라색 개인화 카드 1장만 추가 = 총 7장(개인화 카드는 비슷한 곳/안 가본 곳 토글). 신뢰 임계값 미달이면 이 카드를 빼고 「기본」(6장)으로 강등. 메인 멘트는 고정, 서브 멘트만 교체.
🧭
공통 원칙 — ① 메인 멘트 "수아 님을 위한 추천(FOR YOU)" 고정, 서브 멘트만 상태별 교체 ② 카드 짝은 대안이 아니라 테마 대표 관광지, 소도시는 별도 payoff ③ 「기본」(6장), 행동 쌓이면 「개인화」(맨 앞 보라 추가 7장) ④ 개인화는 신뢰 임계값 통과 시에만(미달→「기본」) ⑤ 측정: 카드 클릭=첫 aha 이벤트, "안 고른 테마 클릭률"로 발견 작동 검증.
