## 2. 문제

일본 데이터는 최초 검토 시 CC BY 4.0 또는 CC BY 4.0과 호환되는 공개 조건으로 처리 가능한 것으로 보고 취득 후보에 포함되었다.

그러나 본격적인 데이터 취득을 진행하기 전에 제한 사항을 확인하는 과정에서 별도 데이터 규약을 검토했고, CC BY 4.0 기준으로만 보면 가능해 보이던 이용 범위가 실제 규약에서는 제한될 수 있음을 확인했다.

이 상태에서 데이터를 취득하면 CC BY 4.0 라이센스 기준만 보고 사용 가능하다고 판단한 내용과, 제공처의 실제 데이터 규약 사이에 충돌이 발생할 수 있었다.

특히 JATO(Japan Adventure Tourism Organization, 일반사단법인 일본 어드벤처 투어리즘 협의회) 자료는 관광 관련 정보로 활용 가능해 보일 수 있으나, 공식 페이지 확인 결과 오픈데이터 제공처라기보다 회원 기반 협의회와 활동 안내 성격이 강했다. 따라서 공개 웹 페이지에 접근 가능하다는 이유만으로 수집, 저장, 벡터화, 서비스 제공에 활용할 수 있다고 판단하기 어렵다.

추가로 일본 정부가 직접 운영하거나 정부 산하·공공기관이 운영하는 관광 관련 출처도 함께 확인했다. 정부 운영 출처는 PDL 1.0 또는 정부 표준 이용규약을 통해 CC BY 4.0 호환 조건으로 볼 수 있는 경우가 있었지만, 모든 정부·공공 사이트가 동일하게 자유 취득 가능한 것은 아니었다. 예를 들어 일본 관광청(JTA) 사이트는 별도 명시가 없는 콘텐츠에 Public Data License 1.0을 적용하지만, JNTO의 Japan Tourism Statistics는 다운로드 과정에서 사용 신청 또는 이용 허가 확인 절차를 요구한다. 따라서 CC BY 4.0 기준으로 내부 처리했더라도, 최종 취득 가능 여부는 출처별 세부 규약을 기준으로 판단해야 한다.

## 3. 원인

| 원인 | 내용 |
| --- | --- |
| CC BY 4.0 기준과 규약의 충돌 | 초기에는 CC BY 4.0 기준으로 처리 가능하다고 보았지만, 별도 데이터 규약은 저장·가공·재배포·API/API 외부 DB 사용에 추가 제한을 포함할 수 있었다. |
| 검토 기준 부족 가능성 | CC BY 4.0 문구만 보면 취득 가능해 보이나, 데이터셋 제공처의 세부 규약과 신청 절차까지 확인해야 실제 사용 가능 범위를 판단할 수 있었다. |
| 서비스 활용 리스크 | 데이터 규약이 CC BY 4.0 기준보다 보수적으로 해석될 경우, 서비스 데이터로 저장·가공·배포하는 과정에서 문제가 발생할 수 있었다. |
| 사후 정정 비용 | 취득 후 규약 문제가 확인되면 이미 수집한 Raw, 전처리 산출물, DynamoDB 적재 데이터, S3 vector index를 모두 제거하거나 재구성해야 한다. |
| JATO 자료 재사용 제한 | JATO 회원규약은 협의회가 발행하는 자료와 데이터의 무단 외부 게재, 제3자 양도·판매, 공표를 제한하고 있어 서비스 데이터셋으로 직접 취득하기 어렵다. |
| 정부·공공 출처별 조건 차이 | 일본 정부·공공 출처는 PDL 1.0, 정부 표준 이용규약, 개별 사이트 정책, API 조건, 신청·허가 절차가 혼재되어 있어 정부 운영 여부만으로 일괄 취득 판단을 할 수 없다. |

## 6. 해결 과정

1. 일본 데이터를 취득 후보로 두고 CC BY 4.0 또는 CC BY 4.0 호환 조건으로 처리 가능한지 1차로 확인했다.
2. JATO 공식 사이트에서 JATO가 Japan Adventure Tourism Organization이며, 일본 어드벤처 투어리즘 협의회라는 점을 확인했다.
3. JATO 입회 안내와 회원규약을 확인해 회원 혜택, 상호·상표 이용 조건, 자료·데이터 재사용 제한을 검토했다.
4. 일본 정부·공공 운영 출처를 추가로 확인해 디지털청 PDL 1.0, 일본 관광청(JTA), JNTO Japan Tourism Statistics, e-Stat의 이용조건 차이를 검토했다.
5. 데이터 규약이 라이센스 문구와 다르게 해석될 수 있고, JATO 자료는 오픈 라이센스 대상이 아니라 무단 재게시·양도·판매·공표 제한에 걸릴 수 있음을 확인했다.
6. 정부·공공 출처도 CC BY 4.0 호환 여부와 별도로 출처 표기, 편집 표시, 제3자 권리, API/외부 DB 조건, 신청·허가 절차를 확인해야 한다고 판단했다.
7. CC BY 4.0 기준과 세부 규약이 충돌할 수 있는 상태에서 데이터를 취득·가공·적재하지 않기로 결정했다.
8. 일본 데이터는 취득 대상에서 제외하고, KR 데이터 중심의 전처리·검색·추천 범위를 우선 유지했다.

## 7. 결과

일본 데이터는 본격적인 취득을 진행하지 않는다.

CC BY 4.0 또는 CC BY 4.0 호환 조건만으로 취득 여부를 확정하지 않고, 데이터 규약까지 확인한 뒤 취득 포기를 결정했다. 이 판단을 통해 규약 위반 가능성, 사후 데이터 제거 비용, 운영 문서와 실제 데이터 사용 조건의 불일치 위험을 줄였다.

JATO 관련 자료는 명시적 허가 전까지 수집 대상, Raw 적재 대상, DynamoDB 적재 대상, S3 vector 생성 대상에서 제외한다. 향후 일본 관광 데이터를 다시 검토할 경우 JATO 자료를 직접 취득 대상으로 두지 않고, 공공 오픈데이터 포털이나 API 제공 조건이 명확한 출처를 우선 검토한다.

일본 정부·공공 운영 출처는 취득 포기 대상과 재검토 가능 대상을 분리한다. PDL 1.0 또는 CC BY 4.0 호환 조건이 명시되고, 출처 표기·편집 표시·제3자 권리·API 조건을 충족할 수 있는 데이터는 후보로 남긴다. 반대로 신청·허가 절차가 있거나, API/외부 DB의 별도 약관이 있거나, 사진·로고·지도처럼 제3자 권리 확인이 필요한 자료는 CC BY 4.0으로 내부 처리했더라도 자동 수집·벡터화 대상에서 제외한다.

## 8. 재발 방지

- 외부 데이터를 취득하기 전 라이센스와 데이터 규약을 반드시 함께 확인한다.
- CC BY 4.0 또는 CC BY 4.0 호환 조건으로 보이더라도 별도 규약, 신청 절차, API 약관이 더 엄격하면 취득을 보류한다.
- 취득 전 확인 항목에 저장, 가공, 재배포, API 제공, 벡터화, 상용 서비스 활용 가능 여부를 포함한다.
- 규약 해석이 애매한 경우 Raw 적재, DynamoDB 적재, S3 vector 생성 전 단계에서 중단한다.
- 협회, 단체, 회원 기반 사이트의 자료는 공공 오픈데이터 포털과 구분해서 검토한다.
- 회원규약, 상호·상표 사용 조건, 자료·데이터 재사용 조항을 별도 확인한다.
- 정부·공공 운영 출처라도 CC BY 4.0 호환 여부, API 약관, 신청·허가 절차를 출처별로 확인한다.
- 정부 자료를 편집·가공해 사용할 경우 출처 표기와 편집 사실 표시를 함께 남긴다.

## 9. 조사 결론

TS-06의 핵심은 일본 데이터 후보를 CC BY 4.0 기준으로 처리하려 했지만, 실제 데이터 규약을 함께 확인했을 때 동일한 범위의 사용 허용으로 보기 어려웠다는 점이다.

JATO 조사 결과, JATO는 Japan Adventure Tourism Organization으로 확인되며 공식 사이트와 입회 안내는 협의회 활동 및 회원 혜택 중심으로 구성되어 있었다. 또한 회원규약상 협의회가 발행하는 자료·데이터는 무단으로 다른 매체에 게재하거나 제3자에게 양도·판매·공표할 수 없으므로, 명시적 허가 없이 프로젝트 데이터셋으로 취득하는 것은 적절하지 않다.

일본 정부·공공 운영 출처까지 포함해 확인한 결과, CC BY 4.0 또는 CC BY 4.0 호환 조건 자체가 취득 가능성을 보장하지는 않는다. 일본 관광청(JTA)처럼 PDL 1.0을 적용하는 출처는 CC BY 4.0 호환 기준으로 재검토 가능성이 있지만, JNTO Japan Tourism Statistics처럼 사용 신청 또는 허가 절차가 붙는 출처도 있다. 또한 e-Stat처럼 정부 표준 이용규약과 CC BY 4.0 호환을 안내하는 경우에도 API·GIS 등 별도 조건이 존재한다.

따라서 일본 데이터는 수집을 강행하지 않고 취득 대상에서 제외하는 것이 맞다. 이후 재검토 시에는 JATO와 같은 협의회 자료는 제외하고, 일본 정부·공공 출처 중 CC BY 4.0 호환 조건이 명확하며 프로젝트의 Raw 저장, DynamoDB 적재, S3 vector 생성, 서비스 제공 범위를 세부 규약상 충족하는 데이터만 별도 후보로 분리한다.
