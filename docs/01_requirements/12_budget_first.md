# 로컬루트 신규 기능 제안 — 예산 역산 추천 (Budget-First)

> 이 제안은 **테마가 아니라 챗봇 기능(진입점)** 입니다.
> 사용자는 테마를 고르는 단계를 거치지 않고, 챗봇에 예산 조건을 바로 질문해 추천을 받습니다.

---

## 개요

여행 결정의 핵심 변수인 **예산을 출발점**으로 삼는 역방향 추천 기능입니다. "30만 원으로 1박 2일 갈 만한 소도시" 같은 질문을 챗봇에 바로 입력하면, 예산 안에서 실현 가능한 소도시와 여정을 거꾸로 제안합니다. 예산은 의외로 기존 추천앱이 약한 영역이며, 참고 문서에도 예산 입력 항목이 이미 있어 자연스럽게 확장됩니다.

| 항목 | 내용 |
|---|---|
| 구분 | **기능(진입점)** — 테마 선택 아님 |
| 기능 ID | `budget-first` |
| 한 줄 정의 | 예산 금액·기간을 입력하면 그 안에서 가능한 소도시·여정을 역으로 제안 |
| 진입 방식 | **테마 선택 단계 없이 챗봇에 바로 질문** (예: "30만 원 1박 2일") |
| 핵심 계산 | 교통비 + 숙박 + 식비 + 액티비티 추정치를 합산해 예산 내 여정 구성 |
| 기존 구조 활용 | 참고 문서의 예산 입력 항목 + 소도시 DB의 이동·여행 팁 데이터 |

---

## 진입 방식 (중요)

이 기능은 **테마(온천·바다 등)를 고르는 흐름을 건너뛴다.** 사용자는 첫 화면이나 대화 입력창에 예산·기간을 직접 말하는 것으로 추천을 시작한다.

```
# 챗봇 직접 질의 예시

"30만 원으로 1박 2일 갈 만한 곳 추천해줘"
"예산 20만 원 안에서 당일치기 소도시"
"1인 50만 원으로 2박 3일, 어디까지 가능해?"
```

시스템은 예산·기간·인원을 파싱한 뒤, 예산 내에서 가능한 소도시를 거꾸로 좁혀 추천한다. (선택적으로 테마·취향을 추가 질문할 수 있으나 필수 아님)

---

## 사용자 시나리오

### 시나리오 B-01 · 예산이 빠듯한 학생의 당일치기

> 대학생 사용자가 챗봇에 "20만 원으로 당일치기 갈 만한 소도시"를 바로 입력한다. 시스템은 테마를 묻지 않고, 출발지 기준 교통비와 현지 예상 지출을 합산해 예산 안에 들어오는 소도시를 추천한다. 각 추천에는 "교통 약 OO원 + 식비·입장료 약 OO원" 식의 대략적 비용 내역을 함께 보여준다.

### 시나리오 B-02 · 정해진 예산으로 1박 2일 계획

> 사용자가 "1인 30만 원으로 1박 2일 다녀올 수 있는 곳"을 요청한다. 시스템은 예산 내에서 숙박 수준(게스트하우스 / 모텔 / 료칸)을 조정하며 가능한 여정을 제안한다. 예산을 초과하는 소도시는 "예산보다 약 OO원 초과"로 명확히 구분 표시하고, 예산을 맞추는 대안(숙박 등급 낮추기 등)을 함께 안내한다.

---

## 유저 스토리

- **US-B1** — 예산이 정해진 사용자로서, 나는 금액과 기간만 입력하면 그 안에서 갈 수 있는 소도시를 추천받고 싶다. 그래야 가격을 일일이 계산하지 않아도 된다.
- **US-B2** — 사용자로서, 나는 테마를 고르지 않고 챗봇에 예산을 바로 질문해 추천을 시작하고 싶다. 그래야 빠르게 결과를 볼 수 있다.
- **US-B3** — 사용자로서, 나는 추천마다 대략적인 비용 내역(교통·숙박·식비)을 보고 싶다. 그래야 예산이 어디에 쓰이는지 판단할 수 있다.
- **US-B4** — 사용자로서, 나는 예산을 초과하는 곳은 얼마나 넘는지, 어떻게 맞출 수 있는지 안내받고 싶다. 그래야 조정 여부를 결정할 수 있다.
- **US-B5** — 사용자로서, 나는 같은 예산이라도 숙박 수준을 조정한 여러 안을 받고 싶다. 그래야 내 우선순위(아끼기 vs 편안함)에 맞게 고를 수 있다.

---

## 추천 시 포함 항목

- 예산 내 가능 여부 (가능 / 약 OO원 초과)
- 대략적 비용 내역(교통 + 숙박 + 식비 + 액티비티)
- 숙박 수준별 대안(게스트하우스 / 모텔 / 료칸 등)
- 추천 이유 + 예산을 맞추는 조정 팁
- 기간·인원 반영 (당일 / 1박 2일 / 2박 3일)

---

## 기존 구조 반영 방식

| 반영 위치 | 내용 |
|---|---|
| 진입점 | 테마 선택 흐름과 **별도**로, 챗봇 직접 질의로 진입 |
| 입력 파싱 | 멀티스텝 에이전트가 발화에서 예산·기간·인원 추출 |
| 비용 추정 | 소도시 DB의 이동(`travel`)·여행 팁(`tip`) + 숙박 등급별 추정치 |
| RAG 연계 | 비용 근거(교통수단·평균 숙박가)를 데이터에 grounding |
| 제약사항 | 실시간 항공·숙박 요금 연동은 제외 — 추정치 기반(참고용 명시) |
