# 로컬루트 신규 기능 제안 — 날씨 기반 추천 (Weather-First)

> 문서 성격: 보조 Markdown
> 대표 문서: `../01_requirements.md`

> 이 제안은 **테마가 아니라 챗봇 기능(진입점)** 입니다.
> 사용자는 테마를 고르는 단계를 거치지 않고, 챗봇에 날씨·시기 조건을 바로 질문해 추천을 받습니다.

---

## 개요

**Wikipedia 월별 날씨 경향 데이터 월별 날씨 경향 데이터를 추천에 능동적으로 활용**하는 기능입니다. 지금은 소도시 상세 정보에서 날씨를 "표시"하는 데만 쓰이는데, 이를 추천의 **입력 변수로 승격**시킵니다. "이번 주말 날씨 좋은 소도시" 같은 질문을 챗봇에 바로 입력하면, 후보 소도시들의 월별 경향를 비교해 날씨가 가장 좋은 곳을 역으로 추천합니다. 이미 가진 자산을 한 번 더 쓰는 것이라 구현 비용이 낮습니다.

| 항목 | 내용 |
|---|---|
| 구분 | **기능(진입점)** — 테마 선택 아님 |
| 기능 ID | `weather-first` |
| 한 줄 정의 | 날씨·시기를 입력하면 월별 경향가 좋은 소도시를 역으로 추천 |
| 진입 방식 | **테마 선택 단계 없이 챗봇에 바로 질문** (예: "이번 주말 맑은 소도시") |
| 핵심 자산 | Wikipedia 월별 날씨 경향 데이터 |
| 기존 구조 활용 | 소도시 DB 좌표 + Open-Meteo 월별 경향를 비교·정렬 |

---

## 진입 방식 (중요)

이 기능은 **테마(온천·바다 등)를 고르는 흐름을 건너뛴다.** 사용자는 첫 화면이나 대화 입력창에 날씨·시기 조건을 직접 말하는 것으로 추천을 시작한다.

```
# 챗봇 직접 질의 예시

"이번 주말 날씨 좋은 소도시 어디야?"
"다음 주에 비 안 오는 곳으로 추천해줘"
"맑고 따뜻한 데로 당일치기 가고 싶어"
```

시스템은 후보 소도시들의 월별 경향(맑음·강수·기온)를 조회·비교한 뒤, 조건에 가장 부합하는 곳을 역으로 추천한다. (선택적으로 테마·취향을 추가 질문할 수 있으나 필수 아님)

---

## 사용자 시나리오

### 시나리오 E-01 · 주말 나들이, 날씨가 결정 기준

> 사용자가 금요일에 챗봇으로 "이번 주말 전국에서 가장 맑은 소도시"를 바로 묻는다. 시스템은 테마를 묻지 않고, 후보 소도시들의 주말 월별 경향를 비교해 강수 확률이 낮고 맑은 곳을 우선 추천한다. 각 추천에는 해당 소도시의 주말 월별 경향(맑음·기온·강수 확률)를 함께 표시한다.

### 시나리오 E-02 · 비를 피하고 싶은 여행자

> 사용자가 "다음 주에 비 안 오는 곳으로 1박 2일"을 요청한다. 시스템은 후보지 월별 경향에서 강수 가능성이 낮은 소도시를 추려 추천하고, 만약 야외 위주 소도시가 우천 월별 경향라면 "이 시기엔 실내 체험형(공예 등)도 고려"하라는 대안을 함께 제시한다. 날씨 데이터의 기준 시각을 함께 노출한다.

---

## 유저 스토리

- **US-E1** — 사용자로서, 나는 "날씨 좋은 곳"이라고만 말해도 월별 경향가 좋은 소도시를 추천받고 싶다. 그래야 날씨를 일일이 확인하지 않아도 된다.
- **US-E2** — 사용자로서, 나는 테마를 고르지 않고 챗봇에 날씨 조건을 바로 질문해 추천을 시작하고 싶다. 그래야 빠르게 결과를 볼 수 있다.
- **US-E3** — 사용자로서, 나는 추천마다 해당 소도시의 월별 경향(맑음·기온·강수)를 함께 보고 싶다. 그래야 추천을 신뢰하고 결정할 수 있다.
- **US-E4** — 비를 피하고 싶은 사용자로서, 나는 강수 월별 경향가 낮은 곳을 우선 추천받고 싶다. 그래야 우천으로 망치는 여행을 피할 수 있다.
- **US-E5** — 사용자로서, 나는 야외지가 우천 월별 경향일 때 실내 체험형 대안을 함께 안내받고 싶다. 그래야 날씨와 무관하게 여행을 확정할 수 있다.

---

## 추천 시 포함 항목

- 해당 소도시 월별 경향(맑음 상태 · 기온 · 강수 확률 · 풍속)
- 날씨 데이터 기준 시각(최신성 표시)
- 조건 부합도(맑음 우선 / 강수 회피 등)
- 우천 월별 경향 시 실내 체험형 대안 제시
- 추천 이유 + 주변 코스

---

## 기존 구조 반영 방식

| 반영 위치 | 내용 |
|---|---|
| 진입점 | 테마 선택 흐름과 **별도**로, 챗봇 직접 질의로 진입 |
| 핵심 자산 | 이미 연동된 Wikipedia 월별 날씨 경향 데이터를 추천 변수로 승격 |
| 입력 파싱 | 멀티스텝 에이전트가 발화에서 날씨 조건·시기 추출 |
| 비교·정렬 | 후보 소도시 좌표로 월별 경향를 조회해 조건순 정렬 |
| 테마 연계 | 우천 시 실내 체험형(공예 등) 테마와 자동 연계 |
| 제약사항 | 월별 경향 정확도는 외부 API 범위에 의존 — 기준 시각 노출로 보완 |
