# 7. 데이터 요구사항

## 7.1 필요 데이터 항목

본 절은 저장 구조 설계가 아니라, 로브 서비스가 추천·설명·검증을 수행하기 위해 확보해야 하는 데이터의 종류를 정의한다.

| 데이터 구분 | 필요 내용 |
| --- | --- |
| 목적지 기본 정보 | 소도시명, 국가, 지역, 대표 설명, 주요 이미지, 위치 좌표, 접근 방법 |
| 테마 분류 정보 | 초기 6개 테마와 공예·웰니스·액티비티 등 확장 테마 관련 태그 |
| 계절·시기 정보 | 월별 또는 계절별 추천 적합도, 방문하기 좋은 시기, 비추천 시기와 그 사유 |
| 축제·행사 정보 | 행사명, 개최 지역, 개최 시기, 간단 설명, 공식 또는 참고 링크 |
| 혼잡도·대체지 정보 | 혼잡 가능성이 높은 관광지, 대체 방문지, 대체 시간대, 분산 추천 사유 |
| 지도·장소 정보 | Google Maps, Kakao Maps, Yahoo Japan 등 외부 플랫폼에서 확인 가능한 지도, 장소 상세, 후속 탐색 링크 |
| 날씨·환경 정보 (표시용) | WeatherAPI 기반 현재 기온, 날씨 상태, 강수 확률, 풍속 등 목적지 상세 화면 표시용 정보 |
| 월별 기상 경향 정보 (추천용) | 소도시별 1~12월 기상 경향(맑음 적합도, 우천·폭설·장마·태풍 경향) |
| 추천 근거 정보 | 추천 이유, 사용자 조건과의 매칭 근거, 참고 출처, 검증 상태, 신뢰도 표시 정보 |
| 온보딩 선호 정보 | 사용자가 선택한 대도시 스타일, 매핑된 기본 테마, 선호 테마 프로필, 재응답 이력 |
| 저장 일정·피드백 정보 | 저장된 추천 일정, 일정 유형, 좋아요/싫어요 피드백, 피드백 대상 소도시와 테마 태그 |
| 추천 실행 이벤트 | 추천 요청 시각, 입력 조건, 진입 방식, 필수 조건 충족 여부 |
| 재추천 이벤트 | 직전 추천 결과, 재추천 요청 시각, 유지 조건, 변경 조건, 제외 또는 감점 후보 |
| 북마크 데이터 | 소도시 북마크, 대도시 북마크, 저장 시각, 국가, 지역, 목적지 ID |
| 실제 방문 기록 | 방문한 소도시, 방문 일자, 여행 루트, 동행 유형, 사용자가 직접 입력한 메모 |
| 방문 소도시 별점 데이터 | 실제 방문한 소도시에 대한 사용자 별점, 평가 시각, 평가 대상 ID, 방문 기록 ID |
| Cold Start 기준 데이터 | 관광공사·지자체 추천 루트, 여행 유튜버 루트, 내부 검증 루트, 출처 URL, 신뢰도 |
| 랭킹 학습·실험 데이터 | 북마크, 방문 기록, 방문 소도시 별점, 좋아요/싫어요, 목적지 태그, 사용자별 소도시 적합도 점수 |
| 위치 반경 추천 데이터 | 현재 위치 또는 사용자가 직접 입력한 기준 위치, 추천 반경, 이동 가능 시간, 위치 권한 동의 여부 |
| 여행 중 조정 데이터 | 남은 예산, 남은 일정, 현재 위치, 변경 사유, 재추천 결과 |
| 애니메이션 성지 정보 | 작품명, 장소명, 도시/지역, 국가, 작품과의 관계, 공식 배경지 여부, 작가 고향, 공식 콜라보, 팬 성지 여부, 분위기 유사 태그, 근거 URL, 신뢰도, 방문 가능 여부, 장소 유형 |
| 일본 목적지 특화 정보 | 일본어 명칭, 도도부현, 지역권, 여행 스타일 태그, 추천·비추천 계절, 주요 도시 기준 이동시간, 환승 횟수, 가까운 공항, 뚜벅이 가능 여부, JR/지역 교통패스, 일본어 검색 키워드 |
| 일본 현지 탐색 링크 | Yahoo Travel, じゃらん, 楽天トラベル, 一休, Booking.com, Agoda, Google Hotels, 食べログ, Retty, Hot Pepper, Google Maps, Instagram 해시태그 검색 링크 |
| 지역 문화 체험 정보 | 체험 유형, 체험 시간, 난이도, 예약 필요 여부, 참여 방식, 우천 적합성, 주변 마을·축제 연계 정보 |
| 치유·웰니스 정보 | 회복 활동 유형, 한적함·저자극 정도, 온천·숲 치유·템플스테이·명상 운영 정보, 참여 방법, 적합 동행 유형 |
| 사계절 액티비티 정보 | 액티비티 종류, 적합 계절·시기, 눈·파도·수온·날씨 조건, 난이도, 강습 가능 여부, 장비 대여 정보, 주변 휴식 코스 |
| 사용자 여행 예산 정보 | 교통비, 숙박비, 식비, 액티비티 비용 추정치, 기간·인원 기준, 예산 초과 금액, 숙박 수준별 조정 대안 |
| 대체 일정 정보 | 월별 기상 경향에 따른 실내 중심 대체 일정, 대체 일정 제공 조건, 토글 표시 여부 |
| 외부 행동 링크 | 지도 열기, 공식 관광 정보, 예약, 교통, 숙박, 맛집, 현지 플랫폼으로 이어지는 링크 |
| 별 관측 후보 데이터 | 광해 등급, 월령, 구름량, 대기 안정도, 투명도, 관측 시설·예약 링크 등 향후 검토 후보 데이터 |

## 7.2 데이터 품질 기준

| 품질 기준 | 요구사항 |
| --- | --- |
| 지역 커버리지 | 국내와 일본 소도시를 모두 포함하되, 국가별 추천 흐름은 독립적으로 운영할 수 있어야 한다. |
| 테마 커버리지 | 각 테마별로 추천 후보를 구성할 수 있을 만큼 충분한 목적지와 설명 데이터를 확보해야 한다. |
| 최신성 | 축제, 영업 정보, 날씨, 혼잡도처럼 변동성이 높은 데이터는 갱신 시점 또는 유효 기간을 함께 관리해야 한다. |
| 출처 추적성 | 추천 결과에 사용된 주요 데이터는 사용자가 신뢰할 수 있도록 출처 또는 확인 가능한 링크를 제공해야 한다. |
| 설명 가능성 | 추천 결과는 단순 목록이 아니라 사용자 조건과 어떤 데이터가 매칭되었는지 설명할 수 있어야 한다. |
| 결측 대응 | 필수 데이터가 부족한 목적지는 추천에서 제외하거나, 부족한 정보와 대체 안내를 명확히 표시해야 한다. |
| 국가별 분리 | 한국 여행 요청에는 한국 소도시 데이터만, 일본 여행 요청에는 일본 소도시 데이터만 사용해야 한다. |

**정의 범위:**

본 문서는 어떤 데이터가 필요한지를 정의한다.

테이블 구조, 컬렉션 구조, 인덱스, 저장소 선택 등 저장 구조 설계는 별도 설계 문서에서 다룬다.
