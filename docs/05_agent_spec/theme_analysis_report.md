## 4. 추천 알고리즘 임계치 제안

### 4-1. 필수 테마 충족 여부 (Required Filter Threshold)
도시가 후보로 선정되기 위한 최소 장소 개수 기준입니다.
> **제안 규칙**
> *   **일반 테마 (`미식`, `역사`, `자연`)**: 도시 내 해당 테마 장소 수 **3개 이상**을 필수로 요구 (일정 구성 가능 최소 개수)
> *   **희소/한정 테마 (`바다`, `예술`, `온천`)**: 사용자가 3개 테마를 복수 선택했을 경우에 한해, 해당 테마는 **1개 이상**만 존재해도 만족으로 완화.
> *   *이유*: `온천·휴양`은 3개 이상 보유한 도시가 11개뿐이고, `예술·감성`은 19개뿐이어서 기준을 3개 이상으로 높일 경우 후보 도시가 소멸합니다.

### 4-2. 테마 특화도 계산 임계치 (Specialization Check Threshold)
어떤 도시가 특정 테마에 '특화'되어 있다고 판정하기 위한 비율 기준입니다. 절대값 20%의 한계를 보완하기 위해 **전체 평균 대비 배율**을 사용합니다.

| 테마 | 전체 평균 비중 | 제안 특화도 임계 비율 | 특화 판정 기준 (비교 수식) | 설명 |
| :--- | :---: | :---: | :--- | :--- |
| **미식·노포** | 33.89% | **42.00%** | 도시 내 비중 > 전체 평균의 1.25배 | 워낙 대중적이므로 평균보다 확연히 높아야 특화 판정 |
| **역사·전통** | 30.06% | **38.00%** | 도시 내 비중 > 전체 평균의 1.25배 | 전통 가옥, 문화재가 평균(30%) 이상으로 뚜렷해야 함 |
| **자연·트레킹** | 23.03% | **28.00%** | 도시 내 비중 > 전체 평균의 1.2배 | 자연 녹지, 계곡, 휴양지가 풍부한 도시 판정 |
| **바다·해안** | 6.08% | **12.00%** | 도시 내 비중 > 전체 평균의 2.0배 | 해안선과 해수욕장 비중이 평균의 2배 이상인 항구/해안 도시 |
| **예술·감성** | 4.15% | **6.00%** | 도시 내 비중 > 전체 평균의 1.5배 | 갤러리, 체험공방, 전시관 비중이 상대적으로 돋보이는 도시 |
| **온천·휴양** | 2.80% | **4.00%** | 도시 내 비중 > 전체 평균의 1.5배 | 온천지구, 웰니스 휴양 거점이 명확히 형성된 도시 |

### 4-3. 희소 테마 가중치 보정 (Sparsity Adjustment Weight)
전체 데이터 양이 적은 테마가 미식이나 역사에 묻히지 않도록 점수를 보정합니다.
> **제안 규칙**
> *   **미식·노포, 역사·전통**: 기본 가중치 **1.0** 적용
> *   **자연·트레킹**: 기본 가중치 **1.1** 적용
> *   **바다·해안, 예술·감성**: 희소도 가중치 **1.3** 적용
> *   **온천·휴양**: 극희소 가중치 **1.5** 적용 (단 1개의 온천이 있어도 강력한 추천 동인으로 작동)

### 4-4. 도시 규모 편향 보정 상한선 (Cap Limit for Place Counts)
특정 대도시가 장소 수의 절대적 우위(최대 440개)로 랭킹을 독점하지 않도록 테마별 점수 계산 시 장소 수에 상한을 둡니다.
> **제안 규칙**
> *   특정 테마 장소 수에 대한 랭킹 점수 가중 시 **최대 10개**를 상한선(Cap)으로 지정합니다.
> *   *이유*: 도시별 테마 개수 통계에서 **75% 분위수(P75)가 대략 10~30개** 사이이며, 10개 장소면 1박 2일 혹은 2박 3일 일정을 겹치지 않고 다양하게 구성하기에 이미 차고 넘치는 숫자입니다. 10개 이상부터는 개수의 절대 우위보다 유사도 및 특화도에 의해 랭킹이 결정되도록 보호막을 씌웁니다.
