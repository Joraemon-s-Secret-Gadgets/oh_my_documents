# 중간발표 서론(도입부) 개선 제안

> 목적: `midterm_vs_final_presentation_guide.md` §7.1(서론 4단계)과 `01_midterm_presentation.md`(현재 6장 구성)을 비교해, **서론 부분을 어떻게 보완할지** 정리한 제안 문서.
> 범위: 서론(도입)에 한정. 본론(시스템 흐름 이후)은 다루지 않는다.
> 단계: 중간발표 (PoC)

## 1. 비교 한눈에

가이드 §7.1이 권장하는 서론 흐름과 현재 발표안의 매핑이다.

| 가이드 §7.1 서론 단계 | 현재 발표안 | 충족도 |
| --- | --- | --- |
| ① 프로젝트 소개 (한 장, 인상만) | S1 프로젝트 소개 (Lovv) | ✅ 충족 |
| ② 오버투어리즘이란? (흥미롭게 소개) | S2 문제: 오버투어리즘 (워드클라우드) | △ 부분 — "현상 나열"에 가깝고 흥미 유발 장치가 약함 |
| ③ 왜 문제인가 (4개 축 + 비즈니스 포인트) | S3 문제의 영향 + S4·S5 시장 근거 | △ 부분 — 단일 축이라 4축 보강 필요. **`s3_problem_axes_revision.md`에서 4축으로 확정** |
| ④ 서론 마무리 — 의미 + 해결방식 + **핵심 컨셉 3가지**(문제↔컨셉 연결) | **없음** (S5 → 바로 S6 시스템 흐름으로 점프) | ❌ 누락 |

**결론 한 줄:** 가장 큰 공백은 ④다. 시장 근거(S5) 다음에 곧바로 시스템 아키텍처(S6)로 넘어가면, 청중이 "그래서 이걸 어떤 컨셉으로 푼다는 거지?"를 정리하지 못한 채 기술 설명으로 진입한다. 문제(서론)와 해결(본론)을 잇는 다리 슬라이드가 필요하다.

## 2. 슬라이드별 개선 제안

### S1 — 프로젝트 소개 (거의 유지 + 한 줄 정의 보강) ★보강★

현재 구성("서울·오사카 말고, 지금은 이곳" + Local·Vibe·Voyage 도식)은 가이드 ①의 "한 장으로 인상만" 원칙에 잘 맞아 **기본은 유지**한다. "서울·오사카"가 한국·일본을 동시에 가리켜 **양방향**을 은근히 깔아주는 장치도 살린다.

**보강 — 화면에 "무엇인가" 한 줄 추가**
첫 슬라이드는 시적 카피만으로는 처음 듣는 청중이 *이게 앱인지 서비스인지 무엇인지* 잡기 어렵다. 평범한 정의 한 줄이 현재 **발표자 노트에만** 있고 화면엔 없으므로, 로고 아래 작게 한 줄을 얹어 톤(카피)과 이해(정의)를 둘 다 잡는다.

> **Lovv** — *서울·오사카 말고, 지금은 이곳*
> (작게, `Sub Text Gray`) 한일 소도시를 찾아주는 대화형 여행 추천 서비스

- 여기까지만: 서비스가 *어떻게* 작동하는지는 S5·본론 몫이다. S1은 **"무엇"까지만** 말하고 멈춘다(오프닝이 무거워지지 않게).
- 나머지(Local·Vibe·Voyage 도식, 비즈니스 포인트, 노트)는 그대로 둔다.

**발표자 노트 (오프닝 ~15초 · 키워드·뉘앙스 큐)**
- 인사 → "저희가 만든 건 **Lovv**"
- "서울·오사카 **말고** 한·일 소도시" · 대화형 추천 
- "오늘 — 왜·어떻게 만들었나"
- ⚠ 소개만. 문제(오버투어리즘)는 S2에서 — 여기서 미리 꺼내지 X.

**화면용 / 노트용 (S1은 이미 미니멀)**
- *화면용*: 로고 `Lovv` + 시적 카피 + 한 줄 정의 + Local·Vibe·Voyage 도식 + (비즈니스 포인트 작게). 글자 적정 — 추가로 덜어낼 것 없음.
- *노트/멘트*: 위 오프닝 대사(약 15초).

### S2 — 오버투어리즘이란? (흥미 강화) ★확정★

**이 슬라이드의 역할**: ① 오버투어리즘이 무엇인지 쉽게 소개하고, ② 워드클라우드로 현상을 *체감*시키며, ③ "왜 문제인지"(S3)로 자연스럽게 넘긴다. 청중이 처음 듣는다는 전제이므로 정의는 한 문장, 나머지는 후킹과 시각자료로 푼다.

**역할 분담 (S2 ↔ S3) — 중요**
S2는 **"현상 인지"**(무엇인가·어떻게 보이는가), S3는 **"비용 입증"**(왜 문제인가)을 맡는다. 따라서 *비용성 단어*(소음·쓰레기·주민 이탈·환경 훼손·교통난·문화 변형)는 S2에 쓰지 않고 **S3로 넘긴다.** S2 워드클라우드는 *현상·체감* 단어만 담아 두 장의 키워드가 겹치지 않게 한다.

**① 후킹 (화면 상단 또는 발표 오프닝)**
공감형 질문 한 줄로 연다. 예: **"다들 아는 그곳, 막상 가보니 사람만 보였던 적 없나요?"** → (한 박자 뒤) **"이게 바로 오버투어리즘입니다."**
- 대안 카피: *"같은 시간, 유명지는 발 디딜 틈 없고 — 한 시간 거리 소도시는 한산하다."* (S3·본론의 '분산' 메시지를 살짝 예고)

**② 정의 (쉽게 한 문장)**
> **오버투어리즘 = over + tourism.** 특정 관광지·시기에 *수용 능력을 넘는* 관광객이 몰리는 현상.

**③ 워드클라우드 (현상·체감 키워드만)**
- 권장 키워드: `오버투어리즘` · `인파` · `포토스팟 웨이팅` · `성수기 쏠림` · `핫플 집중` · `수용 한계 초과` · `유명지 편중` · `SNS 성지` · `한 곳 쏠림`
- 제외(→ S3로): 소음 · 쓰레기 · 주민 불편 · 환경 훼손 · 교통난 · 문화 변형
- 시각: 화면 중앙 워드클라우드 1개, 배경 `Soft Beige`, 핵심어 `Main Orange` 강조. 본문 텍스트 최소화(키워드로 "느끼게").
- [확인 필요] 수치를 1~2개 곁들이면 체감이 커진다(예: 특정 유명지 성수기 일일 방문객). 단 출처 확보 후 사용 — 미확보 시 키워드만.

**④ 수요의 반전 (흥미 포인트 — '쏠림의 원인'으로 착지) ★보강★**
현상을 보여준 뒤 한 박자 비틀어 흥미를 만든다. **단, 반전의 꼬리를 '솔루션'이 아니라 '쏠림의 원인'에 착지시켜 S3(비용)로 매끄럽게 넘긴다.** 핵심 메시지:
> **"사람들이 붐비는 걸 좋아해서 몰리는 게 아니다 — 안 붐비는 곳을 *못 찾아서*, 결국 한 곳에 *쏠린다*."**

- 재프레이밍: 오버투어리즘 = '무개념 관광객 탓'이 아니라 **'정보·발견의 실패'**. → 비난조를 피해 공감을 산다.
- 구체화(추상 → 체감): 소도시 한 곳을 가려면 **축제/관광지 일정 · 교통편 · 숙소 · 후기**를 앱 여러 개 띄워 따로 비교해야 한다 → 게다가 처음 가는 곳이라 **믿을 수 있는지도 확신이 안 서** → 결국 검색 잘 되고 익숙한 유명지로 회귀. "정보가 흩어져 *못 찾고*, 낯설어 *못 믿는다*"를 이 장면으로 보여준다(이 둘이 S5 컨셉 ②·③의 짝).
- **시각 보강 (작은 컷, 씨앗용)**: 이 장면을 말로만 두지 말고 **작은 보조 비주얼**로 곁들인다 — 앱 아이콘 5~6개 또는 브라우저 탭 여러 개가 흩어진 미니 컷. 단 **메인 시각은 워드클라우드(현상 인지) 유지**, 흩어진-앱 컷은 *작게* 둔다(크게 넣으면 S2가 '솔루션'으로 무거워지고 과밀해짐). 풀 버전(흩어진 앱 → Lovv 한 화면, 전후 대비)은 **데모에서 콜백으로 회수**(§2.9).
- **착지점은 "쏠림"**: 반전을 "그래서 우리가 찾아준다(솔루션)"로 끝내지 않는다. *"그래서 결국 한 곳에 쏠린다"*로 끝내면, 이 반전이 '쏠림의 **원인** 설명'이 되어 곧장 S3의 '쏠림의 **비용**'으로 이어진다(인과 강화).
- 솔루션 복선은 화면에 넣지 않고 **멘트로만** 가볍게: *"사람들이 안 가는 게 아니라, 못 찾는 거라면?"* — 답(발견 엔진)은 본론에서 회수(슬라이드당 메시지 1개 원칙).

**⑤ 전환 한 줄 (화면 읽기 순서의 '맨 끝'은 반드시 S3행)**
화면 하단 마지막 줄은 **비용(S3) 전환**으로 끝나야 한다. 청중이 S2를 떠날 때의 마지막 인상이 "솔루션이 온다"가 아니라 "이 쏠림은 비용을 남긴다"여야 S3 설득이 산다.
- **메인 전환(화면 하단) → S3(비용)**: *"문제는, 이 쏠림이 그냥 불편으로 끝나지 않는다는 것."* → (S3) "그래서 어떤 *비용*을 남기는가."
- **솔루션 복선(멘트만) → 본론**: *"…그리고, 못 찾는다는 것."* → 본론 발견 엔진으로 회수.
- 키워드 줄기: "쏠림" → S3 하단 "수요 분산" → 본론 "대체 소도시" / "못 찾는다" → 본론 솔루션.

> 화면 권장 순서: **후킹 → 정의 → 현상 워드클라우드 → 반전(못 찾아서 *쏠린다*) → 하단 전환 "이 쏠림이 비용을 남긴다 →"**. 순수 솔루션 암시는 화면에서 빼고 멘트로.

**발표자 노트 (키워드·뉘앙스 큐 · ~30~40초)**

*큐만 보고 말한다. 순서대로.*

1. **후킹** — "도착하니 사람만, 줄서기·포토스팟" · *(공감 질문, 던지고 한 박자 쉼)*
2. **정의** — 오버투어리즘 = over+tourism, *수용 넘는 쏠림*
3. **워드클라우드** — "이게 그 모습" / 인파·웨이팅·성수기
4. **반전** — 붐벼서가 아니라 *못 찾아서* + 낯설어 *못 믿어서* / 앱 여러 개 비교 → 유명지 회귀
5. **착지** — "그래서 더 쏠린다" → "비용을 남긴다(다음 장)" · *(다음으로 넘기는 텐션, 솔루션은 X)*

**3원칙**: ① 탓하지 않기(관광객 비난 X, '정보 흩어짐' 프레이밍) · ② 솔루션 아직 말 X(복선만, 회수는 본론 §2.9) · ③ 마지막 말은 '비용'으로(솔루션으로 끝내면 S3가 역행).

**화면용 / 노트용 분리 (밀도 관리) ★보강★**

S2도 요소가 많다(후킹·정의·워드클라우드·반전·전환). 화면은 **워드클라우드(메인) + 짧은 텍스트 3줄 + 작은 앱 컷**까지만.

*화면용 (이것만 올림):*
- 메인 시각: 현상 **워드클라우드 1개**
- **정의 한 줄**: over + tourism, 수용 능력을 넘는 관광객이 몰리는 현상
- **반전 한 줄**: *"안 붐비는 곳을 못 찾아서, 결국 한 곳에 쏠린다"* + 작은 흩어진-앱 컷(PPT 애니메이션 삽입)
- **하단 전환 한 줄**: *"이 쏠림이 비용을 남긴다 →"* (S3행)
- (옵션) 후킹 질문은 화면 상단에 *짧게* 넣거나, 아예 말로만

*노트/멘트로 (화면 X):*
- 후킹 질문 대사(말로 던지면 임팩트↑), 반전의 구체화(날씨·축제·교통·숙소·후기 앱 비교 장면 설명), 솔루션 복선 멘트("못 찾는 거라면?"), 발표자 노트 대본(위 작성분)

> 후킹은 *말로* 던지면 더 세다(화면엔 없어도 됨). 정의·반전·전환은 각 **한 줄**로만 — 길게 풀지 않는다. 흩어진-앱은 *작은* 컷(풀 전후 대비는 데모, §2.9).

### S3 — 왜 문제인가 (4개 축으로 확장) ★확정★

현재 S3는 "유명지 집중 ↔ 소도시 소외 → 지역 관광 경제 손실" 한 축에 집중돼 있다. 이를 조사된 **4개 축(주민 생활 · 환경 · 도시 기능 · 지역 문화)**으로 확장한다. 상세 근거·화면 구성·신뢰도 처리는 별도 문서 **`s3_problem_axes_revision.md`**에 정리했고, 핵심만 옮기면 다음과 같다.

- **4축으로 교체**: 단일 메시지 → **2×2 카드**. 축당 🇰🇷·🇯🇵 대표 수치 1개씩만 화면에 띄우고(예: 북촌 주민 1,500명 vs 연 664만 명 / 제주 폐기물 +74.1% / 감천 주민 40% 감소 / 후지산 하루 4,000명 제한), 상세는 발표자 노트로. 화면 수치는 웹 교차검증 완료(상세·주의는 `s3_problem_axes_revision.md` §3).
- **🇰🇷+🇯🇵 페어링은 의도**: 양국을 나란히 둔 건 "오버투어리즘은 한일 공통 문제"임을 보여주려는 것이며, 이는 곧 Lovv의 양방향 타깃과 통합 S4(인바운드 일→한 · 아웃바운드 한→일)를 예고한다 — "문제도, 시장도 한일 양방향."
- **경제 축은 제외(확정)**: 기존 가이드 4축 중 '지역경제 불균형'은 빼고 '지역 문화'를 넣는다. 경제·비즈니스 근거는 이미 **S4·S5(시장 근거)**가 담당하므로 중복을 피한다.
- **"유명지 집중↔소도시 소외"는 하단 전환 문구로 이동(확정)**: 4축(사회·환경 비용)과 층위가 다르므로 S3 하단으로 내려 다리 역할만 시킨다.
- **S4·S5 유지**: ③의 객관 근거를 받치는 자료이므로 서론에 그대로 둔다.
- **비즈니스 포인트는 "기회" 수준으로만 암시(확정)**: 같은 비용이 소외 소도시엔 *유치 기회*라는 양면성만 한 줄로 비춘다. B2G·송객 수수료 같은 구체 수익모델은 단정하지 않고, 발표자 노트나 본론 후반 **확장 방향**으로 미룬다(PoC 톤). 돈 내는 주체는 본론 데모의 **지자체 파트너 대시보드**에서 자연스럽게 드러낸다. 상세는 `s3_problem_axes_revision.md` §2.5.

**하단 전환 문구 (S3 → S4·S5 → 본론 연결, 확정)**
- 화면 카피: **"이 비용은 소수 유명지에 쏠린다 — 답은 수요의 분산이다."**
- 발표 멘트: "4가지 비용은 결국 소수 유명지에 수요가 몰려 생긴다. 소도시로 수요를 나누면 주민 부담은 줄고 소외 지역엔 기회가 생긴다. 그렇다면 그 수요는 정말 존재하는가? → 통합 S4에서 한일 양방향 데이터로 확인."
- 키워드 다리: S3의 **"수요 분산"** → S4의 **"소도시 수요"** → 본론의 **"대체 소도시 발견"**으로 같은 줄기를 잇는다.

**화면용 / 노트용 (이미 분리됨)**
S3의 화면/노트 분리는 `s3_problem_axes_revision.md`에 정리돼 있다 — 화면엔 **2×2 카드 + 축당 대표 수치 1개**만, 4축×2국 상세 수치·인용·신뢰도 주석은 전부 발표자 노트로(추가 작업 불필요).

### S4 — (통합) 한일 양방향 시장 근거 (인·아웃바운드 중심) + 한국 국내 보조 ★재구성★

기존 **S4(일본→한국 인바운드)와 S5(한국→일본 아웃바운드)를 한 장으로 합쳐, Lovv의 핵심인 *한일 국경 간 양방향* 수요를 보여준다.** 좌/우로 인바운드·아웃바운드를 나란히 두고(둘 다 수치 근거 강함), 하단에 **한국 국내 소도시 수요**를 *짧게* 보조로 얹는다. **일본인→일본 국내는 이 슬라이드에서 뺀다** — 근거가 점유율·정책 위주(증가율 펀치 부족)라 다른 칸과 세기가 안 맞기 때문. 제품은 4방향 모두 지원하되, 중간발표 근거 슬라이드는 *강한 것 위주*로 간다.

**핵심 메시지 (화면 카피)**
일본인은 한국 소도시로, 한국인은 일본 소도시로 — 소도시 수요는 이미 양방향으로 움직인다 (국내에서도 마찬가지)

**좌/우 — 한일 양방향 (메인, 칸당 대표 수치 1개)**

| 좌 — 인바운드 (일본 → 한국) | 우 — 아웃바운드 (한국 → 일본) |
| --- | --- |
| **"서울 다음은 어디?"** (라쿠텐, 국가관광전략회의) | 일본 소도시 예약 **+35%** (모두투어) |
| 외국인 3,000만 목표(2029)·FIT 90%+ | (노트) 구마모토 +261%·여객 2,746만·노선 31개 |

- (대안) 아웃바운드 대표를 임팩트 위주로 가려면 "+35%" 대신 **구마모토 +261%**(라쿠텐)로 교체 가능.

**하단 — 한국 국내 보조 (짧게 한 줄)**
> 국내에서도 같은 흐름 — 묵호·영월 등 **소도시 언급 +184%**(썸트렌드).
- 양방향(메인)이 핵심이고, 국내는 "같은 흐름이 국내에서도"라는 *보조 신호* 한 줄로만.
- 왜 한국 국내만 넣나: 데이터가 수치로 강하고(+184%) 청중(한국)에 가까워 검증·공감이 쉽다.

> **[일본 국내 빠짐 — Q&A 대비]** 제품은 일본인→일본 국내도 지원한다. "지방부 숙박의 90%가 일본인이고 정부가 마이크로투어리즘으로 분산을 미는 만큼 수요는 확실하다"로 답한다.

**노트용 — 나머지 수치(화면 X)**
- 인바운드: 외국인 1,893만→3,000만(2029), FIT 90%+
- 아웃바운드: 구마모토 +261%·홋카이도 +204%(라쿠텐, 한국 Z세대), 여객 2,746만·정기노선 31개, 사가 +611%
- 한국 국내: 영월 방문객 22만(영화·반값여행), 청령포 내비 +802.8% / MZ '소도시 재발견'
- (확장용·화면 X) 일본 국내: 지방부 숙박 90%가 일본인·同一현내 ~20%→35% 근거리화(관광청), 정부 분산정책·마이크로투어리즘(호시노리조트)

**하단 — 프로젝트 동기 (양방향+국내를 묶는 한 줄)**
> **수요도, 정보도 있다 — 흩어져 있어 '나에게 맞는 소도시'를 못 고를 뿐이다.**
> *(멘트) 그 흩어진 걸 모아 찾아주는 게 Lovv다.*
- 왜 이렇게 쓰나: "찾아주는 길이 없다"만 말하면 청중이 "정보 많잖아(구글·블로그·여행앱)"로 반박한다. **"정보도 있다"를 먼저 인정**하고 갭을 *흩어짐·비교 불가*로 좁히면, S2에서 보여준 "앱 여러 개 띄워 비교" 고통 장면이 떠올라 즉시 납득된다.
- 이 동기가 다음 장 S5(핵심 컨셉 3가지)로 바로 이어진다. 동기는 *왜 만드는가*(필요성)까지만, *어떻게*(컨셉)는 S5에서. "못 고른다 → 못 찾는다"가 S2 복선과 S5 '발견'으로 연결되는 키워드.

**근거 등급 (화면 3개 중심, 웹 검증 반영)**

| 칸 | 근거 세기 | 비고 |

### ★ 신설 S5 — 서론 마무리: 우리의 해결 방식 + 핵심 컨셉 3가지

가이드 ④에 대응하는 **새 슬라이드**. (A) 서론에서 던진 문제를 한 줄로 *정리*하고 → **① 프로젝트의 의미 + ② 어떤 방식으로 푸는지(핵심 컨셉 3가지 = 강점)**를 압축한다. 3개 컨셉은 **앞의 문제와 1:1로 연결**하고, (B) 컨셉(개인)과 의미(분산)를 한 줄로 잇는다. 소도시 *추천+일정*을 함께 짜준다는 점이 컨셉②에 드러나야 한다.

**(A) 서론 문제 한 줄 정리 (recap, 멘트) ★보강★**
> **여기까지 — 유명지 쏠림은 4가지 비용을 남기고(S3), 그럼에도 소도시 수요는 한일 양방향으로(+국내) 움직인다(S4). 막는 건 셋뿐이다: ① 대안을 *모르고* · ② 정보는 *흩어졌고* · ③ 낯선 곳은 *못 믿는다*(S2).**
- 서론 여정(현상→비용→수요)을 한 줄로 묶고, "막는 것 셋"이 그대로 아래 컨셉 3개의 짝이 된다. 이 recap이 있어야 청중이 문제를 *정리*한 뒤 해결로 넘어간다.
- **거시 루프 닫기(멘트 필수)**: recap 끝에 *"한마디로, **오버투어리즘을 '분산'으로 푼다**"*를 반드시 말한다. 화면엔 안 올라가도, 이 한 마디가 S2~S3에서 세운 헤드라인 문제("오버투어리즘")와 S3의 답("분산")을 명시적으로 잇는다(화면 closing이 해결책 위주라 거시 루프가 "분산" 한 단어에만 걸리는 걸 멘트로 보완).

**핵심 메시지 (화면 카피, 예시)**
조건만 말하면 — 나에게 맞는 소도시부터 여행 일정까지, 근거 있는 한 번의 대화로

**① 프로젝트의 의미 (한 줄) ★여행자 이득 강화★**
> **붐비는 유명지 대신 '나에게 맞는' 소도시로 — 여행자는 한적하고 더 나은 여행을 얻고, 그 선택이 모이면 과밀은 풀리고 소외된 소도시엔 기회가 된다. Lovv는 '나에게 좋은 선택'이 곧 분산이 되게 한다.**
- 화면용 짧은 버전: *"남들 다 가는 곳 말고, 나에게 맞는 소도시 — 더 나은 여행이 곧 분산이 된다."*
- 핵심: **여행자 이득을 *먼저*·구체적으로**, 사회적 의미(분산·소도시 기회·과밀 완화)는 그 *결과*로 잇는다 — **개인에게 좋은 선택 = 사회적으로도 좋은 결과**라는 win-win. S3의 "비용은 곧 양면 기회"를 여기서 회수.

**② 핵심 컨셉 3가지 = 서비스의 강점 (문제 → 컨셉 → 왜 강점/차별점) ★강점 프레이밍·일정 강조★**

흐름은 **발견 → 일정 → 근거** 순. 소도시 *추천*에서 끝나지 않고 **그 소도시의 여행 일정까지** 짜주는 게 ②의 핵심이다.

| 서론에서 깐 문제 | Lovv 핵심 컨셉 | 왜 강점·차별점인가 |
| --- | --- | --- |
| 대안 소도시를 몰라 막막 (S2·S4 "서울 다음은 어디?") | ① **대체 소도시 발견** — "어디로?" 대신 "언제·어느 나라로?"만 답하면 조건에 맞는 소도시를 *대신 찾아줌* | 도시를 정해놓고 검색하는 기존 여행앱과 *반대* — 목적지를 **제안받는** 탐색 |
| 정보가 흩어져 못 고름·비교 어려움 (S2·S4 동기) | ② **멀티스텝 Agent** — 추천 소도시의 **여행 일정(축제·날씨·동선)까지 한 번에 짜줌** | 추천에서 끝나지 않고 **일정까지** — 앱 여러 개 띄워 직접 짜맞출 필요가 없음 |
| 낯선 소도시는 믿고 고르기 어려워 유명지로 회귀 (S2 반전) | ③ **Explainable RAG** — 근거·출처·검증을 함께 제시 | 일반 생성형 추천과 달리 **'왜 이곳인지' 근거가 보여** 처음 가는 곳도 신뢰 |

**(B) 컨셉 ↔ 의미 연결 한 줄 ★보강★**
> **이 셋으로 개인의 '더 나은 발견 + 일정'을 돕고, 그 선택이 모이면 곧 분산(위 ① 의미)이 된다.** — 미시(개인 경험)와 거시(수요 분산)가 한 줄로 이어진다.

> 컨셉③의 문제축은 "근거 불투명"(서론에 없던 새 문제)이 아니라 **S2에서 깐 "낯선 소도시 → 유명지 회귀"**로 잡아 1:1 연결을 살렸다. 컨셉 문구는 S6·`00_project_plan.md`와 일치. 실제 컨셉 3개는 팀 합의에 맞춰 조정.

**화면용 / 노트용 분리 (밀도 관리) ★보강★**

S5는 정보가 많은 슬라이드라, 화면엔 **메시지 1 + 컨셉 카드 3 + 의미 1줄**만. 나머지(recap·강점 설명·1:1 상세·연결)는 발표 멘트로.

*화면용 (이것만 올림):*
- 상단 메시지: *"조건만 말하면 — 소도시부터 일정까지, 근거 있는 한 번의 대화로"*
- 중앙 **3 컨셉 카드** (각 = 작은 문제 태그 + 컨셉 + 한 줄, PPT 애니메이션 삽입):
  - `[대안을 모름]` ① **대체 소도시 발견** — "어디로?" 대신 "언제·어느 나라로?"
  - `[정보 흩어짐]` ② **일정까지 짜는 Agent** — 소도시 추천 + 여행 일정
  - `[못 믿음]` ③ **Explainable RAG** — 근거·출처가 보이는 추천
- 하단 의미 한 줄: *"더 나은 여행이 곧 분산이 된다"*
- **마무리 bookend (S1 태그라인 회수) ★보강★**: 슬라이드 맨 끝(또는 닫는 멘트)을 **S1의 그 문장 그대로** 받는다 — *"서울·오사카 말고, 지금은 이곳 — 그게 Lovv입니다."* S1에서 던진 카피를 S5에서 똑같이 닫아 서론을 수미상관으로 마무리(청중이 "처음 그 말!"을 느끼게). 패러프레이즈 말고 *원문 그대로* 써야 효과가 산다.

*노트/멘트로 (화면 X):*
- (A) 서론 recap 문장 — S4에서 넘어오는 *전환 멘트*로(화면에 문장 X)
- 각 카드의 "왜 강점·차별점인가"(표 셋째 열) + 문제 1:1 상세
- (B) 컨셉↔의미 연결 한 줄, 의미 긴 버전

> recap을 화면에서 빼는 대신 **카드마다 2~3단어 문제 태그**를 남기는 이유: 그래야 "문제→컨셉 1:1"이 화면에서도 보인다. 태그까지 빼면 화면엔 *해결책만* 남아 1:1 연결이 말에만 의존하게 된다.

**역할**: 이 슬라이드가 서론과 본론을 잇는 다리다. 청중은 여기서 "문제 → 해결 컨셉"을 머릿속에 정리한 뒤 S6(시스템 아키텍처 개요)로 진입한다.

## 2.9 복선-회수 짝 (S2 미끼 ↔ S5/본론 회수)

S2에서 던진 솔루션 복선을 뒤에서 닫는다. **회수의 정석은 "같은 단어를 다시 쓰는 것"** — S2에서 쓴 *못 찾는다 / 흩어져 있다*를 회수 지점에서 *찾아준다 / 모아준다*로 의도적으로 받아 청중이 "아, 아까 그거"로 연결하게 한다. 본론 슬라이드 제작 시 아래 짝을 그대로 가져다 쓴다.

| 복선 (S2에서 던짐) | 1차 회수 (S5 컨셉) | 완전 회수 (본론 데모/유저플로우) |
| --- | --- | --- |
| *"안 가는 게 아니라, 못 찾는 거라면?"* | 컨셉 ① **대체 소도시 발견** — "어디로?"가 아니라 "언제·어느 나라로?"에서 시작 = 답의 방향 제시 | "S2에서 12개 앱 띄우던 게 → 이제 한 번의 대화로" 식 **콜백 데모** |
| *날씨·축제·교통·숙소·후기가 흩어져 비교 어렵다* | 컨셉 ② **멀티스텝 Agent** — 흩어진 정보를 일정으로 한 번에 **모아준다** | 실제로 분산 정보가 한 화면에 모이는 유저플로우 시연 |
| *(검색 잘 되는) 유명지로 회귀* | 컨셉 ③ **Explainable RAG** — 근거·출처가 보이니 처음 가는 소도시도 **믿고** 선택 | 추천 이유·출처가 함께 나오는 결과 화면 |

**회수 운영 규칙**
- S2의 미끼 문구와 회수 문구는 **미리 짝지어** 확정한다(워딩이 어긋나면 콜백 효과가 사라짐).
- 1차 회수(S5)는 *답의 방향*만, 완전 회수(데모)는 *실제 동작*으로 — 두 번에 나눠 닫아 김이 빠지지 않게 한다.
- 본론 데모에서 S2에서 보여준 "고통 장면(앱 여러 개 비교)"을 **그대로 다시 띄웠다가** Lovv로 해결되는 대비를 보여주면 회수가 가장 강력하다.
- **시각 회수 규칙**: S2에는 흩어진-앱 *작은 컷(씨앗)*만, 데모에서 **같은 그림을 키워** "이게(흩어진 앱들) → 이렇게(Lovv 한 화면)" 전후 대비로 터뜨린다. 같은 비주얼을 두 번 써야 "아, 아까 그거"가 완성된다(S2 컷과 데모 컷의 아이콘·레이아웃을 의도적으로 일치시킬 것).

## 3. 권장 서론 최종 흐름

```
S1 프로젝트 소개
  └ 시적 카피 + 로고 아래 "무엇인가" 한 줄 정의 (인상 + 이해)
S2 오버투어리즘이란? (후킹 + 정의 + 현상 워드클라우드 + 반전[못 찾아서 쏠린다] → 하단은 S3 비용行, 솔루션은 멘트 복선)
S3 왜 문제인가 (주민생활·환경·도시기능·지역문화 4축 + 하단 전환 문구)
S4 (통합) 한일 양방향 — 좌: 인바운드(일→한) / 우: 아웃바운드(한→일) + 하단 한국 국내 보조(+184%) + 프로젝트 동기  ※일본 국내는 확장으로
S5 ★신설★ 서론 마무리 — 프로젝트 의미 + 해결 방식(핵심 컨셉 3가지, 문제↔컨셉 1:1)
──────── 여기서 본론(S6 시스템 흐름) 진입 ────────
```

S4·S5 통합(−1장)과 컨셉 슬라이드 신설(+1장)이 상쇄돼 서론은 **5장**(S1~S5)으로 유지된다. 전체는 가이드의 10~20장 권장 범위 안.
