## 발표자 노트

> 키워드 큐 — 순서대로.

- **협업 원칙** — 기획, 일정, 코드, 문서, 디자인을 따로 관리하되 산출물 흐름은 끊기지 않게 맞춘다.
- **진행·일정** — Notion으로 데일리 스크럼과 진행 상황을 공유하고, GitHub Projects로 WBS를 관리한다.
- **개발** — GitHub에서 역할별 구현과 PR 리뷰를 진행한다.
- **문서·산출물** — 요구사항, 설계, 발표 자료는 GitHub Pages 문서 사이트에서 공유한다.
- **기획·디자인** — 회의록과 서비스 기획은 Notion, 화면 설계는 Figma로 분리한다.
- **효과** — 각 도구가 따로 놀지 않고 다음 장의 진행 상황을 설명할 수 있는 근거가 된다.

## 제작 체크
