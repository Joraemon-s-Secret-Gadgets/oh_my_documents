## 발표자 노트

> 키워드 큐 — 순서대로.

- **협업 원칙** — 영역별로 도구를 나누되 한 흐름으로 / 기획·일정·코드·산출물 끊김 없이
- **진행·일정** — Notion 애자일·데일리 스크럼으로 진행상황 공유 / WBS는 GitHub Projects로 관리
- **개발** — GitHub 역할 분담 + PR 코드 리뷰
- **문서·산출물** — 명세·산출물을 GitHub Pages 사이트로 관리·공유
- **기획·디자인** — 서비스 기획·회의록은 Notion / UI는 Figma
- **효과** — 협업이 한 흐름이라 진행 상황(다음 장) 추적이 쉬움
