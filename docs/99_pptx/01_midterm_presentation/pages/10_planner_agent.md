## 화면 문구

**검증된 정보만 일정에 씁니다**

| 단계 | 처리 |
| --- | --- |
| 후보 입력 | 도시·장소·축제 |
| 일정 배치 | 오전·오후·저녁 |
| 검증 | 장소·날짜·조건 |
| 사용자 출력 | 일정·이유·링크 |

**검증 규칙**
후보 외 장소 금지 / 검증된 축제만 / 한계 안내

## 발표자 노트

- Planner는 새로운 장소를 검색하는 Agent가 아니라, Candidate Evidence가 넘긴 후보를 일정으로 배치하는 Agent다.
- 일정은 오전·오후·저녁처럼 사용자가 이해할 수 있는 시간대 단위로 만든다.
- 축제는 날짜가 확인된 경우에만 일정에 직접 넣고, 미확인 축제는 참고 정보나 안내 문구로 분리한다.
- 맛집·숙소처럼 실시간성과 상업 플랫폼 의존성이 큰 항목은 직접 확정 추천하지 않고 탐색 링크로 연결한다.
- 정보가 부족하면 지어내지 않고 자유시간, 대체 안내, 신뢰도 하향으로 처리한다.
