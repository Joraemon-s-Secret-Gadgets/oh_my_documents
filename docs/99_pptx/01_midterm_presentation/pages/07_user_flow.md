# 슬라이드 7 — 사용자 흐름

> 원본 위치: `../01_midterm_presentation.md`
> 상태: Slide Content
> 역할: 사용자가 도시가 아니라 조건에서 시작한다는 차이를 설명

## 화면 문구

**도시를 몰라도 시작합니다**

조건 입력 → 소도시 추천 → 일정 실행

| 단계 | 내용 |
| --- | --- |
| 조건 입력 | 국가 · 시기 · 취향 |
| 소도시 추천 | 도시 1곳 · 추천 이유 |
| 일정 실행 | 시간대 일정 · 지도 링크 |

## 발표자 노트

- Lovv는 사용자가 도시명을 알고 있다고 가정하지 않는다.
- 사용자는 "일본, 5월, 조용한 바다, 축제"처럼 조건만 말하면 된다.
- 시스템은 국가와 시기, 취향을 구조화한 뒤 소도시 1곳과 추천 이유를 먼저 보여준다.
- 이후 선택한 목적지를 오전·오후 일정, 지도, 상세 링크로 실행 가능한 형태로 바꾼다.
- 화면에는 흐름만 남기고, 구체 예시는 발표자가 말로 보충한다.
