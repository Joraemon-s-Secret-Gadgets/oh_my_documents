# 슬라이드 B5 — 활용 데이터

> 원본 위치: `../01_midterm_presentation.md`
> 상태: Slide Content
> 역할: 어떤 데이터를 어떤 추천 판단에 쓰는지 요약

## 화면 문구

**데이터는 많이 모으는 것이 아니라, 추천 판단 단위로 정리한다**

| 데이터 | 용도 | 추천 연결 |
| --- | --- | --- |
| 도시·지역 | 후보 기본 단위 | 소도시 후보 생성 |
| 관광지 | 방문 장소 후보 | 일정 구성 |
| 축제·이벤트 | 시기성 판단 | 여행 월 매칭 |
| 날씨 경향 | 계절 적합성 | 추천 보정 |
| 지도·링크 | 실행 가능성 | 상세 확인·이동 |

## 레이아웃

| 영역 | 내용 |
| --- | --- |
| 중앙 | 데이터, 용도, 추천 연결 3열 표 |
| 하단 | 데이터 양보다 추천 판단 단위가 중요하다는 메시지 |

## 발표자 노트

- PoC에서는 데이터 양보다 추천 판단에 필요한 형태로 정리하는 것이 중요했습니다.
- 도시, 축제, 날씨, 장소 데이터가 Candidate Evidence로 연결됩니다.

## 제작 체크

- [ ] 데이터 목록을 나열하지 말고 추천 용도와 연결한다.
- [ ] 표는 5행 이내로 유지한다.
