## 발표자 노트

- **프레임** — 4주 동안 문제 정의에서 PoC 검증까지 단계적으로 좁혀왔다.
- **스프린트 운영** — 1주 단위 스프린트로 진행했고, 1주차는 킥오프 기간이라 요구사항과 협업 환경을 먼저 고정했다.
- **1주차** — 프로젝트 기획, 요구사항 정의, WBS, 협업 환경 구축을 완료했다.
- **2주차** — 추천에 필요한 데이터 기준을 잡고, 한국 40개 도시를 우선 검증하면서 데이터 수집과 DB 설계를 진행했다.
- **3주차** — Candidate Evidence와 Planner 중심으로 멀티에이전트를 재설계하고, RAG와 S3 Vector 구조를 잡았다.
- **4주차** — PoC 흐름을 구현하고, 인증/API, 프론트 목업까지 연결해 중간 발표에서 보여줄 수 있는 상태로 만들었다.
- **현재** — 핵심은 "조건 입력 → 근거 후보 → 일정 생성" 흐름이 end-to-end로 이어진다는 점이다.
- **다음** — 중간 발표 이후에는 데이터 범위, 추천 품질, 자동 갱신, 운영 기능을 Production 범위로 확장한다.
