## 화면 문구

**PoC -> Production 확장**

> 좌우 2단 대비 — 왼쪽은 PoC에서 검증한 현재, 오른쪽은 Production에서 확장할 방향.

| 확장 방향 | 현재 (PoC) | → 확장 (Production) |
| --- | --- | --- |
| 데이터 범위 | 한국 40개 도시 | 한국 전국 + 일본 전국 도시 |
| 추천 품질 | 사용자 의도 반영 제한적 | 의도 구체화 — 해시태그 매칭으로 추천 요소 확대 |
| 데이터 갱신 | 정적·수동 데이터 | 축제·날씨·장소 자동 갱신 파이프라인 |
| 운영·파트너 | 추천 경험에 집중 | 지자체 대시보드 + 관리자 검토 |

## 발표자 노트

- **프레임** — PoC 핵심 가설 좁게 검증 / "여기까지 됐고 → Production 이렇게 확장" · 4개 축
- **데이터 범위** — 한국 40개 도시 → 한국 전국 + 일본 전국
- **추천 품질** — 현재 intent 활용 적음 → 의도 구체화 / 관광지 설명·사용자 입력 양쪽 해시태그 추출 → 일치 매칭 / 예: '산' 태그 → 산 관련 목적지 확장 인출 → 추천 점수 추가 요소 / 평가 기준 아님, 요소 자체 확대
- **데이터 갱신** — 정적·수동 → 축제·날씨·장소 자동 갱신 파이프라인 / 축제일 공식 출처 실시간 검증 고도화
- **운영·파트너** — PoC 추천 경험 집중 → 지자체 대시보드(노출·클릭·관심도) + 관리자 검토(제안·승인) / 수익 모델 단정 X, 방향만 / 비용 = 소도시 유치 기회(양면성 회수)
- **마무리** — 핵심 가설 검증 완료 → 데이터·품질·운영 확장 / 출처·Q&A 연결
