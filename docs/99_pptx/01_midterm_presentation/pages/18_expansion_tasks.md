## 화면 문구

**PoC -> Production 확장**

> 좌우 2단 대비 — 왼쪽은 PoC에서 검증한 현재, 오른쪽은 Production에서 확장할 방향.

| 확장 방향 | 현재 (PoC) | → 확장 (Production) |
| --- | --- | --- |
| 데이터 범위 | 한국 40개 도시 | 한국 전국 + 일본 전국 도시 |
| 추천 품질 | 조건 기반 추천 | 의도·피드백 반영 |
| 데이터 갱신 | 정적·수동 데이터 | 자동 갱신 파이프라인 |
| 운영·파트너 | 추천 경험 중심 | 관리자 검토·지역 운영 |

## 발표자 노트

- **프레임** — PoC는 핵심 가설을 좁게 검증했고, Production은 같은 흐름을 넓히는 단계다.
- **데이터 범위** — 현재는 한국 40개 도시를 기준으로 먼저 검증했고, 이후 한국 전국과 일본 전국 도시로 확장한다.
- **추천 품질** — 현재는 국가, 시기, 취향 조건 중심이고, Production에서는 온보딩 선호, 피드백, 더 세분화된 의도 태그를 반영한다.
- **데이터 갱신** — 정적·수동 데이터에서 축제 날짜, 장소 정보, 날씨 경향을 주기적으로 갱신하는 파이프라인으로 확장한다.
- **운영·파트너** — 사용자 추천 경험에서 끝나지 않고, 관리자 검토와 지역 운영자가 데이터를 제안·관리할 수 있는 구조로 넓힌다.
- **마무리** — 다음 단계의 핵심은 기능을 많이 붙이는 것이 아니라 검증된 추천 흐름을 데이터·품질·운영 측면에서 키우는 것이다.
