## 화면 문구

**좋아 보이는 후보가 아니라, 기준에 맞는 후보를 고른다**

| 평가 기준 | 의미 |
| --- | --- |
| 계절성 | 여행 월과 축제·날씨 경향이 맞는가 |
| 테마 적합도 | 사용자의 취향과 도시 콘텐츠가 맞는가 |
| 실행 가능성 | 일정으로 만들 수 있는 장소와 동선이 있는가 |
| 근거 품질 | 출처와 설명이 충분한가 |

## 레이아웃

| 영역 | 내용 |
| --- | --- |
| 좌측 | 후보 A/B/C 카드 |
| 중앙 | scoring 기준 4개 |
| 우측 | 최종 1개 추천 + reserve 후보 |

## 발표자 노트

- 검색 결과를 그대로 보여주면 추천 품질이 흔들립니다.
- Ranker는 최적 공식을 주장하기보다, 왜 이 후보를 골랐는지 설명 가능한 기준을 남기는 장치입니다.
- PoC에서는 완전 자동 최적화보다 검증 가능한 선정 흐름을 우선했습니다.
