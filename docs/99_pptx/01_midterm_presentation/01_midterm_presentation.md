## 슬라이드 1 — 프로젝트 소개 (Lovv)

**핵심 메시지 (화면 카피)**
서울·오사카 말고, 지금은 이곳

**서브 카피**
지도에서 흐릿한 곳, 여행에선 가장 빛나는 곳

**화면 구성**
- 중앙: 로고 타이포 "Lovv"
- 하단: Local(소도시) · Vibe(분위기·감성) · Voyage(여행) 3개 아이콘 도식
- 본문 텍스트 없음. 배경 `Soft Beige`, 로고 `Main Orange`

**비즈니스 포인트 (작게)**
한일 소도시 여행 추천 / 인바운드·아웃바운드 양방향 타깃

**발표자 노트**
- Lovv = Local + Vibe + Voyage. 혼잡한 유명 관광지 대신 덜 알려진 소도시·지역 경험을 추천하는 대화형 여행 추천 서비스.
- 한 줄 정의: 한일 여행자에게 대도시 대신 대체 소도시를 발견하게 하는 "목적지 탐색 엔진".
- 첫 질문은 "어디로?"가 아니라 "언제, 어느 나라로?" — 도시를 이미 정했다고 가정하지 않는다.

## 슬라이드 2 — 문제: 오버투어리즘 (현상)

**핵심 메시지 (화면 카피)**
특정 관광지·시기에 수요가 과도하게 집중된다

**화면 구성**
- 화면 전체: 워드클라우드 1개
- 키워드: 혼잡 · 주민 불편 · 환경 훼손 · 만족도 저하 · 수요 집중 · 성수기 · 유명 관광지 편중 · 소도시 정보 부족
- 본문 텍스트 없음 (키워드로 문제를 "느끼게" 함)

**발표자 노트**
- 오버투어리즘: 특정 관광지·시기에 방문 수요가 과도하게 몰리며 혼잡·주민 불편·환경 훼손·만족도 저하를 유발하는 문제.
- 여행자는 덜 붐비는 곳을 원하지만, 소도시 정보(계절성·축제·이동·주변 관광)는 흩어져 있어 비교가 어렵다.

> [확인 필요] 워드클라우드에 통계 수치를 넣으려면 출처 데이터 별도 필요. 현재는 키워드만 존재.

## 슬라이드 3 — 문제의 영향 (왜 중요한가)

**핵심 메시지 (화면 카피)**
유명지 집중 ↔ 소도시 소외

**화면 구성**
- 좌우 대비 도식 1개: (좌) 유명 관광지 인파 집중 / (우) 소도시 노출 기회 부족
- 하단 한 줄 강조 (`Muted Brown`)

**비즈니스 포인트 (하단 강조 한 줄)**
수요 불균형 = 지역 관광 경제 손실

**발표자 노트**
- 일부 지역에 관광객이 집중되는 동안 다른 지역은 노출 기회를 잃는다 → 지역 관광 수요 불균형.
- 데이터(관광지·축제·교통·날씨·지도)가 여러 플랫폼에 분산돼 여행자가 대안을 찾기 어렵다.
- 즉, 여행자 불편이자 동시에 지역 관광 경제의 기회 손실 문제다.

## 슬라이드 4 — 시장 근거 ① 일본→한국 인바운드: "서울 다음은 어디?"

**핵심 메시지 (전문가 인용)**
> "일본 젊은 여성들은 한국을 깜짝 놀랄 정도로 다 압니다. 홍대도, 성수도, 치킨도, K팝도. 진짜 문제는 '서울 다음은 어디인가'입니다."

**출처·화자**
서태석 — 라쿠텐트래블 한국지사장 / 제11차 국가관광전략회의(이재명 대통령 주재) / 데일리브리프, 2026.3.14

**화면 숫자 (3개)**

| 숫자 | 라벨 |
| --- | --- |
| `1,893만→3,000만` | 외국인 관광객 (2025 → 2029 목표) |
| `FIT 90%+` | 일본 개별자유여행 비중 (2016 이후) |
| `서울 다음?` | 일본 인바운드의 미해결 질문 |

**화면 구성**
- 직함 + 큰따옴표 핵심 문장 1개 (AWS 키노트 "전문가 인용" 형식)
- 인물 사진 대신 `국가관광전략회의 / 대통령 주재` 라벨로 권위 강조
- 보조 숫자 3개를 `Card Peach` 카드로

**비즈니스 포인트**
대통령 주재 회의에서 업계 전문가가 직접 제기한 "서울 다음은 어디?" = Lovv가 정확히 푸는 문제. "소도시 분산 수요"는 가설이 아니라 국가 관광정책의 핵심 과제다.

**발표자 노트**
- 우리가 정의한 문제(유명 도시 다음의 소도시 발견)는 제11차 국가관광전략회의에서 라쿠텐 한국지사장이 직접 제기한 질문이다.
- 일본 여행자는 서울을 반복 방문해 다 안다(한 일본 직원은 한국을 52번 방문 — *개별 일화*). 다음 갈 곳을 못 찾는 것이 핵심 공백.
- 정부 목표(2029년 3,000만 명), 수도권→지역 분산 전략, 일본 FIT 90%+ 전환이 모두 Lovv 방향과 일치.
- 이재명 대통령은 이 제안에 "애정에 기반한 아주 진지한 제안에 감사드린다"고 답함 — 정책적 무게를 보여주는 마무리 멘트로 활용 가능.
- (보조 근거) 일본 내 소도시 예약 증가: 라쿠텐 김태광 **동북아시아 비즈니스 총괄 이사** 자료 기준 홋카이도 +204%, 구마모토 +261% (서울경제TV 2025.7.22).

> [사실 주의] 서태석(라쿠텐트래블 **한국지사장**)과 김태광(라쿠텐 트래블 **동북아시아 비즈니스 총괄 이사**)은 **다른 인물**이다. 직함을 섞지 말 것.
> [신뢰도·범위] 수치·발언은 기사 직접 인용으로 높음. "52번 방문"은 개별 일화. 5149는 *일본 인바운드* 관점이므로 한일 양방향 전체를 1:1 입증하지 않는다 → 이 슬라이드는 "인바운드 축" 근거.

## 슬라이드 5 — 시장 근거 ② 한국→일본 아웃바운드: 소도시 수요 + 항공 공급

**핵심 메시지 (화면 카피)**
한국 여행자도 일본 소도시로 — 수요와 공급이 함께 움직인다

**보조 인용 (기사형)**
> "도쿄·오사카 등 대도시 수요가 견조한 가운데 최근에는 시즈오카·다카마쓰·사가 등 소도시를 찾는 여행객이 빠르게 늘고 있다." (한국경제, 2026.2.6)

**화면 숫자 (3개)**

| 숫자 | 라벨 |
| --- | --- |
| `+35%` | 일본 소도시 상품 예약률 (모두투어, 2026.5 전년 동기 대비) |
| `2,746만` | 2025년 일본 노선 여객 수 (국제선 1위) |
| `LCC+FSC` | 일본 지방공항 취항 확대 (공급이 수요를 따라감) |

**화면 구성**
- 상단: 기사형 인용 텍스트 카드(`Travel Trend`)
- 하단: 숫자 강조 3개 + 작은 "LCC·FSC 지방공항 취항" 도식 (개별 노선은 노트로)
- 인물 사진 미사용

**비즈니스 포인트**
일본 소도시 수요 증가는 감각적 트렌드가 아니라, 여객 수 증가(수요)와 LCC·FSC의 지방공항 취항 확대(공급)로 동시에 확인되는 실수요 변화다. "소도시 분산 수요"는 항공·여행업계가 이미 대응 중인 시장 변화다.

**발표자 노트**
- 수요: 모두투어 집계 2026년 5월 일본 *소도시* 상품 예약률 전년 동기 대비 +35%. 다카마쓰(시코쿠 카가와현)·나오시마(예술의 섬)가 대표 부상지. 일본 재방문객이 유명지보다 지역 문화·자연·미식을 찾는 흐름.
- 교통 실수요: 2025년 일본 노선 여객 2,746만3889명(국제선 국가 1위), 인천공항 일본행 정기 노선 31개로 확대(나리타 18·이타미 26보다 많음). 신규 소도시 정기 노선에 오비히로·이바라키 포함.
- 공급(LCC): 제주항공 시즈오카 1일 2회 증편, 에어부산 부산~시즈오카 주 3회, 에어로케이 청주~마츠모토(주 3회 한시)·청주~고베, 티웨이 제주~도야마·제주~이즈모. 엔저 배경 지방공항 취항 확대.
- 공급(FSC): 대한항공 일본·중국 지방공항 포함 30여 개 노선망 유지, 인천~아오모리 주 3회→주 5회 증편.
- Lovv는 이 변화된 수요를 더 쉽게 탐색·연결하는 도구.

> [신뢰도·범위] 모두투어 +35%는 단일 여행사 내부 집계(신뢰도 높음, 업계 전체 아님). LCC 개별 노선 수치는 정리 매체 기반이라 항공사 공식 공지 1차 확인 권장(중간). 통폐합 감편 병존(인천~구마모토 단항 2025.10.26, 인천~하코다테 휴항 2026.6.8) → 화면에는 개별 노선이 아니라 "지방공항 취항의 **구조적 확대**" 방향성만 표기.

## 슬라이드 6 — 해결: Lovv 시스템 흐름 (Explainable RAG)

**핵심 메시지 (화면 카피)**
Explainable RAG — 추천에 근거·출처·검증을 함께

**화면 구성**
- 좌→우 파이프라인 다이어그램 1개:
  사용자 입력(국가·시기·취향) → RAG 검색 → 멀티스텝 Agent → 근거 포함 추천 결과
- (선택) Agent 내부 단계 확대: Intent → Supervisor → Retriever/Festival Verifier → Ranker → Itinerary Writer → Output Validator

**비즈니스 포인트**
단순 생성형 답변이 아니라 "근거가 보이는 추천". 슬라이드 4·5에서 확인한 한일 양방향 소도시 수요를, 출처·검증이 따라붙는 추천 경험으로 연결한다.

**발표자 노트**
- 사용자가 국가·시기를 먼저 입력 → 조건에 맞는 대체 소도시 1곳을 RAG로 검색·선정.
- 멀티스텝 Agent가 일정·축제·날씨·동선을 구성하고 추천 이유·출처·검증 상태를 함께 제시.
- 차별점: Explainable RAG. 시스템 5계층 — Client Web / Backend API / Recommendation Agent / Data Store / External APIs.

**마무리 (PoC → Production)**
- 중간발표(PoC): P1 핵심 기능이 좁은 범위에서 실제 동작함을 보여준다(범위: P1 구현, P2~P3 설계, P5 제외).
- 최종발표(Production): 라이브 데모 + 추천 품질·응답속도(추천 8초·RAG 1초 목표) 검증 + 지자체 파트너 확장.
