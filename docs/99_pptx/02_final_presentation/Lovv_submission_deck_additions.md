# 제출용 PPT — 발표용 덱 대비 추가·보강 슬라이드 정리

- 기준: 프로젝트 발표회 발표자료 가이드 vs 현재 발표용 `Lovv_v8.pptx`(32장)
- 원칙: 발표용 덱은 그대로 두고, 제출용은 여기서 정리한 **신규 6~8장 추가 + 기존 3곳 보강**으로 파생한다. 가이드의 나머지 항목(주제·배경·차별성·구조·기술·구현 결과)은 기존 슬라이드가 이미 충족한다.

---

## B. 슬라이드별 추가·보강 내역

### [보강] 1p 표지
- 팀원 명단 아래 **멘토 성함·소속** 한 줄 추가. (팀 프로젝트명·팀명·팀원은 이미 충족)

### [신규 ①] "개발 환경 · 훈련 연관성" — 3p 목차 뒤 삽입
- 헤드라인(안): **"배운 것으로 만들었다 — 개발 환경과 커리큘럼 연계"**
- 좌: 개발 환경 표 — 프론트(React 19·Vite·Tailwind·Vercel) / 백엔드(AWS SAM·Lambda·Cognito) / AI(Bedrock AgentCore·LangGraph·gpt-oss-120b·Titan Embed) / 데이터(DynamoDB·S3 Vectors·TourAPI) / 협업(Notion·Discord·GitHub Projects·Figma). 14p 아키텍처에서 추출만 하면 됨.
- 우: 훈련 과정 연관성 — 커리큘럼 모듈(LLM/RAG, 에이전트, AWS 인프라, 데이터 전처리, 협업)을 프로젝트 적용 지점과 1:1 매핑한 표. ※ 커리큘럼 모듈명은 팀이 확인해 기입.

### [신규 ②] "활용 방안 · 기대 효과" — BM(비즈니스 모델) 슬라이드 뒤 삽입
- 헤드라인(안): **"추천 하나가 지역으로 흐른다 — 활용 방안과 기대 효과"**
- 3열 구성: 사용자(정보 탐색 비용 절감 — 설문 근거: 불편 86%·재이용 71%) / 산업(예약 플랫폼 전환 트래픽 — 돈 낼 기능 1위 '예약 연결' 71%) / 공공(오버투어리즘 분산·지역 관광 활성화 — 5p 문제의식 회수, B2G 콘솔 연계).
- 기존 5p·23p(BM)·24p(B2G) 데이터 재조합이라 신규 조사 불필요.

### [신규 ③] "팀 구성 및 역할" — 목차 바로 뒤(개요 앞) 삽입
- 헤드라인(안): **"조라에몽의 만능 도구들 — 역할과 책임"**
- 4인 카드: 이름 / 역할 태그 / **주도적으로 참여한 부분** 2~3개 (가이드가 '주도 참여 중심'을 명시 — 균등 나열 금지).
  - 예시 틀: 최수아(기획·시장분석·설문 설계/분석·BM), 이창우(시스템·에이전트 아키텍처·검증), 전종혁·조동휘(※ 담당 파트 팀 확인 후 기입 — 데이터 파이프라인/프론트/인프라 등)
- 하단: 멘토 성함 + 담당(멘토링 범위).

### [신규 ④] "수행 절차 · WBS" — 팀 구성 뒤 삽입 (1~2장)
- 1장차: 단계 타임라인 — 기획·요구정의 → 시장조사·사전 설문 → 데이터 수집/전처리 → 설계(DB·에이전트) → 구현 → 검증·배포 → 사후 설문·개선. 각 단계에 산출물 문서(`docs/00~11`) 매핑하면 문서화 역량 어필.
- 2장차(선택): 주차별 WBS 간트 — 담당자 표기. 소스: `docs/00_project_plan`.
- 트러블슈팅(현 16번째 장)과 배치가 이어지면 "계획 → 실행 → 막힌 지점 대응"의 서사가 생긴다.

### [신규 ⑤] "피드백과 보완 과정" — 사용자 검증 슬라이드 뒤 삽입
- 헤드라인(안): **"들은 것을 고쳤다 — 피드백 반영 내역"**
- Before→After 표 2블록:
  - 멘토·중간발표 피드백 → 반영 내용 (소스: `docs/99_pptx/01_midterm_presentation/meta/presentation_review_feedback_2026-06-13.md`)
  - 사용자 설문 주관식(스크롤·저장 버그, UI 밀도, 축제 최신성, 후기 부재) → 수정 완료 / 로드맵 구분 표기
- 반영 못 한 항목을 숨기지 말고 '로드맵'으로 명시 — 자체 평가(⑥)와 연결됨.

### [신규 ⑥] "자체 평가" — 감사합니다 앞 삽입 (1~2장)
- 완성도 자평: **n/10점** + 채점 근거 한 줄 (점수는 팀 합의로 결정. 근거 후보: 프로덕션 배포 완료·전 경로 실동작 vs UI 직관성 61%·표본 한계)
- 잘한 점: 데이터로 결정한 것들(설문 2회·트러블슈팅 3건의 도입 전 검증·WTP 기반 BM 유보)
- 아쉬운 점: UI 직관성 61%, 사후 표본 n=31, 일본 데이터 취득 중단
- 개선·보완 계획: 사용자 후기, UI 리디자인, 일본 공공 데이터 재검토, B2B 제휴 확정 (25p 확장과 정합 유지)
- 느낀 점: 팀원별 한 줄 (개인 작성)

### [보강] 결과물 화면 — 제출용은 placeholder 불가
- 12p 핵심 컨셉 캡처 3종, B2G 관리자 콘솔 캡처, 부록 A(경쟁 서비스)·C(에이전트)·D(콘솔) 캡처 전부 삽입. 발표용은 구두로 때울 수 있어도 제출용은 심사자가 문서만 본다.

## C. 별도 산출물 — 시연 영상 (5~10분 · 100MB 이하)

발표 영상이 아니라 **기능 시연** 요구. 구성안(총 ~7분):

1. 온보딩·테마 선택 (30초) → 2. 자연어 조건 입력·AI 일정 대화 (2분, 핵심) → 3. 추천 결과·추천 이유·출처 표시 (1분 30초 — '납득' 증명) → 4. 일정 상세·대안 일정 (1분) → 5. 지도 탐색·랭킹 (1분) → 6. 관리자 콘솔: 제안→승인→감사 로그 (1분 — B2G 주장의 증거).

- 화면 녹화 + 자막(또는 차분한 나레이션). 100MB 제한 → 1080p H.264, 비트레이드 ~1.5Mbps 내외로 인코딩.
