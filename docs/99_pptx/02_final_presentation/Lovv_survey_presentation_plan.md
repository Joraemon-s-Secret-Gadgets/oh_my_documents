# Lovv 최종 발표 — 설문 기반 스토리라인·슬라이드·대본 기획안

- 근거 데이터: 사용 전 설문 N=84(오차범위 ±10.7%p), 사용 후 설문 N=31(±17.6%p). 모든 수치는 원본 xlsx `설문지 응답 시트1`에서 직접 집계·검증함(문서 말미 대조표 참조).
- 기준 자료: `Lovv_v7.pptx`(35장), `Lovv_final_script.md`
- 이 문서의 역할: 대본의 placeholder(7p, 부록 B 31~32p)를 확정 수치로 채우고, 사용 후 설문을 본편에 편입하는 신규 슬라이드 1장을 제안한다. 확정되면 `Lovv_final_script.md`에 반영한다.

# 0. 현재 구성의 갭 — 먼저 짚는다

**갭 1. 사용 후 설문이 본편에 없다.** 현재 본편의 설문 슬라이드는 7p(사용 전) 하나뿐이고, 사용 후 결과는 부록(32p)에만 있다. 사용 후 설문은 이 발표에서 유일한 "제품이 실제로 작동했다"는 1차 증거인데, 부록에 두면 평가관 대부분이 못 본다. 특히 사용 후 데이터는 22p 비즈니스 모델의 논리(B2C 과금 유보, B2B 중심)를 정면으로 뒷받침하므로 본편 편입이 필요하다.

**갭 2. 두 설문을 잇는 스토리가 없다.** 사전·사후를 각각 나열하면 "설문 했습니다" 이상이 안 된다. 두 설문에는 같은 축으로 비교 가능한 문항이 있고(신뢰 요소: 사전 Q14 ↔ 사후 Q1 / 기대 기능: 사전 Q13 ↔ 사후 Q2), 여기서 이 발표의 가장 강한 데이터 스토리가 나온다(아래 §1-B).

**갭 3. 약점 데이터의 처리 방침이 없다.** WTP(0원 32%), UI 직관성(Top-2 61%), n=31 표본 한계는 숨기면 Q&A에서 잡히고, 먼저 말하면 신뢰 가산점이 된다. 아래 기획은 셋 다 발표자가 먼저 꺼내는 구조다.

**데이터의 한계 (발표 전 반드시 인지):**

- 사전·사후 응답자는 동일 집단이 아니다(paired 비교 불가). Before/After 비교는 "경향" 수준으로만 말한다.
- 표본 편향: 사전 20대 85%, 사후 20대 87%. 주관식에 지인 응답 흔적이 있다("수아꽁 화이팅"). "20대 중심 초기 타깃 검증"으로 스스로 한정하면 방어된다.
- 신뢰 요소 비교(사전 Q14 ↔ 사후 Q1)는 선택지 구성이 다르다: 사전에 있던 '사용자 후기'(60%)가 사후 설문·제품에는 없다. 비교는 공통 5개 항목으로 한정하고 각주로 명시한다. 이 '후기 부재'는 오히려 로드맵 근거로 회수한다(§3 신규 슬라이드 대본).
- 사용 의향 88%는 intent이지 behavior가 아니다. "그래서 실제로 쓰게 해봤다"로 사후 설문이 이 한계를 받아치는 구조로 말한다.

# 1. 스토리라인 및 핵심 메시지 전략

## 1-A. 설문이 뒷받침하는 논리 구조

| 발표 논리 | 근거 설문 데이터 | 배치 |
|---|---|---|
| 문제는 실재한다 (문제 정의) | 혼잡으로 만족도 하락 경험 86% · 정보 불편 1위 "신뢰도 판단 어려움" 52%, 2위 "정보 흩어짐" 49% | 7p |
| 수요는 말뿐이 아니다 (상충 검증) | 한적함을 위해 불편을 1개 이상 감수 95% (이동 1시간 추가 51%) | 7p |
| 서비스 방향이 맞다 (수요 검증) | 조건 기반 소도시 추천 사용 의향 Top-2 88% | 7p |
| 기능 우선순위가 맞다 (기능 타당성) | 대안 일정 80% > 이동시간 제시 76% > 추천 이유 71% > 축제 68% (의향 상승 Top-2) — Lovv 핵심 기능 4개와 1:1 대응 | 7p + 신규 슬라이드 |
| 제품이 신뢰를 만들었다 (제품 검증) | 신뢰 요소 역전: 지도·경로 검증 35%→58%(1위), 실제 일정 예시 18%→55% / 만족 77%·재이용 71%·추천 의향 77% | 신규 슬라이드 |
| BM 설계가 데이터와 정렬된다 (사업 타당성) | 유료 지불 의향 68%이나 대부분 1,000원 이하 → B2C 과금 유보 / "돈 낼 기능" 1위 예약 연결 71% → B2B 딥링크 모델 | 신규 슬라이드 → 22p |

## 1-B. 이 발표의 킬러 데이터 — "신뢰의 이유가 바뀌었다"

사용 전, 사람들은 AI 추천을 믿는 조건으로 '최근 업데이트 표시'(64%)를 1위로 꼽았다. '지도·교통 검증'은 35%, '실제 일정 예시'는 18%로 하위권이었다. 그런데 실제로 써본 뒤에는 신뢰에 도움이 된 요소 1위가 '지도·경로 검증'(58%), 2위가 '실제 일정 예시'(55%)로 완전히 역전됐다.

이 역전이 중요한 이유: **사용자가 써보기 전에는 상상하지 못한 가치를 제품이 만들어냈다는 증거**이고, Part 2(이창우 파트)가 설명한 검증 파이프라인·결정론적 설계가 기술 자랑이 아니라 실제 사용자 신뢰로 전환됐음을 보여주는 유일한 데이터다. "발견·납득·실행" 3단어 척추 구조에서 정확히 **'납득'의 사후 검증**에 해당하며, Part 2와 Part 3를 잇는 다리가 된다.

## 1-C. 전체 기승전결 (기존 35장 구조 유지 + 신규 1장)

| 단계 | 슬라이드 | 서사 | 설문의 역할 |
|---|---|---|---|
| 기 | 1~6p | 다들 한적한 곳을 원하는데 늘 대도시로 간다. 오버투어리즘 + 시장 데이터 | — |
| 승 | 7p | "외부 통계가 아니라 우리가 직접 물었다" — 문제·수요·기능 우선순위 확정 | 사용 전 설문 (기대) |
| 전 | 8~18p | 그래서 이렇게 만들었다 — 차별점·아키텍처·시연 | 7p의 기능 우선순위가 12p 핵심 컨셉의 근거가 됨 |
| 전′ | 20~21p + **신규 21.5p** | 데이터·트러블슈팅, 그리고 **"실제로 써보게 했더니"** | 사용 후 설문 (확인) — 신뢰 역전 + 만족·재이용 |
| 결 | 22~25p | 그래서 돈은 이렇게 벌고, 여기까지 왔다 | WTP·유료 기능 데이터가 BM 논리를 지탱 |

신규 슬라이드 위치는 **21p(트러블슈팅)와 22p(BM) 사이**. 이유: ① Part 2의 '납득'을 받아 회수하는 다리 역할 ② WTP 데이터가 바로 다음 장 BM의 "B2C 과금 유보" 결정의 근거로 직결 ③ 19p 대본의 3부 예고("데이터 → 돈 → 확장")를 "데이터 → **사용자 검증** → 돈 → 확장"으로 한 단어만 고치면 목차 정합성 유지.

시간 조정: 신규 슬라이드 약 70초 추가. Part 3가 6분→7분이 되므로 20p에서 10초, 24p에서 10초 줄이고, Part 1의 3p 목차를 15초로 끊는다. 21p(트러블슈팅)는 기존 대본 지침대로 줄이지 않는다.

# 2. 슬라이드별 화면 구성안

## 2-1. [7p 확정] 헤드라인: **"84명에게 물었다 — 문제는 취향이 아니라 도구다"**

기존 placeholder("팀 설문 결과 요약 통계/그래프")를 아래로 교체. 좌→우 3블록, 읽는 순서가 곧 논리 순서(불편 실재 → 감수 의향 → 사용 의향).

|---|---|---|

## 2-2. [신규 21.5p] 헤드라인: **"써보기 전과 후, 신뢰의 이유가 바뀌었다"**

섹션 번호는 `10 · 사용자 검증`으로 확정하고 이후 섹션을 민다: BM·B2G 거버넌스(22~23p) `10 → 11`, 협업·확장(24p) `11 → 12`. 3p 목차의 PART 03 항목도 `9．활용 데이터 · 트러블슈팅 / 10．사용자 검증 / 11．비즈니스 모델 / 12．협업 · 확장`으로 갱신 필요.

| 영역 | 시각화 | 데이터 |
|---|---|---|
| 좌 (슬라이드의 60%) — 신뢰 요소 Before/After | **덤벨 차트(슬로프 차트)** 5개 항목: 사용 전(회색 점)→사용 후(브랜드 컬러 점) 화살표 | 지도·경로 검증 35→58%(↑1위) · 실제 일정 예시 18→55%(↑) · 추천 이유 60→55%(유지) · 최근 업데이트 64→23%(↓) · 출처 제시 54→13%(↓) |
| 우상 — 결과 지표 | 카드 3개 (숫자 크게) | 전반 만족 77% · 재이용 의향 71% · 지인 추천 의향 77% (Top-2) |
| 우하 — 과금 현실 | 텍스트 카드 1개 | "지불 의향 68% — 단, 대부분 1,000원 이하 / 돈 낼 기능 1위: 예약까지 연결 71%" → 다음 장 BM 예고 |

각주: `실사용자 N=31 · ±17.6%p · 신뢰 요소는 사전·사후 공통 5개 항목 비교('사용자 후기'는 사후 미포함)`. 덤벨 차트에서 상승 2개만 컬러, 하락 2개는 회색 — 시선을 역전 스토리에 고정.

## 2-3. [22p 보강] 비즈니스 모델

화면 수정 최소화: B2C 블록에 배지 2개만 추가 — `지불 의향 68% · 1,000원 이하` / `돈 낼 기능 1위: 예약 연결 71%`. B2B 블록 화살표 옆에 "사용자 지불 의향 1위 기능 = 예약 연결" 한 줄. 대본이 주로 바뀐다(§3).

## 2-7. [24p 보강] 협업·확장 슬라이드

오른쪽 '기능 확장' 항목(현재 `예산 역산`)에 **`사용자 후기·리뷰`**를 추가한다. 근거: 사용 전 설문에서 AI 추천 신뢰 요건으로 '사용자 후기' 60%(공동 2위), 사용 후 주관식에서도 후기 부재 지적 2건. 신규 21.5p 대본에서 "다음 로드맵의 1순위"라고 선언하므로, 확장 슬라이드에 해당 항목이 없으면 말과 화면이 어긋난다.

# 3. 슬라이드별 발표 대본

기존 대본과의 관계: 1p 오프닝·25p 클로징은 기존 대본을 그대로 유지한다(수정 없음). 7p는 placeholder 확정판으로 전면 교체, 21.5p는 신규, 19p·22p는 부분 수정, 24p는 한 문장 보강.

## [7p / 시장 수요 — 설문으로 확인한 수요] (약 60초, 전면 교체)

> 방금 건 외부 통계였고요, 저희가 직접 물어봤습니다. 84명이 응답했고, 오차범위는 ±11%포인트입니다. 다만 이런 설문은 묻는 방식이 결과를 좌우합니다. "한적한 곳 좋아하세요?"라고 물으면 누구나 "네"라고 하거든요. 그래서 저희는 뭔가를 포기해야 하는 질문으로 바꿔 물었습니다. 화면 왼쪽부터 보시면 — 첫째, 불편은 실재합니다. 86%가 유명 관광지의 혼잡 때문에 여행 만족도가 실제로 떨어진 경험이 있다고 답했습니다. 그리고 정보를 찾을 때 가장 큰 불편 1위가 '신뢰도 판단이 어렵다' 52%, 2위가 '정보가 흩어져 있다' 49%였습니다. 둘째, 이 수요는 말뿐이 아닙니다. 95%가 한적한 소도시를 위해서라면 이동시간 1시간 추가 같은 불편을 최소 하나는 감수하겠다고 답했습니다. 셋째, 그래서 조건만 넣으면 소도시와 일정을 추천해주는 서비스를 쓰겠냐고 물었더니 — 88%가 쓰고 싶다고 답했습니다. 마지막으로 아래 네 개의 막대를 봐주세요. 어떤 조건이 있으면 소도시 여행 의향이 올라가는지 물었더니, 대안 일정 80%, 이동시간 제시 76%, 추천 이유 71%, 축제 정보 68%. 저희 서비스의 기능 우선순위는 감이 아니라 이 네 개의 숫자에서 나왔습니다.

스피치 팁: "86%"는 이 슬라이드에서 처음 등장하는 핵심 숫자이므로 말한 뒤 반 박자 쉬어 소화 시간을 줄 것. 마지막 문장 "감이 아니라 이 네 개의 숫자"가 이 슬라이드의 한 줄 — 12p 핵심 컨셉으로 가는 복선이므로 또렷하게. 숫자를 다 읽었으므로 하단 띠는 손으로 쓸어가듯 한 번만 짚는다.

## [신규 21.5p / 사용자 검증 — 써보기 전과 후, 신뢰의 이유가 바뀌었다] (약 70초)

> 1부에서 88%가 "쓰고 싶다"고 답했다고 말씀드렸는데요, 의향은 의향일 뿐이죠. 그래서 실제로 써보게 했습니다. 실사용자 31명의 사후 설문입니다. 표본이 작아서 오차범위가 ±18%포인트라는 점은 먼저 말씀드리고요, 그래도 방향은 분명했습니다. 전반 만족 77%, 재이용 의향 71%, 지인 추천 의향 77%. 그런데 저희가 정말 보여드리고 싶은 건 왼쪽 차트입니다. 쓰기 전에 "AI 추천을 믿으려면 뭐가 필요하세요?"라고 물었을 때 1위는 '최신 업데이트 표시' 64%였습니다. '지도·경로 검증'은 35%, '실제 일정 예시'는 18%로 하위권이었어요. 그런데 써본 뒤에 "실제로 뭐가 신뢰에 도움이 됐나요?"라고 물으니 순위가 뒤집혔습니다. 1위가 지도·경로 검증 58%, 2위가 실제 일정 예시 55%. 사용자들은 써보기 전엔 이 가치를 상상하지 못했다가, 써본 뒤에야 "아, 이래서 믿을 수 있구나"를 경험한 겁니다. 방금 2부에서 창우 님이 보여드린 검증 파이프라인 — 그 기술이 실제 사용자의 신뢰로 전환됐다는 증거입니다. 한 가지 숙제도 얻었습니다. 쓰기 전 60%가 원했던 '사용자 후기'는 아직 저희 제품에 없고, 사후 주관식에서도 지적됐습니다. 다음 로드맵의 1순위입니다.

스피치 팁: "의향은 의향일 뿐이죠"는 평가관이 던질 반박을 발표자가 먼저 던지는 문장 — 가볍게, 그러나 분명히. 덤벨 차트는 상승 화살표 두 개만 손으로 짚는다. 마지막 '후기' 문장은 약점 고백이 아니라 로드맵 예고 톤으로 — 24p 확장 슬라이드에서 화면과 대본으로 회수한다(§2-7, §3 24p).

## [22p 비즈니스 모델 — B2C 단락 교체] (해당 단락만)

> 먼저 B2C. 사용자에게 바로 과금하는 건 서두르지 않습니다 — 그리고 이건 방금 보신 데이터로 내린 결정입니다. 사후 설문에서 68%가 지불 의향이 있다고 했지만, 금액은 대부분 1,000원 이하였습니다. 지금 B2C 유료화를 밀어붙일 단계가 아니라는 거죠. 대신 눈여겨본 건 "어떤 기능에 돈을 내겠냐"는 질문의 1위가 '예약까지 연결', 71%였다는 점입니다. 사용자가 돈 낼 가치를 느끼는 지점이 예약 연결이라면, 과금 주체를 사용자가 아니라 예약 플랫폼으로 옮기면 됩니다. 그래서 실제 매출의 중심은 B2B입니다. (이하 기존 대본 유지)

이 수정의 효과: 기존 대본은 "WTP 문항으로 검증하는 단계"라고 미래형으로 말하는데, 이제 검증이 끝났으므로 **결과를 근거로 BM을 정당화하는 과거형**으로 바뀐다. "낮은 WTP"라는 약점 데이터가 B2B 중심 설계의 근거로 반전되는 지점 — 이 발표에서 기획 역량을 보여주는 최고 순간이므로 "과금 주체를 옮기면 됩니다" 문장은 천천히.

## 4-2. 예상 질문과 답변 라인

- **"n=31이면 통계적으로 의미 있나?"** → "맞습니다, ±18%포인트입니다. 그래서 절대 수치보다 사전·사후의 방향 역전과, 만족·재이용·추천이 일관되게 70%대라는 패턴을 근거로 삼았습니다. 정식 베타에서 n을 늘리는 게 다음 단계입니다."
- **"응답자가 지인 아닌가?"** → "초기 표본은 20대·주변 네트워크 중심이 맞습니다. 다만 지인 표본은 보통 만족도를 부풀리는데, 저희 설문에는 UI 직관성 61%, 지불 의향 1,000원 이하 같은 불리한 답과 신랄한 주관식 지적이 그대로 들어 있습니다. 응답이 관대하게 왜곡됐다면 나오기 어려운 결과입니다."
- **"재이용 71%면 29%는 아니라는 건데?"** → "'그렇지 않다'는 10%고 19%는 보통입니다. 그 10%의 주관식을 보면 UI 복잡성과 버그 지적이 대부분이라, 컨셉 거부가 아니라 완성도 문제로 읽고 있습니다. 개선 우선순위에 반영했습니다."
- **"UI 직관성 61%는 낮지 않나?"** → 숨기지 말고 시인: "네, 저희 지표 중 가장 낮습니다. 주관식에서 텍스트 과다·스크롤 문제가 반복 지적됐고, 정보 밀도를 줄이는 리디자인이 확장 로드맵에 있습니다." (24p에서 회수 가능)
- **"사전 88%와 사후 만족 77%, 떨어진 것 아닌가?"** → "다른 질문, 다른 집단입니다. 88%는 '써보고 싶다'는 기대고, 77%는 실제 사용 후 만족입니다. 기대 지표와 경험 지표가 모두 70~80%대로 정합한다는 게 저희 해석입니다."
