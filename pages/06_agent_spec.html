로브 (Lovv) Agent 명세서

문서 버전: v0.1

문서 상태: 기획 단계 (Planning)

기준 문서: docs/01_requirements/01_requirements.md v1.5

1. 문서 개요

1.1 목적

본 문서는 로브 추천 Agent의 역할, 단계별 입력·출력, 도구 사용, 폴백, 검증 기준을 정의한다. Agent는 단일 LLM 호출이 아니라 조건 분류, 검색, 선정, 일정 구성, 설명 생성으로 나뉜 멀티스텝 파이프라인으로 동작한다.

2. Agent 목표

목표설명
조건 구조화자연어와 지도 진입 조건을 추천 조건 객체로 변환
근거 기반 추천RAG 검색 결과와 정적 데이터를 근거로 소도시 1곳 선정
개인화 반영온보딩 선호와 좋아요/싫어요 피드백 반영
일정 구성일정 유형에 맞는 일별 세부 일정 생성
설명 가능성추천 이유와 일정 흐름 이유를 자연어로 제공
안전한 폴백데이터 부족, API 실패, 조건 충돌 시 대체 안내

3. Agent 입력

입력출처설명
naturalLanguageQuery챗봇사용자 자연어 요청
entryTypeUIchatbot 또는 map_marker
countryUI/챗봇KR 또는 JP
travelMonthUI/챗봇1~12월
destinationId지도마커 클릭 시 고정 목적지
tripTypeUI/챗봇daytrip~5d4n
includeFestivalsUI/챗봇축제·행사 포함 여부
onboardingProfile저장 데이터기본 선호 테마
feedbackHistory저장 데이터좋아요/싫어요 이력

4. Agent 출력

출력설명
selectedDestination선정된 소도시 1곳
itinerary일정 유형에 맞춘 일별 세부 일정
recommendationReasons사용자 조건, 계절, 혼잡도, 기상 경향 기반 추천 이유
itineraryFlowReason일정 순서와 동선 흐름 이유
alternativeItinerary기상 악화 시 실내 중심 대체 일정
externalLinks지도, 숙소 검색, 현지 탐색 링크
confidence추천 신뢰도와 데이터 결측 안내

5. 파이프라인

단계Agent 역할입력출력
1Condition Parser자연어, UI 조건구조화된 여행 조건
2Entry Classifier구조화 조건map_marker, weather_first, theme_first 등
3Retriever국가, 월, 테마후보 소도시 목록
4Ranker후보, 혼잡도, 기상 경향, 피드백상위 1곳
5Itinerary Planner선정 소도시, 일정 유형, 축제 포함 여부일별 일정
6Explanation Writer조건, 점수, 일정추천 이유와 일정 흐름 이유
7Output Validator최종 결과누락·환각·정책 위반 검증

6. 단계별 명세

6.1 Condition Parser

항목내용
책임자연어에서 국가, 월, 테마, 동행, 일정 유형을 추출
필수 출력country, travelMonth, tripType
폴백필수 값 누락 시 추가 질문 생성

6.2 Retriever

항목내용
검색 대상목적지 기본 정보, 테마 태그, 계절 적합도, 월별 기상 경향, 축제 정보
제약한국 요청에는 한국 데이터만, 일본 요청에는 일본 데이터만 사용
출력후보 소도시 N개와 근거 데이터

6.3 Ranker

점수 요소방향
사용자 조건 매칭높을수록 가점
계절 적합도높을수록 가점
혼잡도낮을수록 가점
월별 기상 악화 경향높을수록 감점
좋아요 테마 태그가점
싫어요 소도시감점

6.4 Itinerary Planner

일정 유형구성 기준
당일치기핵심 명소 2~3개, 이동 부담 최소화
1박2일대표 명소, 식사, 휴식, 숙소 검색 링크
2박3일 이상일자별 테마 분리, 과도한 이동 방지

6.5 Explanation Writer

추천 이유는 아래 항목 중 2개 이상을 포함한다.

  • 사용자 조건과의 매칭
  • 계절 적합도
  • 혼잡 회피 근거
  • 월별 기상 경향
  • 일정 흐름과 이동 효율

7. 도구 및 데이터 사용

도구/데이터사용 목적
Destination Search후보 검색
Weather Monthly Trends추천 스코어링과 대체 일정 분기
Festival Search축제 포함 일정 구성
Map Link Builder지도 링크 생성
Stay Link Builder숙소 검색 링크 생성
WeatherAPI Proxy현재 날씨 표시용 데이터 조회

8. 금지 사항

  • 대화 로그 전문을 장기 저장하지 않는다.
  • WeatherAPI 실시간 예보를 추천 후보 스코어링 기준으로 사용하지 않는다.
  • 출처 없는 장소 정보를 확정 사실처럼 제시하지 않는다.
  • 한국 요청에 일본 목적지를 섞거나, 일본 요청에 한국 목적지를 섞지 않는다.
  • 숙소를 직접 추천하지 않고 검색 링크만 제공한다.

9. 품질 검증 기준

검증 항목기준
조건 충족국가, 월, 일정 유형이 결과에 반영됨
근거성추천 이유가 데이터 근거와 연결됨
단일 목적지최종 추천은 소도시 1곳 중심
설명 가능성추천 이유와 일정 흐름 이유가 모두 존재
안전성결측 데이터와 API 실패를 안내
