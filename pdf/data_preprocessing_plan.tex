\section*{1. 문서 개요}
\addcontentsline{toc}{section}{1. 문서 개요}


\DocNeedspace{24\baselineskip}
\subsection*{1.1 목적}
\addcontentsline{toc}{subsection}{1.1 목적}


\DocNeedspace{3\baselineskip}
본 문서는 로브(Lovv)의 한국·일본 지역 추천 데이터가 수집된 이후 AWS S3에 원본을 적재하고 AWS Lambda로 전처리한 뒤 NoSQL 저장소인 Amazon DynamoDB에 서비스용 데이터를 적재하기 위한 원본 정제, 표준화, 품질 검증, 파생 데이터 생성, 적재 기준을 정의한다.


\DocNeedspace{3\baselineskip}
수집 계획서는 City, Attraction, Festival 데이터를 먼저 폭넓게 취득하고 이후 검증·보정하는 방향을 가진다.\newline 본 전처리 계획서는 그 다음 단계에서 원본 데이터의 표현 차이와 누락·중복·최신성 문제를 정리하여 서비스 사용 가능한 데이터셋으로 변환하는 데 초점을 둔다.


\DocNeedspace{24\baselineskip}
\subsection*{1.2 전처리 원칙}
\addcontentsline{toc}{subsection}{1.2 전처리 원칙}


\begin{itemize}
\item \textbf{Raw 보존}: API 응답, HTML 추출 결과, 수동 검수 입력값은 JSON 문서로 저장한 뒤 S3 Raw 영역에 보존한다.
\item \textbf{Lambda 기반 배치 처리}: S3 Raw Bucket에 일정 기간 누적된 JSON 원본을 Lambda가 배치 단위로 읽어 전처리한다.
\item \textbf{DynamoDB 적재}: 서비스 조회용 데이터는 Raw와 분리하여 DynamoDB 테이블에 정규화 문서 형태로 적재한다.
\item \textbf{국가별 차이 흡수}: 한국 시·군·구와 일본 시·정·촌·구의 명칭, 행정구역, 날짜, 주소 표현 차이를 공통 스키마로 매핑한다.
\item \textbf{상태 기반 처리}: 모든 필드는 \texttt{collected}, \texttt{needs\_\allowbreak{}review}, \texttt{missing}, \texttt{blocked} 상태를 유지하고 전처리 결과에도 상태를 전파한다.
\item \textbf{출처 추적}: 정규화 값은 \texttt{source\_\allowbreak{}name}, \texttt{source\_\allowbreak{}url}, \texttt{collected\_\allowbreak{}at}, \texttt{verified\_\allowbreak{}at}, \texttt{data\_\allowbreak{}confidence}와 연결한다.
\item \textbf{추천 사용성 우선}: 설명, 태그, 계절성, 좌표, 운영정보는 추천·일정 생성에 바로 사용할 수 있는 형태로 가공한다.
\end{itemize}

\section*{11. 법적·운영 유의사항}
\addcontentsline{toc}{section}{11. 법적·운영 유의사항}


\begin{itemize}
\item Wikipedia 기반 설명은 출처와 원문 링크를 남기되 서비스 본문에는 내부 요약문을 사용한다.
\item TourAPI, JNTO, JTA, 지자체 관광 사이트의 이용 조건과 출처 표기 조건을 보존한다.
\item 사진은 사용 가능 조건이 확인된 경우에만 서비스 노출 후보로 둔다.
\item 상업 플랫폼의 숙박·맛집 데이터는 직접 저장하지 않고 검색 딥링크 또는 사용자 이동 경로로 처리한다.
\item 운영시간, 입장료, 축제 기간은 변경 가능성이 높으므로 서비스 응답에는 확인일 기준 문구를 연결한다.
\end{itemize}
