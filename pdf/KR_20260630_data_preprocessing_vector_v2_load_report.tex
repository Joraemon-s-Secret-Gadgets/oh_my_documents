\ReportSubsection{1.3 S3 전처리 후 적재 흐름}

\DocNeedspace{3\baselineskip}
전처리 후 S3 적재는 raw 보존, Lambda 전처리, 결과 prefix 분리 순서로 진행된다. 입력 날짜는 실행 기준일을 \texttt{YYYYMMDD} 형식으로 적은 값이며, 예를 들어 2026-06-29 실행이면 입력 날짜는 \texttt{20260629}다. S3 prefix는 \texttt{raw/\allowbreak{}KR/\allowbreak{}details/\allowbreak{}YYYYMMDD/\allowbreak{}}와 \texttt{processed/\allowbreak{}KR/\allowbreak{}details/\allowbreak{}YYYYMMDD/\allowbreak{}} 형식으로 구성한다.

\DocNeedspace{5\baselineskip}
\begin{itemize}[leftmargin=1.35em, itemsep=5pt, topsep=3pt]
\item \textbf{Raw 보존:} 원천 JSON은 \texttt{raw/\allowbreak{}KR/\allowbreak{}details/\allowbreak{}YYYYMMDD/\allowbreak{}}에 저장한다. 이번 실행 예시는 \texttt{raw/\allowbreak{}KR/\allowbreak{}details/\allowbreak{}20260629/\allowbreak{}}이며, 수집 원본을 날짜별 prefix에 남겨 동일 기준으로 재처리할 수 있게 한다.
\item \textbf{전처리 실행:} \texttt{kr-\allowbreak{}pipeline-\allowbreak{}transform}이 raw 객체를 순회하며 스키마 검증, 값 정리, 좌표와 날짜 정규화, 품질 분류를 수행한다. 이번 실행에서는 raw 객체 240개가 처리됐다.
\item \textbf{정상 산출:} 정규화 결과는 \texttt{processed/\allowbreak{}KR/\allowbreak{}details/\allowbreak{}YYYYMMDD/\allowbreak{}passed/\allowbreak{}}에 저장한다. 이번 실행 예시는 \texttt{processed/\allowbreak{}KR/\allowbreak{}details/\allowbreak{}20260629/\allowbreak{}passed/\allowbreak{}}이며, \texttt{passed/\allowbreak{}} 산출 item은 8,010건이다.
\item \textbf{검토 산출:} 자동 확정이 어려운 품질 이슈와 보강 필요 항목은 \texttt{processed/\allowbreak{}KR/\allowbreak{}details/\allowbreak{}YYYYMMDD/\allowbreak{}review/\allowbreak{}}에 분리한다. 이번 실행의 Review record는 1,202건이다.
\DocNeedspace{5\baselineskip}
\item \textbf{실패와 품질:} 처리 실패 원인과 품질 리포트는 각각 \texttt{failed/\allowbreak{}}, \texttt{quality/\allowbreak{}} 하위 prefix에 남긴다. 이번 실행의 실패 invocation은 0건이다.
\end{itemize}

\DocNeedspace{8\baselineskip}
추가 확인이 필요하면 02 프로젝트 담당 스레드에 아래 프롬프트를 전달한다.

\begin{quote}
\footnotesize
\textbf{확인 요청 프롬프트}\quad
\texttt{02\_\allowbreak{}lovv\_\allowbreak{}data\_\allowbreak{}collect}의 KR 입력 날짜 \texttt{YYYYMMDD} 전처리 실행 기준으로, \texttt{kr-\allowbreak{}pipeline-\allowbreak{}transform}이 \texttt{raw/\allowbreak{}KR/\allowbreak{}details/\allowbreak{}YYYYMMDD/\allowbreak{}}를 읽어 \texttt{processed/\allowbreak{}KR/\allowbreak{}details/\allowbreak{}YYYYMMDD/\allowbreak{}passed/\allowbreak{}}, \texttt{review/\allowbreak{}}, \texttt{failed/\allowbreak{}}, \texttt{quality/\allowbreak{}}에 쓰는 정확한 S3 write 단계, manifest 생성 주체, 각 prefix별 JSON 스키마와 count 산정 기준을 확인해 주세요. 예시는 \texttt{YYYYMMDD=20260629}이며, 현재 보고서에는 raw 240개, \texttt{passed/\allowbreak{}} 8,010 item, Review record 1,202건, S3 URL 치환 6,975건으로 반영하려 합니다.
\end{quote}
