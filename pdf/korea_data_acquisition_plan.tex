\section*{1. 목적}
\addcontentsline{toc}{section}{1. 목적}


\DocNeedspace{3\baselineskip}
본 문서는 여행 추천 Multi-Agent 서비스에서 한국 소도시 추천에 필요한 도시·관광지·축제 데이터를 한 번에 취득하기 위한 범위, 데이터 소스, 검증 방식, 저장 구조를 정의한다.


\DocNeedspace{3\baselineskip}
이번 초안의 핵심은 다음과 같다.


\begin{itemize}
\item City, Attraction, Festival의 정의된 항목을 모두 수집 대상으로 둔다.
\item 도의 간략 정보와 산하 도시 목록을 먼저 취득하고, 산하 도시 목록에 포함된 City만 Wikipedia 문서 크롤링 대상으로 둔다.
\item 한국관광공사 TourAPI와 대한민국구석구석 기반 데이터를 중심으로 관광지·축제 정보를 취득한다.
\item 취득 결과는 JSON 문서로 저장한 뒤 S3 Raw Bucket에 적재한다.
\item 운영시간, 운영기간, 입장료, 사진처럼 출처별 표현 차이가 큰 항목도 최초 취득 범위에 포함한다.
\item 자동 수집, 공식 사이트 확인, Web Search Worker, 수동 검수를 하나의 취득 파이프라인으로 묶어 처리한다.
\item 추천 DB는 \texttt{City 1:\allowbreak{}N Attraction}, \texttt{City 1:\allowbreak{}N Festival} 관계를 기준으로 설계한다.
\end{itemize}

\subsection*{5.3 데이터 정규화 규칙}

\begin{itemize}
\item 도시명은 \texttt{city\_\allowbreak{}name\_\allowbreak{}ko}, \texttt{city\_\allowbreak{}name\_\allowbreak{}en}을 분리 저장한다.
\item 시·군·구는 행정구역 코드와 내부 \texttt{city\_\allowbreak{}id}를 매핑한다.
\item 도 간략 정보와 산하 도시 목록의 Wikipedia API 응답 원본, 산하 도시 Wikipedia HTML 크롤링 원본을 Raw 데이터로 보존한다.
\item 관광지와 축제는 반드시 \texttt{city\_\allowbreak{}id}를 가진다.
\item TourAPI의 \texttt{contentid}, \texttt{contenttypeid}, \texttt{areacode}, \texttt{sigungucode}, \texttt{cat1}, \texttt{cat2}, \texttt{cat3}를 원본 참조 필드로 저장한다.
\item 기간 정보는 원문 문자열(\texttt{period\_\allowbreak{}text})과 정규화 값(\texttt{start\_\allowbreak{}date}, \texttt{end\_\allowbreak{}date}, \texttt{month})을 함께 저장한다.
\item 운영시간과 입장료는 최신성이 낮을 수 있으므로 \texttt{data\_\allowbreak{}confidence}를 별도 기록한다.
\item 외부 설명문은 원문 복제가 아니라 내부 요약문으로 재작성한다.
\end{itemize}

\begin{scriptsize}
\ttfamily

\setlength{\parskip}{1pt}
\textcolor{LovvGreen}{\textbf{text}}\\[0.2em]

↓\\

수집 원본은 DB에 바로 쓰지 않고 JSON 파일로 저장한 뒤 S3 Raw Bucket에 적재한다.\newline S3 객체는 국가, 출처, 엔티티 유형, 수집일 기준 Prefix로 구분한다.\newline 일정 기간 누적된 Raw Prefix는 Lambda 배치 전처리의 입력으로 재사용하며, 전처리 결과만 DynamoDB에 정규화 Item으로 적재한다.

\section*{9. 법적·운영 제약}
\addcontentsline{toc}{section}{9. 법적·운영 제약}


\begin{itemize}
\item Wikipedia API 및 크롤링 기반 설명은 라이선스 조건에 따라 출처와 원문 링크를 기록한다.
\item TourAPI와 공공데이터포털 데이터는 공공누리 유형, API 이용 조건, 출처 표기 조건을 확인한다.
\item 지자체 관광 페이지는 이용약관과 크롤링 허용 범위를 확인한다.
\item 사진은 저작권 위험이 크므로 자동 수집 값이 있더라도 사용 가능 조건을 확인하고, 불명확하면 \texttt{needs\_\allowbreak{}review} 또는 \texttt{blocked}로 관리한다.
\item 운영시간과 입장료는 변경 가능성이 높으므로 "확인일 기준" 문구를 서비스에 표시한다.
\end{itemize}
