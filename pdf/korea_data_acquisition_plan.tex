\DocNeedspace{8\baselineskip}
\subsection*{1.1 목적}
\addcontentsline{toc}{subsection}{1.1 목적}

\DocNeedspace{3\baselineskip}
본 문서는 여행 추천 Multi-Agent 서비스에서 한국 소도시 추천에 필요한 도시·관광지·축제 데이터를 한 번에 취득하기 위한 범위, 데이터 소스, 검증 방식, 저장 구조를 정의한다.


\DocNeedspace{3\baselineskip}
이번 초안의 핵심은 다음과 같다.


\begin{itemize}
\item City, Attraction, Festival의 정의된 항목을 모두 수집 대상으로 둔다.
\item 한국관광공사 TourAPI와 대한민국구석구석 기반 데이터를 중심으로 관광지·축제 정보를 취득한다.
\item 운영시간, 운영기간, 입장료, 사진처럼 출처별 표현 차이가 큰 항목도 최초 취득 범위에 포함한다.
\item 자동 수집, 공식 사이트 확인, Web Search Worker, 수동 검수를 하나의 취득 파이프라인으로 묶어 처리한다.
\item 한국 데이터는 City 기준 JSON 파일 안에 Attraction과 Festival 배열을 포함하는 구조로 저장한다.
\end{itemize}

\subsection*{5.3 데이터 정규화 규칙}
\addcontentsline{toc}{subsection}{5.3 데이터 정규화 규칙}


\begin{itemize}
\item 도시명은 \texttt{city\_name\_ko}, \texttt{city\_name\_en}을 분리 저장한다.
\item 시·군·구는 행정구역 코드와 내부 \texttt{city\_id}를 매핑한다.
\item 관광지와 축제는 반드시 \texttt{city\_id}를 가진다.
\item TourAPI의 \texttt{contentid}, \texttt{contenttypeid}, \texttt{areacode}, \texttt{sigungucode}, \texttt{cat1}, \texttt{cat2}, \texttt{cat3}를 원본 참조 필드로 저장한다.
\item 기간 정보는 원문 문자열(\texttt{period\_text})과 정규화 값(\texttt{start\_date}, \texttt{end\_date}, \texttt{month})을 함께 저장한다.
\item 운영시간과 입장료는 최신성이 낮을 수 있으므로 \texttt{data\_confidence}를 별도 기록한다.
\item 외부 설명문은 원문 복제가 아니라 내부 요약문으로 재작성한다.
\end{itemize}

정의된 모든 필드가 JSON 파일에 들어가도록 시도하는 것이 목표다. 운영시간·입장료·사진도 최초 수집 대상에 포함하며, 자동 수집 실패 시 \texttt{missing} 또는 \texttt{needs\_review} 상태로 남기고 같은 파이프라인 안에서 공식 확인 또는 수동 검수로 채운다.


\DocNeedspace{3\baselineskip}
예시:


\begin{verbatim}
사용자 질문: 순천만국가정원 운영시간 알려줘

1. JSON 파일에서 순천만국가정원 Attraction 조회
2. opening_hours가 없거나 오래된 값이면 Web Search Worker 실행
3. 공식 사이트 또는 공식 관광 페이지 확인
4. 운영시간과 확인 출처를 함께 반환
\end{verbatim}

\subsection*{7.3 저장 원칙}

\begin{itemize}
\item City, Attraction, Festival의 관계는 JSON 내부의 중첩 구조로 표현한다.

\end{itemize}

\section*{9. 법적·운영 제약}
\addcontentsline{toc}{section}{9. 법적·운영 제약}


\begin{itemize}
\item Wikipedia 기반 설명은 라이선스 조건에 따라 출처와 원문 링크를 기록한다.
\item TourAPI와 공공데이터포털 데이터는 공공누리 유형, API 이용 조건, 출처 표기 조건을 확인한다.
\item 지자체 관광 페이지는 이용약관과 크롤링 허용 범위를 확인한다.
\item 사진은 저작권 위험이 크므로 자동 수집 값이 있더라도 사용 가능 조건을 확인하고, 불명확하면 \texttt{needs\_review} 또는 \texttt{blocked}로 관리한다.
\item 운영시간과 입장료는 변경 가능성이 높으므로 "확인일 기준" 문구를 서비스에 표시한다.
\end{itemize}
