\DocNeedspace{8\baselineskip}
\subsection*{1.1 목적}
\addcontentsline{toc}{subsection}{1.1 목적}


\DocNeedspace{3\baselineskip}
본 문서는 로브(Lovv) 서비스의 아이디어 기획 단계에서 제품의 핵심 기능,\newline API 연동 요구사항, 비기능 요구사항을 정의한다.

\DocNeedspace{3\baselineskip}
개발 착수 전 이해관계자 간 요구사항에 대한 공통 이해를 확보하는 것을 목적으로 한다.


\DocNeedspace{8\baselineskip}
\subsection*{1.2 범위}
\addcontentsline{toc}{subsection}{1.2 범위}


\DocNeedspace{3\baselineskip}
본 문서는 다음 범위를 포함한다.


\begin{itemize}
\item 대화형 챗봇 인터페이스 및 기본·확장 테마 기반 소도시 집중 일정 추천 엔진
\item 국내 소도시 (강원·전남·경남 등) 및\newline 일본 소도시 (호쿠리쿠·기후·산인 등) 추천 --- 나라별 독립 트랙
\item 여행 시기(달/계절) 입력에 따른 계절 적합 소도시 및 축제·행사 연동
\item 메인 지도 기반 소도시 탐색, 국가·월·테마 필터, 지도 마커 기반 추천 진입점
\item 온보딩 선호 프로필, 추천 피드백, 저장 일정, 북마크, 방문 기록, 방문 소도시 별점 기반 개인화
\item 사용자 행동 데이터와 외부 검증 루트를 활용한 cold start 보완 및 개인화 랭킹
\item Production 단계의 사용자 계정 기반 개인화 데이터 누적 및 개인화 랭킹 1차 운영
\item 현재 위치 또는 직접 입력한 기준 위치와 사용자 지정 반경 기반 추천
\item 현재 여행 중 남은 예산·일정 변화에 따른 즉흥 일정 조정
\item 지역 문화 체험, 스키 액티비티 등 P1\textasciitilde{}P4 범위의 확장 테마 추천
\item 여행 시기(월)를 직접 입력해 월별 기상 경향이 좋은 소도시를 추천받는 챗봇 진입점
\item 별 관측, 치유·웰니스, 예산 역산 추천은 이번 프로젝트 대상에서 제외하고\newline 향후 단계에서 검토
\item RAG(검색증강) 기반 grounding 및 멀티스텝 에이전트 추천 파이프라인
\item Google Maps Platform, Kakao Maps, Yahoo Japan, WeatherAPI 연동
\item 지도 표시, 장소 상세 정보, 현재 날씨 표시, 숙소 검색 링크, 외부 탐색 링크 제공 기능
\end{itemize}

\subsection*{2.4 권한 원칙}

\begin{itemize}
\item 각 역할은 수행 목적에 필요한 최소 범위의 데이터와 기능만 사용한다.
\item 여행자에게 노출되는 공개 데이터와 내부 운영 지표, 추천 정책, API Key는 분리한다.
\item 관광 운영자와 데이터 제공 기관은 데이터를 제안·제공할 수 있으나 공개 반영은 서비스 관리자 승인 후 적용한다.
\item 사용자 취향 입력은 추천 목적에 한정해 처리하며, 대화 로그 전문은 저장하지 않는다. 일정·선호·피드백을 저장하는 경우 저장 범위, 보관 기간, 삭제 기준을 명확히 한다.
\item 배포 환경의 외부 API Key는 이 서비스를 개발·운영하는 백엔드/운영 개발자가 서버 환경 변수 또는 백엔드 프록시에서 관리하고 클라이언트 노출을 최소화한다.
\item 서비스 관리자는 API Key 값을 직접 관리하지 않고, 키 발급·교체·만료 대응이 필요할 때 개발·운영 담당자에게 요청한다.
\end{itemize}

\DocNeedspace{3\baselineskip}

\DocNeedspace{3\baselineskip}
로브(Lovv)는 오버투어리즘 회피를 목적으로 국내 및 일본의 비주류 소도시 1곳을 사용자의 취향과 여행 시기에 맞는 집중 일정으로 추천하는 대화형 챗봇 서비스다.

\DocNeedspace{3\baselineskip}

\DocNeedspace{3\baselineskip}
한국은 한국 소도시만, 일본은 일본 소도시만 추천하는 나라별 독립 트랙으로 운영된다.


\DocNeedspace{3\baselineskip}
서비스명 로브(Lovv)는 Local(소도시)과 Vibe(분위기·감성), 또는 Voyage(항해·여행)의 의미를 결합한 이름이다.

\DocNeedspace{3\baselineskip}
일반적인 Love와 달리 알파벳 v를 겹쳐 사용해 감성적인 여행 서비스이면서도 트렌디한 스타트업·테크 서비스의 인상을 전달한다.


\DocNeedspace{3\baselineskip}
추천 시 여행 시기(월)에 빛나는 소도시와 사용자가 포함을 선택한 축제·행사를 일정에 결합하며, 각 결과에는 Google Maps, Kakao Maps, Yahoo Japan, WeatherAPI의 4개 플랫폼 API가 역할 분담하여 지도·장소 상세·현재 날씨·링크 정보를 보완 제공한다.

\DocNeedspace{3\baselineskip}
지역 문화 체험, 스키 액티비티 같은 P1\textasciitilde{}P4 확장 테마와 월별 기상 경향을 기준으로 한 직접 질의형 추천도 같은 추천 엔진 안에서 처리한다.

\DocNeedspace{3\baselineskip}
별 관측, 치유·웰니스, 예산 역산 추천은 이번 프로젝트 대상에서 제외하고\newline 향후 단계에서 검토한다.

\section*{5. API 연동 요구사항}
\addcontentsline{toc}{section}{5. API 연동 요구사항}


\DocNeedspace{6\baselineskip}
각 플랫폼 API는 추천 결과에 필요한 지도, 장소 상세, 현지 탐색 링크, 현재 날씨 정보를 역할별로 보완 제공한다.
클라이언트 측 직접 호출이 어려운 REST API는 우선 딥링크로 대체하고, Production 단계에서는 백엔드 프록시를 통해 단계적으로 연동한다.

\DocNeedspace{8\baselineskip}
\begin{itemize}[leftmargin=2.0em, itemsep=0.45em]
\item \textbf{Google Maps Platform}: 전 지역 공통 지도, 장소 상세, 사진, 주변 명소 정보를 제공한다. Maps JavaScript API와 Places API는 클라이언트 직접 연동을 기본으로 한다.
\item \textbf{Kakao Maps}: 국내 소도시의 카카오 지도 표시와 주변 맛집·카페·숙박 검색을 담당한다. 지도 SDK는 직접 연동하고, Local REST API는 REST Key 설정 여부에 따라 선택 적용한다.
\item \textbf{Yahoo Japan}: 일본 여행, 숙박, 맛집 탐색은 Yahoo Travel, Jalan, Tabelog 등 현지 플랫폼 딥링크를 우선 제공한다. REST API 연동은 CORS 제약을 고려해 백엔드 프록시 적용 후 검토한다.
\item \textbf{WeatherAPI}: 목적지 상세 화면에 현재 기온, 날씨 상태, 풍속, 습도, 강수 확률을 표시한다. 추천 후보 스코어링에는 실시간 예보가 아니라 월별 기상 경향 데이터를 사용한다.
\end{itemize}

\subsection*{8.1 기술적 제약사항}
\addcontentsline{toc}{subsection}{8.1 기술적 제약사항}


\DocNeedspace{10\baselineskip}
\begin{itemize}[leftmargin=2.0em, itemsep=0.55em]
\item \textbf{Yahoo Japan Maps SDK 지원 중단}: Yahoo Japan Maps JavaScript SDK(Yjans)가 지원 중단되어 일본 소도시 지도는 Google Maps API로 대체 제공한다.
\item \textbf{Yahoo Japan REST API CORS}: Local Search 등 REST API는 브라우저 직접 호출에 CORS 제한이 있다. 향후 백엔드 프록시 서버를 통해 연동하고, App ID는 후속 적용을 위해 입력 항목만 준비한다.
\item \textbf{Kakao REST API 도메인 제약}: \texttt{file://} 로 실행하는 로컬 파일 환경에서는 Kakao Local REST API 호출이 제한될 수 있다. 등록된 도메인 환경에서 정상 동작하는 것을 기준으로 한다.
\item \textbf{Google Places API 과금}: Places API는 무료 한도 초과 시 과금이 발생하므로, 서비스 규모와 호출량을 기준으로 별도 비용 산정이 필요하다.
\end{itemize}


\DocNeedspace{8\baselineskip}
\subsection*{8.2 가정사항}
\addcontentsline{toc}{subsection}{8.2 가정사항}


\begin{itemize}[leftmargin=2.0em, itemsep=0.4em]
\item 서비스 초기에는 단일 HTML 파일 기반 데모로 운영하며, API 키는 사용자가 직접 입력한다.
\item 초기 데이터는 정적 파일 또는 외부 API 기반으로 관리하며, 저장 구조는 별도 데이터 설계 단계에서 결정한다.
\item 추천 엔진은 RAG(검색증강) + 멀티스텝 에이전트 구조로 구현하며, LLM API를 활용한다.
\item 자체 모델 학습(sLLM)은 본 범위에서 제외하며, 기성 LLM API + RAG + 에이전트 설계로 품질을 확보한다.
\item 축제·계절 정보는 월·계절 단위로만 관리하여 연도별 갱신 부담과 환각 위험을 최소화한다.
\item 이동 동선은 방문 순서 제안 수준으로 한정하며, 실시간 교통·항공편 최적화는 제외한다.
\item WeatherAPI는 목적지 상세 화면의 현재 날씨 표시용으로 사용하며, 추천 후보 스코어링과 대체 일정 분기에는 월별 기상 경향 데이터를 사용한다.
\item PoC 단계의 개인화 데이터는 브라우저 로컬 스토리지 중심으로 관리하며, Production 단계에서는 로그인 사용자 계정 기반으로 누적·조회·삭제할 수 있도록 확장한다.
\item 위치 반경 추천은 사용자의 명시적 위치 동의 또는 직접 입력한 기준 위치를 전제로 하며, 국가 자동 선택보다 사용자가 지정한 반경 안의 소도시 후보를 좁히는 것을 목적으로 한다.
\item 여행 중 즉흥 일정 조정은 현재 여행 중 남은 예산·남은 일정 기반 재조정이며, 전체 예산을 먼저 입력해 여행지를 역산 추천하는 예산 역산 추천과 구분한다.
\end{itemize}
