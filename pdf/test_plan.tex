\DocNeedspace{10\baselineskip}
\section*{4. 공통 검증 원칙}
\addcontentsline{toc}{section}{4. 공통 검증 원칙}
\addtocontents{toc}{\protect\TocNoChildTight}


\begin{itemize}
\item 테스트 범위는 요구사항 ID, Agent 입출력, API 계약, 데이터 모델과 연결한다.
\item 부분 평가 문서는 전체 시스템 합격으로 해석하지 않는다.
\item 실행 결과는 통과·실패 수치뿐 아니라 범위, 제외 항목, 해석 한계를 함께 기록한다.
\item LLM judge나 합성 fixture를 사용하는 경우 실제 사용자 검증을 대체하지 않는다고 명시한다.
\item 성능, 비용, observability 항목은 기술 명세서와 배포·운영 문서의 기준을 따른다.
\end{itemize}
