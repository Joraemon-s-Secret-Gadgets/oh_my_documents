\subsection*{1.2 수집 원칙}
\addcontentsline{toc}{subsection}{1.2 수집 원칙}


\begin{itemize}
\item \textbf{전체 취득}: City, Attraction, Festival의 정의된 필드와 국가별 방문·관광 통계 보조 데이터는 최초 수집 대상에 포함한다.
\item \textbf{우선 취득 범위}: 한국은 강원과 경북의 40개 도시를 우선적으로 취득한 뒤 일본의 관동 지역 지자체를 수집한다.
\item \textbf{공식 출처 우선}: 관광지 및 축제 정보는 양국 공식 관광 데이터, 공공데이터, 지자체 공식 사이트, 공식 관광 포털을 우선한다.
\item \textbf{최신성 관리}: 운영시간, 운영기간, 입장료, 축제 기간처럼 변경 가능성이 높은 데이터는 확인일과 출처를 함께 저장한다.
\item \textbf{일관성 유지}: 서로 다른 출처의 데이터를 통합할 때 명칭, 행정구역 코드, 좌표, 날짜 포맷의 매핑 기준을 통일한다.
\item \textbf{출처 추적}: 모든 자동 수집 데이터는 \texttt{source\_\allowbreak{}name}, \texttt{source\_\allowbreak{}url}, \texttt{collected\_\allowbreak{}at}, \texttt{data\_\allowbreak{}confidence}를 기록한다.
\end{itemize}

\DocNeedspace{8\baselineskip}
\subsection*{2.1 프로젝트 주제}
\addcontentsline{toc}{subsection}{2.1 프로젝트 주제}


\DocNeedspace{3\baselineskip}
로브(Lovv)는 대도시 편중 현상(오버투어리즘)을 완화하고 여행자에게 덜 알려진 한국(시/군/구 단위) 및 일본(시/정/촌/구 단위)의 매력적인 지역과 그에 최적화된 여행 일정 및 문화 콘텐츠를 추천하는 서비스이다.


\DocNeedspace{3\baselineskip}
본 계획서는 한국과 일본의 지역 기반 추천을 위해 도시, 관광지, 축제 데이터를 공통 구조로 취득하고 국가별 출처 차이를 매핑하는 방안을 다룬다.

\subsection*{2.2 데이터 모델}
\addcontentsline{toc}{subsection}{2.2 데이터 모델}


\DocNeedspace{3\baselineskip}
추천 DB는 국가별 수집 범위 확장과 검증 상태 변화에 대응할 수 있도록 Dynamic 데이터 모델로 구축한다.
상위 행정 단위는 여러 City를 가지며, City 아래에 관광지·축제·방문객 통계가 연결된다.


\DocNeedspace{3\baselineskip}
데이터 관계는 Prefecture를 상위 행정 단위로 두고, 하나의 Prefecture가 여러 City를 포함하는 구조로 관리한다.

\DocNeedspace{3\baselineskip}
각 City는 여러 Attraction, Festival, VisitorStatistics를 하위 데이터로 가지며, 추천·일정·통계 보조 지표는 모두 City를 기준으로 연결한다.

기후\newline 정합성 & City \texttt{climate}는 Wikipedia 취득값을 기준으로 하되 한국은 기상청, 일본은 일본기상청(JMA) 자료와 비교해 불일치 여부를 기록한다. \\
링크\newline 유효성 & 공식 URL과 딥링크는 HTTP 상태와 리다이렉션을 주기적으로 점검한다. \\
저작권 & 사진과 설명문은 사용 가능 조건을 확인한다. 설명문은 내부 요약문으로 저장한다. \\
다국어\newline 매핑 & 일본 지역명·축제명은 일본어 원문과 한국어 표기가 같은 대상을 가리키는지 대조한다. \\
\bottomrule
\end{longtable}
\end{small}

\section*{5. 법적 요소 검토}
\addcontentsline{toc}{section}{5. 법적 요소 검토}
\addtocontents{toc}{\protect\TocNoChildTight}


\DocNeedspace{3\baselineskip}
공공데이터 및 오픈 백과사전을 활용하는 만큼 기본 라이선스 기준에 근거하여 법적 의무 사항을 준수한다.


\begin{itemize}
\item \textbf{위키피디아 라이선스 준수}: 위키피디아 한국어/일본어 버전의 텍스트 콘텐츠는 CC BY-SA 라이선스 조건에 따라 출처와 원문 링크를 기록한다.
\item \textbf{한국 공공데이터 활용 조건 준수}: 한국관광공사 TourAPI, 대한민국구석구석, 공공데이터포털, 지자체 데이터는 공공누리 유형과 API 이용 조건을 확인한다.
\item \textbf{일본 공식·상업 플랫폼 제약 준수}: JNTO, JTA, 지자체 관광 사이트, Yahoo Japan, Jalan, Rakuten Travel, Tabelog 등은 이용약관과 크롤링 허용 범위를 확인한다.
\item \textbf{사진 사용 조건 확인}: 사진은 저작권 위험이 크므로 자동 수집 값이 있더라도 사용 가능 조건을 확인하고, 불명확하면 \texttt{needs\_\allowbreak{}review} 또는 \texttt{blocked}로 관리한다.
\item \textbf{변동 정보 표시}: 운영시간, 입장료, 축제 기간은 변경 가능성이 높으므로 서비스 화면에 "확인일 기준" 문구를 표시한다.
\end{itemize}

\section*{6. 수집 정책 및 제외 범위}
\addcontentsline{toc}{section}{6. 수집 정책 및 제외 범위}
\addtocontents{toc}{\protect\TocNoChildTight}


\begin{itemize}
\item \textbf{수집 주기}: City 기본 정보는 연 1회 이상, 운영시간·입장료·축제 기간은 최신성 상태에 따라 재확인한다.
\item \textbf{Raw 재사용}: 수집 JSON은 S3 Raw Bucket에 일정 기간 보관하여 재처리, 검수 기준 변경, DynamoDB 재적재에 재사용한다.
\item \textbf{전처리 실행}: Raw 데이터가 일정 기간 또는 처리 기준만큼 누적되면 Lambda 배치 전처리를 실행하고 DynamoDB에 정규화 결과를 적재한다.
\item \textbf{실시간 응답 보강}: DB 값이 없거나 오래된 경우 Web Search Worker가 공식 사이트 또는 공공 관광 페이지를 확인한다.
\item \textbf{상업 플랫폼 데이터}: 숙박·맛집 플랫폼 데이터는 직접 저장보다 검색 딥링크 제공을 우선한다.
\item \textbf{본문 원문 저장 제한}: 외부 설명문은 원문 전체 복제가 아니라 내부 요약문으로 저장한다.
\end{itemize}
