\section*{1. 목적}
\addcontentsline{toc}{section}{1. 목적}


\DocNeedspace{3\baselineskip}
본 문서는 여행 추천 Multi-Agent 서비스에서 일본 소도시 추천에 필요한 도시·관광지·축제 데이터를 한 번에 취득하기 위한 범위, 데이터 소스, 검증 방식, 저장 구조를 정의한다.


\DocNeedspace{3\baselineskip}
이번 초안의 핵심은 다음과 같다.


\begin{itemize}
\item City, Attraction, Festival의 정의된 항목을 모두 수집 대상으로 둔다.
\item 도도부현의 간략 정보와 산하 도시 목록을 먼저 취득하고, 산하 도시 목록에 포함된 City만 Wikipedia 문서 크롤링 대상으로 둔다.
\item 취득 결과는 JSON 문서로 저장한 뒤 S3 Raw Bucket에 적재한다.
\item 운영시간, 운영기간, 입장료, 사진처럼 출처별 표현 차이가 큰 항목도 최초 취득 범위에 포함한다.
\item 자동 수집, 공식 사이트 확인, Web Search Worker, 수동 검수를 하나의 취득 파이프라인으로 묶어 처리한다.
\item 추천 DB는 \texttt{City 1:\allowbreak{}N Attraction}, \texttt{City 1:\allowbreak{}N Festival} 관계를 기준으로 설계한다.
\end{itemize}

\begin{itemize}
\item 도시명은 \texttt{city\_\allowbreak{}name\_\allowbreak{}ja}, \texttt{city\_\allowbreak{}name\_\allowbreak{}ko}, \texttt{city\_\allowbreak{}name\_\allowbreak{}en}을 분리 저장한다.

\item 기간 정보는 원문 문자열(\texttt{period\_\allowbreak{}text})과 정규화 값(\texttt{start\_\allowbreak{}date}, \texttt{end\_\allowbreak{}date}, \texttt{month})을 함께 저장한다.

\item 외부 설명문은 원문 복제가 아니라 내부 요약문으로 재작성한다.
\end{itemize}

\begin{scriptsize}
\ttfamily
\RaggedRight
\setlength{\parskip}{1pt}
\textcolor{LovvGreen}{\textbf{text}}\\[0.2em]

↓\\
JSON 직렬화\\
↓\\

↓\\

\section*{9. 법적·운영 제약}
\addcontentsline{toc}{section}{9. 법적·운영 제약}


\begin{itemize}
\item Wikipedia 기반 설명은 라이선스 조건에 따라 출처와 원문 링크를 기록한다.
\item JNTO, JTA, 지자체 관광 페이지는 이용약관과 크롤링 허용 범위를 확인한다.
\item 사진은 저작권 위험이 크므로 자동 수집 값이 있더라도 사용 가능 조건을 확인하고, 불명확하면 \texttt{needs\_\allowbreak{}review} 또는 \texttt{blocked}로 관리한다.
\item 상업 플랫폼의 숙박·맛집 데이터는 직접 저장보다 딥링크 제공을 우선한다.
\item 운영시간과 입장료는 변경 가능성이 높으므로 "확인일 기준" 문구를 서비스에 표시한다.
\end{itemize}
