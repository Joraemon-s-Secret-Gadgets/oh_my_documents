\section*{1. 목적}
\addcontentsline{toc}{section}{1. 목적}
\addtocontents{toc}{\protect\TocNoChildTight}


\DocNeedspace{3\baselineskip}
본 문서는 여행 추천 Multi-Agent 서비스에서 일본 소도시 추천에 필요한 도시·관광지·축제·방문객 통계 데이터를 한 번에 취득하기 위한 범위, 데이터 소스, 검증 방식, 저장 구조를 정의한다.


\DocNeedspace{3\baselineskip}
이번 문서의 핵심은 다음과 같다.


\begin{itemize}
\item City, Attraction, Festival, VisitorStatistics의 정의된 항목을 모두 수집 대상으로 둔다.
\item 일본은 관동 지역 지자체를 우선 수집 범위로 둔다.
\item 도도부현의 간략 정보와 산하 도시 목록을 먼저 취득하고, 산하 도시 목록에 포함된 City만 Wikipedia 문서 크롤링 대상으로 둔다.
\item 취득 결과는 JSON 문서로 저장한 뒤 S3 Raw Bucket에 적재한다.
\item 로컬 검증 산출물은 \texttt{data/\allowbreak{}JP/\allowbreak{}*.\allowbreak{}json} 형태로 관리한다.
\item 운영시간, 운영기간, 입장료, 사진처럼 출처별 표현 차이가 큰 항목도 최초 취득 범위에 포함한다.
\item 자동 수집, 공식 사이트 확인, Web Search Worker, 수동 검수를 하나의 취득 파이프라인으로 묶어 처리한다.
\item 추천 DB는 \texttt{City 1:\allowbreak{}N Attraction}, \texttt{City 1:\allowbreak{}N Festival}, \texttt{City 1:\allowbreak{}N Visitor\allowbreak{}Statistics} 관계를 기준으로 설계한다.
\end{itemize}

\begin{itemize}
\item 도시명은 \texttt{city\_\allowbreak{}name\_\allowbreak{}ja}, \texttt{city\_\allowbreak{}name\_\allowbreak{}ko}, \texttt{city\_\allowbreak{}name\_\allowbreak{}en}을 분리 저장한다.

\item 관광지와 축제는 반드시 \texttt{city\_\allowbreak{}id}를 가진다.
\item 기간 정보는 원문 문자열(\texttt{period\_\allowbreak{}text})과 정규화 값(\texttt{start\_\allowbreak{}date}, \texttt{end\_\allowbreak{}date}, \texttt{month})을 함께 저장한다.

\item 외부 설명문은 원문 복제가 아니라 내부 요약문으로 재작성한다.
\end{itemize}

\begin{scriptsize}

\textcolor{LovvGreen}{\textbf{text}}\\[0.2em]

\DocNeedspace{3\baselineskip}
수집 원본은 DB에 바로 쓰지 않고 JSON 파일로 저장한 뒤 S3 Raw Bucket에 적재한다.\newline S3 객체는 국가, 출처, 엔티티 유형, 수집일 기준 Prefix로 구분한다.\newline 일정 기간 누적된 Raw Prefix는 Lambda 배치 전처리의 입력으로 재사용하며, 전처리 결과만 DynamoDB에 정규화 Item으로 적재한다.

\item 운영시간과 입장료는 변경 가능성이 높으므로 "확인일 기준" 문구를 서비스에 표시한다.

\section*{10. 운영 반영 방향}
\addcontentsline{toc}{section}{10. 운영 반영 방향}
\addtocontents{toc}{\protect\TocNoChildTight}


\DocNeedspace{3\baselineskip}
본 문서의 핵심 내용은 일본 데이터 취득 계획의 실제 운영 기준으로 정리한다.

\DocNeedspace{3\baselineskip}
문서 확정 후에는 대표 데이터 취득 계획, 전처리 계획, 공유용 HTML, PDF 산출물이 같은 기준을 바라보도록 순차적으로 동기화한다.


\begin{itemize}
\item 일본 우선 수집 범위는 관동 지역 지자체를 기준으로 시작하고, 이후 동일한 구조로 다른 지역을 확장한다.
\item City는 도도부현 간략 정보와 산하 도시 목록을 먼저 확보한 뒤 산하 City 문서만 크롤링한다.
\item Attraction과 Festival은 JNTO, JTA, 공식 관광 사이트를 우선하고 누락 항목은 Web Search Worker 또는 수동 검수로 채운다.
\item VisitorStatistics는 JNTO Statistics, e-Stat, RESAS의 제공 가능한 집계 단위를 City 보조 지표로 연결한다.
\item 수집 결과는 \texttt{data/\allowbreak{}JP/\allowbreak{}*.\allowbreak{}json} 로컬 검증 산출물로 확인한 뒤 S3 Raw Prefix 적재 대상으로 확정한다.
\item 공유용 HTML과 PDF 산출물은 확정된 데이터 취득 계획을 사용자가 바로 확인할 수 있는 형태로 갱신한다.
\end{itemize}
