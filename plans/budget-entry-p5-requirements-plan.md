# 예산 역산 추천 P5 정리 계획

## 목적

`FR-ENTRY-BUDGET-*` 예산 역산 추천 진입점을 P5 항목으로 명확히 정리한다.

P5는 요구기능 상세 표에서 제외하고, 4.6의 `우선순위 및 단계별 추진 범위` 표에만 기록한다. 따라서 `FR-ENTRY-BUDGET-*`는 상세 기능 요구사항 표에서 제거하고, 예산 역산 추천은 P5 행에서만 추적한다.

## 현재 상태

- 4.6 우선순위 요약 표의 P5 행에 `예산 역산 추천`이 포함되어 있다.
- 4.6 상세 기능 요구사항 표에서는 P5 기능 ID를 제외한다.
- 1.2 범위와 3.1 서비스 소개에서는 예산 역산 추천이 이번 프로젝트 대상에서 제외되고 향후 검토된다고 설명되어 있다.
- 4.4 `FR-RAG-005`는 날씨 또는 시기 조건 중심의 직접 질의 진입점으로 유지한다.

## 수정 방향

### 1. 4.6 상세 기능 요구사항에서 P5 제외

`FR-ENTRY-BUDGET-*`의 상세 요구사항 행은 제거한다.

같은 기준으로 P5에 해당하는 다음 상세 행도 제거한다.

- `FR-ASTRO-*`
- `FR-THEME-WELLNESS-*`
- `FR-ENTRY-BUDGET-*`

P5 기능은 삭제하지 않고 4.6 상단 `우선순위 및 단계별 추진 범위` 표의 P5 행에서만 기록한다.

### 2. `FR-ENTRY-BUDGET-*`의 P5 성격 명확화

상세 표에 P5 기능 ID를 남기지 않는다.

대신 4.6 설명 문단과 P5 요약 행에서 다음 의미가 명확해야 한다.

- 예산 역산 추천은 P5 검토 항목이다.
- PoC 구현 범위가 아니다.
- 이번 요구기능 상세 표에는 포함하지 않는다.

### 3. `FR-RAG-005` 문구 정리

현재:

```text
시스템은 사용자가 테마를 고르지 않고 사용자 여행 예산, 기간, 인원, 날씨, 시기 조건을 바로 입력해 추천을 시작할 수 있도록 지원해야 한다.
비고: budget-first/weather-first
```

수정 방향:

```text
시스템은 사용자가 테마를 고르지 않고 날씨 또는 시기 조건을 바로 입력해 추천을 시작할 수 있도록 지원해야 한다.
비고: weather-first 중심
```

목표:

- P1인 날씨 기반 직접 질의 진입점과 P5인 예산 역산 진입점을 분리한다.
- `FR-RAG-005`가 예산 역산을 현재 구현 범위로 포함하는 것처럼 보이지 않게 한다.

### 4. 3.2 핵심 가치 제안 표현 확인

직접 질의형 추천 설명은 현재 다음 의미를 유지한다.

- 날씨 조건 기반 역제안은 현재 범위에 포함된다.
- 예산 역산은 향후 검토 대상이다.

필요 시 다음처럼 더 명확히 정리한다.

```text
직접 질의형 추천: 사용자가 날씨 조건을 바로 말하면 테마 선택 없이 가능한 소도시를 역으로 제안한다. 예산 역산은 P5 후보로 향후 검토한다.
```

### 5. 변경 이력 문구 정리

현재:

```text
지역 문화 체험, 치유·웰니스, 사계절 액티비티 확장 테마와 예산 역산·날씨 기반 직접 질의 진입점 반영
```

수정 방향:

변경 이력은 현재 문서 상태를 설명하는 수준으로 유지하되, 예산 역산이 현재 상세 요구기능으로 오해되지 않도록 필요 시 `예산 역산은 P5 추진 범위로만 기록`이라고 보강한다.

목표:

- 예산 역산이 날씨 기반 직접 질의와 같은 우선순위처럼 보이지 않게 한다.

## 완료 기준

- `FR-ENTRY-BUDGET-*`는 4.6 상세 기능 요구사항 표에 남아 있지 않다.
- `FR-ASTRO-*`, `FR-THEME-WELLNESS-*`도 상세 기능 요구사항 표에 남아 있지 않다.
- P5 기능 범위는 4.6 우선순위 및 단계별 추진 범위 표에만 기록되어 있다.
- `FR-RAG-005`가 예산 역산을 현재 직접 구현 범위처럼 포함하지 않는다.
- `budget-first/weather-first` 표현이 현재 구현 범위처럼 남아 있지 않다.
- 날씨 기반 직접 질의는 P1, 예산 역산은 P5 추진 범위 기록 항목이라는 차이가 문서에서 읽힌다.
- Markdown과 HTML의 의미가 동기화되어 있다.

## 주의 사항

- `FR-ENTRY-BUDGET-*` 기능 ID는 상세 요구기능 표에서 제외한다.
- P5 항목은 추적을 위해 우선순위 및 단계별 추진 범위 표에 유지한다.
- 상세 기능 표에 `P5`, `제외`, `향후 검토` 같은 단계 문구를 반복하지 않는다.
- 예산 역산 추천은 프로젝트 예산안이 아니라 사용자 여행 예산 기반 추천 기능임을 유지한다.
