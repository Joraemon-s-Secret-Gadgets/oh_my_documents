# 4.6 미디어 성지순례 및 철도 경로 요구사항 정리 계획

## 목적

4.6 서비스별 세부 기능에서 P2의 `FR-KDRAMA-*`와 `FR-ANIME-*`를 하나의 테마로 묶고, P3의 철도 패스 기반 기능을 "패스 비용 비교" 중심에서 "철도 경로 안내와 주변 관광지 안내" 중심으로 정리한다.

이번 작업은 요구사항 정의서 수준의 정리이며, 구현 방식이나 API 상세 설계는 포함하지 않는다.

## 반영할 결정 사항

- K-Drama 촬영지와 애니메이션 성지순례는 별도 서비스가 아니라 하나의 `미디어 촬영지·성지순례` 테마로 묶는다.
- 한국은 K-Drama 촬영지 중심, 일본은 애니메이션 성지순례 중심으로 국가별 대표 콘텐츠를 구분한다.
- 두 기능은 작품명, 장소, 공식성, 신뢰도, 근거 URL, 주변 소도시 여정 연결이라는 공통 구조를 가진다.
- 철도 패스 기반 기능은 패스 구매비와 절약 금액 비교만 강조하지 않는다.
- 철도 기반 경로 추천은 이동 경로를 안내하면서 역 또는 경로 주변의 관광지, 소도시, 체험 요소를 함께 안내하는 방향으로 정리한다.

## 수정 방향

### 1. P2 요약 표 문구 정리

현재:

```text
K-Drama 촬영지 탐색, 애니메이션 성지순례
```

수정 방향:

```text
미디어 촬영지·성지순례 추천(K-Drama 촬영지, 애니메이션 성지순례)
```

목표:

- K-Drama와 Anime가 같은 테마 안의 국가별 대표 사례라는 점을 드러낸다.
- 기존 기능 범위를 삭제하지 않고 상위 테마로 묶는다.
- P2의 "소도시 촬영지 중심 추천" 의미를 상세 기능과 연결한다.

### 2. P2 상세 기능 ID 유지 및 영역명 통합

기능 ID는 변경하지 않는다.

- `FR-KDRAMA-*`는 유지한다.
- `FR-ANIME-*`는 유지한다.
- 서비스/영역명은 통합 테마가 보이도록 정리한다.

수정 예시:

| ID | 현재 서비스/영역 | 수정 방향 |
| --- | --- | --- |
| FR-KDRAMA-001 | K-drama 촬영지 탐색·검증 | 미디어 촬영지·성지순례 |
| FR-KDRAMA-002 | K-drama 촬영지 탐색·검증 | 미디어 촬영지·성지순례 |
| FR-ANIME-001 | 애니메이션 성지순례 | 미디어 촬영지·성지순례 |
| FR-ANIME-002 | 애니메이션 성지순례 | 미디어 촬영지·성지순례 |
| FR-ANIME-003 | 애니메이션 성지순례 | 미디어 촬영지·성지순례 |

요구사항 내용에는 한국/K-Drama, 일본/애니메이션 구분을 문장 안에서 유지한다.

### 3. P2 요구사항 문구 보강

정리 기준:

- `FR-KDRAMA-001`: 한국 K-Drama 촬영지 데이터의 수집·정제·검증 요구사항으로 유지한다.
- `FR-KDRAMA-002`: 촬영지를 소도시 여정 중심으로 탐색할 수 있어야 한다는 의미를 보강한다.
- `FR-ANIME-001`: 일본 애니메이션 작품명·지역 조건 기반 추천으로 유지한다.
- `FR-ANIME-002`: 장소 관계 유형, 공식성, 신뢰도, 근거 URL 표시 요구사항으로 유지한다.
- `FR-ANIME-003`: 성지 후보를 주변 소도시 관광지, 온천, 미식, 교통 조건과 결합하는 여정 구성 요구사항으로 유지한다.

주의:

- K-Drama와 Anime를 합친다고 기능 ID를 하나로 병합하지 않는다.
- 국가별 콘텐츠 특성이 사라지지 않도록 문장 안에 구분을 남긴다.

### 4. P3 요약 표 문구 정리

현재:

```text
철도 패스 기반 소도시 경로 추천, 지역 문화 체험 추천
```

수정 방향:

```text
철도 경로 기반 주변 관광지 안내, 지역 문화 체험 추천
```

목표:

- 철도 패스를 비용 비교 기능으로만 보지 않도록 한다.
- 경로 안내와 주변 관광지 안내가 핵심임을 드러낸다.
- 패스 정보는 경로 안내를 보조하는 정보로 유지한다.

### 5. P3 철도 상세 요구사항 문구 정리

수정 대상:

- `FR-RAIL-001`
- `FR-RAIL-002`

수정 방향:

| ID | 현재 초점 | 수정 방향 |
| --- | --- | --- |
| FR-RAIL-001 | 철도망, 교통 패스, 관광 상품 결합 | 철도 경로를 기준으로 이동 가능한 소도시와 경로 주변 관광지를 안내 |
| FR-RAIL-002 | 패스권 구매비, 절약 금액, 추천 경로 비교 | 경로별 역, 환승, 소요 시간, 주변 관광지, 패스 활용 가능 여부를 함께 표시 |

비고 방향:

- `출발지·이동시간·환승 난이도 반영`은 유지 가능하다.
- `패스 가격과 구간 요금 갱신 기준 필요`는 `패스 적용 가능 구간과 주변 관광지 연결 기준 필요`처럼 바꾼다.

### 6. Markdown과 HTML 동기화

- Markdown 변경 후 HTML의 4.6 영역을 동일한 의미로 수정한다.
- HTML의 행 순서와 기능 ID는 Markdown과 동일하게 유지한다.
- 4.7 운영·관리·검증 기능은 이번 작업에서 수정하지 않는다.

## 작업 순서

1. `docs/01_requirements/01_requirements.md`의 4.6 요약 표에서 P2/P3 기능 범위 문구를 수정한다.
2. `FR-KDRAMA-*`, `FR-ANIME-*`의 서비스/영역명을 통합 테마로 정리한다.
3. P2 상세 요구사항 문구에서 국가별 구분과 소도시 여정 연결성을 보강한다.
4. `FR-RAIL-*`의 서비스/영역명과 요구사항 내용을 철도 경로 기반 주변 관광지 안내 방향으로 수정한다.
5. `pages/01_requirements.html`의 동일 영역을 Markdown과 맞춘다.
6. 기능 ID 누락, 순서, 문구 불일치를 검증한다.

## 완료 기준

- P2 요약 표가 K-Drama와 Anime를 하나의 미디어 촬영지·성지순례 테마로 표현한다.
- `FR-KDRAMA-*`, `FR-ANIME-*`가 같은 서비스/영역명으로 묶여 있다.
- P2 요구사항 내용에서 한국/K-Drama와 일본/Anime의 구분은 유지되어 있다.
- P3 요약 표가 철도 경로와 주변 관광지 안내 중심으로 표현되어 있다.
- `FR-RAIL-*`가 패스 비용 비교만이 아니라 경로, 역, 환승, 주변 관광지 안내를 포함한다.
- Markdown과 HTML의 4.6 내용이 같은 의미로 동기화되어 있다.
