# MD/HTML 개행 원복 및 내용 반영 유지 계획

## 목적

`docs/01_requirements/01_requirements.md`와 `pages/01_requirements.html`에 반영된 변경 중 PDF 레이아웃 조정을 위해 들어간 단순 개행 변경은 되돌리고, 요구사항 내용 자체가 변경된 항목은 유지한다.

## 대상 파일

- `docs/01_requirements/01_requirements.md`
- `pages/01_requirements.html`

## 되돌릴 항목

아래 항목은 문서 내용 변경이 아니라 표 안의 줄바꿈을 위한 표시 변경이므로 원문 형태로 되돌린다.

| 파일 | 현재 값 | 되돌릴 값 | 사유 |
| --- | --- | --- | --- |
| Markdown | `멀티스텝<br>에이전트` | `멀티스텝 에이전트` | 단순 개행 처리 |
| HTML | `멀티스텝<br>에이전트` | `멀티스텝 에이전트` | 단순 개행 처리 |
| Markdown | `사용자 시나리오 /<br>유저 스토리` | `사용자 시나리오 / 유저 스토리` | 단순 개행 처리 |
| HTML | `사용자 시나리오 /<br>유저 스토리` | `사용자 시나리오 / 유저 스토리` | 단순 개행 처리 |

## 유지할 항목

아래 항목은 요구사항 내용 반영에 해당하므로 유지한다.

| 항목 | 유지 사유 |
| --- | --- |
| `Agent 설계` 후속 문서 추가 | 후속 문서 범위 확장 |
| `데이터 취득 계획서` 후속 문서 추가 | 후속 문서 범위 확장 |
| `FR-COM-001` 비고를 `여행 조건 입력 통합`으로 변경 | 기존 비고를 문서 톤에 맞게 정리한 내용 변경 |
| `FR-JP-005` 비고를 `P2`로 정리 | 예시를 비고에서 분리한 내용 구조 변경 |
| `FR-JP-005` 일본어 검색 키워드 예시 캡션 추가 | 표 하단 캡션으로 예시를 유지하기 위한 내용 반영 |

## 작업 체크리스트

- [x] Markdown에서 단순 개행 변경을 원복한다.
- [x] HTML에서 단순 개행 변경을 원복한다.
- [x] Markdown과 HTML의 내용 반영 항목은 그대로 유지한다.
- [x] `git diff -- docs/01_requirements/01_requirements.md pages/01_requirements.html`로 남은 변경이 유지 대상뿐인지 확인한다.
- [x] 이번 작업에서는 TeX/PDF를 재생성하지 않는다.

## 주의 사항

`pdf/requirements_definition.tex`는 PDF 레이아웃 조정을 위해 직접 수정된 상태다. Markdown에서 TeX를 재생성하면 최근의 수작업 PDF 레이아웃 변경이 덮일 수 있으므로, 이 계획의 범위는 MD/HTML 정리에 한정한다.
