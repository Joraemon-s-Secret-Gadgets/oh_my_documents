# 서비스 진입 경로 요약 제거 계획

## 목적

`pages/01_requirements.html`의 3.4 서비스 흐름 요약 아래에 남아 있는 "서비스에는 두 가지 진입 경로가 있으며..." 설명과 진입 경로 표를 제거한다.

현재 프로젝트 기획 기준은 국가와 여행 시기를 먼저 입력하고, 이후 자연어 취향을 반영해 추천 결과를 제공하는 방향이다. 따라서 기존의 `테마 직접 선택`과 `챗봇 자연어 입력`을 별도 진입 경로로 나누는 설명은 최신 흐름과 충돌할 수 있으므로 삭제한다.

## 제거 대상

대상 파일:

- `pages/01_requirements.html`

제거 블록:

```text
서비스에는 두 가지 진입 경로가 있으며, 두 경로 모두 추천 결과 단계에서 공통 흐름으로 합류한다.
```

제거 표:

| 진입 경로 | 사용자 행동 | 시스템 처리 방식 |
| --- | --- | --- |
| 테마 직접 선택 | 6개 테마(온천·바다·역사·미식·자연·예술) 중 하나 이상 선택 | 선택 테마 기반 RAG 파이프라인 실행 → 추천 결과 표시 |
| 챗봇 자연어 입력 | 취향·여행 시기·동행자·이동수단·예산을 자연어 또는 조건 칩으로 입력 | 멀티스텝 에이전트 처리(분류 → RAG 검색 → 군집 → 여정 생성) → 추천 결과 표시 |

## 수정 계획

### 1. HTML 블록 제거

`pages/01_requirements.html`에서 다음 영역을 제거한다.

- `<!-- ── 서비스 진입 경로 요약 ── -->` 주석
- 진입 경로 설명 문단
- 진입 경로 표 전체

수정 후 3.4 서비스 흐름 표 다음에 바로 4. 기능 요구사항 섹션이 이어지도록 한다.

### 2. Markdown 원본과의 불일치 확인

`docs/01_requirements/01_requirements.md`에는 동일한 진입 경로 표가 없는 상태로 확인되었다.

수정 후 다음 기준을 확인한다.

- Markdown에는 제거 대상 문구가 없어야 한다.
- HTML에도 제거 대상 문구가 없어야 한다.
- 3.4 서비스 흐름의 기본 단계 설명은 유지되어야 한다.

### 3. 최신 서비스 흐름과 충돌하는 표현 점검

제거 후 다음 표현이 남는지 확인한다.

- `서비스에는 두 가지 진입 경로`
- `테마 직접 선택`
- `챗봇 자연어 입력`
- `조건 칩`

단, `6개 테마` 표현은 용어 정의나 변경이력처럼 과거/분류 기준을 설명하는 문맥에서는 별도 판단 없이 제거하지 않는다.

## 검증 계획

다음 명령으로 제거 여부를 확인한다.

```powershell
rg -n "서비스에는 두 가지 진입|테마 직접 선택|챗봇 자연어 입력|조건 칩" docs pages
```

기대 결과:

- `pages/01_requirements.html`에서 제거 대상 문구가 검색되지 않는다.
- `docs/01_requirements/01_requirements.md`에서도 동일 문구가 검색되지 않는다.

HTML 구조 확인:

```powershell
rg -n "3\\.4 서비스 흐름 요약|4\\. 기능 요구사항" pages\01_requirements.html
```

기대 결과:

- 3.4 섹션이 유지된다.
- 4. 기능 요구사항 섹션이 정상적으로 이어진다.

## 완료 기준

- [ ] `pages/01_requirements.html`에서 두 가지 진입 경로 설명과 표가 제거된다.
- [ ] `docs/01_requirements/01_requirements.md`와 충돌하는 별도 진입 경로 설명이 남지 않는다.
- [ ] 최신 기준인 국가·여행 시기 선입력 + 자연어 취향 반영 흐름은 유지된다.
- [ ] `rg` 검증에서 제거 대상 문구가 검색되지 않는다.
