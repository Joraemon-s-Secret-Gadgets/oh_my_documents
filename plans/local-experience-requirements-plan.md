# 지역 문화 체험 요구사항 정리 계획

## 목적

`FR-THEME-CRAFT-*` 요구사항을 공예·장인 체험에만 제한하지 않고, 지역 문화를 직접 경험하는 체험 추천으로 확장한다.

이번 작업은 요구사항 정의서 수준의 정리이며, 특정 체험 카테고리나 구현 방식에 과도하게 묶지 않는다.

## 반영할 결정 사항

- `FR-THEME-CRAFT-*` 기능 ID는 유지한다.
- 서비스/영역명은 `공예·장인 체험 여정 추천`에서 `지역 문화 체험 추천`으로 변경한다.
- 체험 예시는 공예에 한정하지 않고 차, 지역 요리, 전통문화, 농어촌 활동 등으로 확장한다.
- `완성품 수령 방식`처럼 공예에만 맞는 조건은 제거한다.
- 우천 시 대안도 `공예 등 실내 체험`이 아니라 `실내 또는 지역 문화 체험 대안`으로 표현한다.

## 수정 계획

### 1. P3 요약 문구 수정

현재:

```text
철도 경로 기반 주변 관광지 안내, 공예·장인 체험형 여행 추천
```

수정:

```text
철도 경로 기반 주변 관광지 안내, 지역 문화 체험 추천
```

### 2. `FR-THEME-CRAFT-*` 상세 요구사항 수정

| ID | 수정 방향 |
| --- | --- |
| FR-THEME-CRAFT-001 | 공예 중심 예시를 지역 문화 체험 전반으로 확장 |
| FR-THEME-CRAFT-002 | 완성품 수령 방식 대신 참여 방식, 예약 필요 여부, 우천 적합성 중심으로 정리 |
| FR-THEME-CRAFT-003 | 공예 마을 중심 표현을 주변 마을, 축제, 로컬 관광 자원으로 확장 |

### 3. 관련 문서 표현 정리

- 문서 범위와 서비스 소개에서 `공예·장인 체험` 표현을 `지역 문화 체험`으로 변경한다.
- 데이터 요구사항에서 `공예 종류`, `완성품 수령 방식`, `주변 공예 마을` 표현을 제거한다.
- 변경 이력의 확장 테마 명칭도 최신 표현으로 맞춘다.

### 4. HTML 동기화

- Markdown 변경과 같은 의미로 `pages/01_requirements.html`을 수정한다.
- 기능 ID, 행 순서, 비고 의미는 유지한다.

## 완료 기준

- `FR-THEME-CRAFT-*`의 서비스/영역명이 `지역 문화 체험 추천`으로 통일되어 있다.
- 공예에만 해당하는 조건이 상세 요구사항에서 제거되어 있다.
- 지역 문화 체험의 예시가 공예, 차, 지역 요리, 전통문화, 농어촌 활동 등으로 넓게 표현되어 있다.
- Markdown과 HTML의 표현이 같은 의미로 동기화되어 있다.
- 기능 ID는 변경되지 않았다.

## 주의 사항

- 기능 ID는 추적성을 위해 변경하지 않는다.
- `CRAFT`라는 ID 접두어가 남아 있더라도, 현재 단계에서는 기능 ID 변경보다 요구사항 의미 확장을 우선한다.
- 특정 체험 유형을 과도하게 나열하지 않고, 예시 수준으로만 둔다.
