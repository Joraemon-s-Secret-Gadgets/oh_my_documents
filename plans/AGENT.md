# Plans Agent Instructions

이 디렉터리는 작업 계획, 문서 반영 계획, 복구 계획, 검증 계획을 Markdown으로 기록한다.

## Folder Rule

계획 문서는 진행 상태에 따라 아래 폴더로 분류할 수 있다.

- `before/`: 아직 실행하지 않은 계획
- `during/`: 현재 실행 중이거나 일부만 완료된 계획
- `after/`: 실행과 검증이 완료된 계획

루트 `plans/`에는 공통 지침, 인덱스, 임시 계획만 둔다. 상태가 확정된 계획은 가능한 한 위 상태 폴더 중 하나에 배치한다.

## Plan Metadata Rule

새로운 계획 문서에는 문서 상단에 YAML front matter로 작성자와 상태를 기록한다.

권장 형식:

```markdown
---
작성자: llm팀
상태: 진행전
---
```

`상태` 값은 `진행전`, `진행중`, `진행후` 중 하나를 사용한다.

## Plan Format Rule

새로운 계획 문서를 작성할 때는 실행 항목을 반드시 체크박스 목록으로 포함한다.

권장 형식:

```markdown
## 작업 체크리스트

- [ ] 변경 대상 파일 확인
- [ ] 유지할 내용과 되돌릴 내용을 분리
- [ ] 필요한 파일 수정
- [ ] diff 검증
- [ ] 결과 요약
```

## Writing Rules

- 계획 문서는 목적, 대상 파일, 유지할 항목, 변경할 항목, 작업 체크리스트, 검증 방법을 포함한다.
- 계획 문서에는 `작성자`와 `상태` 속성을 포함한다.
- 체크박스는 실제 작업 순서대로 작성한다.
- 완료된 작업을 반영하는 계획 문서라면 이미 끝난 항목은 `- [x]`로 표시할 수 있다.
- 단순 설명 문단만으로 계획을 끝내지 않는다.
- PDF 또는 HTML 생성물이 관련된 계획에는 원본 문서와 생성물의 관계를 명시한다.

## Caution

`plans/`의 문서는 작업 지침과 이력 기록이다. 계획 문서를 수정하는 것만으로 실제 `docs/`, `pages/`, `pdf/` 파일을 변경한 것으로 간주하지 않는다.
